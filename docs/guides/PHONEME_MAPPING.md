# 音素マッピングについて

## 現在の実装

現在のPiper日本語実装では、OpenJTalkが生成する音素を以下のように処理しています：

### 音素セット（58音素）
- **基本母音**: a, i, u, e, o（5音素）
- **無声化母音**: A, I, U, E, O（5音素）- 日本語特有の無声化現象に対応
- **その他**: 子音、長母音、特殊記号など（48音素）

### Private Use Area (PUA) マッピング
複数文字の音素（例：「ch」「ts」「sh」）は、Unicode Private Use Area（U+E000〜）にマッピングされ、内部的に単一のコードポイントとして処理されます。

### 処理例
- 「ち」: "ch" → U+E00E（PUA）
- 「つ」: "ts" → U+E00F（PUA）
- 「し」: "sh" → U+E010（PUA）
- 「です」: "d e s U" → "d e s U"（「す」の母音Uは無声化）

## モデルの互換性

### 最新モデル（58音素対応）

最新のPiper日本語モデルは無声化母音を含む58音素に対応しています：

1. **音素数**: 58音素（従来の53音素から拡張）
2. **主な改善点**:
   - 無声化母音（A, I, U, E, O）の正確な表現
   - 日本語の自然な韻律の再現
   - より高品質な音声合成

### 旧モデル（53音素）との互換性

旧モデルは無声化母音を小文字に変換して処理していたため、以下の制限があります：

1. **音素の区別**
   - 無声化母音が通常の母音として処理される
   - 「雨（あめ）」と「飴（あめ）」のような微妙な違いが表現できない

2. **使用は可能**
   - 基本的な音声合成は問題なく動作します
   - ただし、最新モデルに比べて自然性が劣る場合があります

### より高品質な音声を求める場合

以下の2つの選択肢があります：

#### 選択肢1: モデルの再学習（推奨）

Python側の前処理を修正して、複数文字音素を適切に扱うようにします：

```python
# preprocess.py の修正例
def process_phonemes(phonemes):
    # OpenJTalkの複数文字音素を保持
    # "ch", "ts", "sh" などをそのまま使用
    return phonemes  # 分割しない
```

#### 選択肢2: 音素マッピングテーブルの使用

C++側で音素マッピングテーブルを実装：

```cpp
// 音素マッピングの例
std::map<std::string, std::string> phoneme_map = {
    {"ch", "t"},  // "ch" を "t" にマッピング
    {"ts", "t"},  // "ts" を "t" にマッピング
    {"sh", "s"},  // "sh" を "s" にマッピング
};
```

## 推奨事項

### 新規プロジェクトの場合
- Python側とC++側で同じ音素体系を使用
- OpenJTalkの音素をそのまま使用することを推奨

### 既存モデルを使用する場合
- 現在の実装（音素分割）で問題なく動作
- 警告は無視して構いません

### 高品質な商用利用の場合
- モデルの再学習を検討
- 音素マッピングの最適化を実施

## 技術詳細

|------|-----------|-----------|
| ch   | ち        | U+E00E    |
| ts   | つ        | U+E00F    |
| sh   | し        | U+E010    |
| ky   | きゃ      | U+E006    |
| ny   | にゃ      | U+E013    |
| hy   | ひゃ      | U+E012    |

### 無声化母音の例

| 通常 | 無声化 | 例          |
|------|--------|-------------|
| a    | A      | ました (shitA)|
| i    | I      | しき (shikI) |
| u    | U      | です (desU)  |
| e    | E      | せき (sEki)  |
| o    | O      | こと (kOto)  |

### 音素IDマッピングの仕組み

1. OpenJTalkがテキストから音素列を生成（無声化母音は大文字で出力）
2. 複数文字音素をPUAコードポイントにマッピング
3. 各音素を対応するIDに変換（58音素体系）
4. モデルの`phoneme_id_map`でニューラルネットワークへ入力

### 58音素体系の内訳

- **特殊記号**: 7個（_, ^, $, ?, #, [, ]）
- **母音**: 10個（a, i, u, e, o, A, I, U, E, O）
- **長母音**: 5個（a:, i:, u:, e:, o:）※PUAマッピング
- **子音・その他**: 36個

この体系により、日本語の音韻的特徴をより正確に表現し、高品質な音声合成を実現しています。
