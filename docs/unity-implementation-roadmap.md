# Unity Piper TTS 実装ロードマップ

## 概要

このドキュメントは、Piper TTS の Unity プラグイン実装における詳細なタスクリスト、マイルストーン、テスト計画、CI/CD 要件を定義する統合ドキュメントです。計画書（`planning.md`、`architecture.md`、`unity-piper-planning.md`）に基づいて作成されています。

各タスクは最大2人日（16時間）単位に分解されており、明確な成果物と完了条件を持ちます。全7フェーズ、90人日の包括的な実装計画を提供します。

**重要な変更**: Unity Sentis の代わりに、Unity AI Interface (Inference Engine) を使用します。これはSentisの次世代版で、Unity 6.2以降で正式サポートされており、将来的なアップデートに対応しやすくなります。

## リスク管理

### 技術的リスク

1. **Unity バージョン互換性**
   - リスク: Unity 6 の API 変更
   - 対策: 
     - 条件付きコンパイル（#if UNITY_6_0_OR_NEWER）
     - Unity 2022.3 LTS へのフォールバックプラン
     - API 抽象化レイヤーの実装

2. **プラットフォーム固有の問題**
   - リスク: ネイティブライブラリの互換性
   - 対策:
     - 各プラットフォーム専用のビルドスクリプト
     - 動的ライブラリローディングの実装
     - プラットフォーム別のフォールバック機能

3. **パフォーマンス問題**
   - リスク: モバイルでの処理速度
   - 対策:
     - 3段階の品質設定（高/中/低）
     - 音素化結果のキャッシング（LRU）
     - バックグラウンドスレッド処理

4. **メモリ制限（特に WebGL）**
   - リスク: 2GB メモリ制限
   - 対策:
     - モデルの段階的ロード
     - 使用済みリソースの積極的解放
     - メモリプロファイリングツールの活用

### スケジュールリスク

1. **依存関係の遅延**
   - リスク: Inference Engine アップデートの影響
   - 対策:
     - Inference Engine 2.2.x でバージョン固定
     - 重要な機能は独自実装を準備
     - Unity との定期的な技術同期

2. **プラットフォーム審査**
   - リスク: iOS App Store 審査の遅延
   - 対策:
     - Phase 4 完了時点で事前審査申請
     - Privacy Manifest の早期準備
     - TestFlight での継続的テスト

3. **OpenJTalk 統合の複雑性**
   - リスク: 計画書との実装差異
   - 対策:
     - Phase 1 で基本実装を検証
     - 必要に応じて espeak-ng への切り替え
     - 段階的な機能追加アプローチ
