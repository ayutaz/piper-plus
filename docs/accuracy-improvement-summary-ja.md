# Piper-Plus 精度向上調査結果サマリー

## エグゼクティブサマリー

piper-plusの音声合成品質向上について調査した結果、**モデルサイズを維持しながらMOSを最大0.2ポイント向上**できる3つの改善策を特定しました。

## 調査背景

### 現状の課題
- イントネーションとアクセントの不自然さ
- ファインチューニング時の話者再現性の低下
- 追加学習による品質劣化

### 制約条件
- ONNXモデルサイズ：現行の"high"モデル以下
- Unity Sentis 2.1との互換性維持
- 単一ONNXファイルでの動作

## 推奨改善策

### 優先度順の実装提案

| 順位 | 改善項目 | 期待効果 | 実装工数 | リスク |
|------|---------|---------|---------|--------|
| 1 | EMA（指数移動平均） | MOS +0.03-0.06 | 1-2日 | 低 |
| 2 | アクセント記号埋め込み | MOS +0.05-0.08 | 2-3日 | 低 |
| 3 | F0予測器 | MOS +0.10 | 3-5日 | 中 |

## 各改善の詳細

### 1. EMA（指数移動平均）- 最もコストパフォーマンスが高い

**概要**: HiFi-GANのパラメータを安定化

**メリット**:
- 実装が最も簡単
- ファインチューニング時の品質劣化を防ぐ
- 推論時のオーバーヘッドなし

**実装内容**:
```python
# 学習時にEMAを有効化するだけ
python -m piper_train --use-ema --ema-decay 0.9995
```

### 2. アクセント記号埋め込み - 日本語特有の改善

**概要**: 既存のKurihara記法を拡張し、詳細なプロソディ制御を実現

**メリット**:
- 日本語のアクセント表現が大幅に改善
- 既存の前処理パイプラインとの互換性
- モデルサイズへの影響が最小

**新規アクセントマーク**:
- `↑↓`: アクセント核の制御
- `⤴⤵`: 句末イントネーション
- `|‖`: 句境界の明示

### 3. F0予測器 - 最大の品質向上

**概要**: テキストから直接ピッチを予測し、自然なイントネーションを実現

**メリット**:
- 最も大きな品質向上（MOS +0.10）
- プロソディ情報との相乗効果
- 不確実性モデリングによる安定性

**技術詳細**:
- 離散F0ビン（256段階）
- Multi-head attentionによる文脈考慮
- 話者条件付けによるマルチスピーカー対応

### Phase 3（3週間）: F0予測器

## 期待される成果

### 品質向上
- **総合MOS向上**: +0.18〜0.24ポイント
- **自然性**: 特にイントネーションとアクセント
- **安定性**: ファインチューニング耐性の向上

### 技術的利点
- ONNXサイズ: +50MB以下（許容範囲内）
- 推論速度: +10-15%（実用範囲内）
- Unity Sentis互換性: 完全維持

## リスクと対策

### リスク1: F0予測の不安定性
**対策**: 段階的な損失重み調整（0.01→0.1）

### リスク2: 学習時間の増加
**対策**: 並列化とバッチサイズの最適化

### リスク3: 既存モデルとの互換性
**対策**: 後方互換性を保つフラグ管理

## 結論

3つの改善策は**独立して実装可能**であり、段階的な品質向上が期待できます。特にEMAは即座に実装可能で、投資対効果が最も高い改善です。

全ての改善を実装した場合、piper-plusは商用レベルの日本語TTSシステムに匹敵する品質を、**軽量なモデルサイズを維持しながら**実現できると考えられます。
