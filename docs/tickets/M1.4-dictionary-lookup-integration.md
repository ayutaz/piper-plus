# M1.4 辞書ルックアップ統合 -- SwedishPhonemizer パイプライン完成

## マイルストーン紐づけ
- **マイルストーン**: M1.4
- **Phase**: Phase 1
- **対応要件**: FR-01, FR-05
- **依存先チケット**: M1.1 (NST辞書データ準備 + SAMPA→IPA変換), M1.3 (Rule-based G2Pエンジン)
- **ブロックするチケット**: M1.5 (マルチリンガル統合)

## 達成したい内容

NST辞書ルックアップ (FR-01) と Phonemizer ABC 準拠 (FR-05) を統合し、`SwedishPhonemizer` クラスを完成させる。具体的には以下を実現する:

1. **辞書ロード機構**: JSON/JSON.gz 形式のNST辞書 (Core ~238K語) をコンストラクタ引数または環境変数 `PIPER_SV_DICT_PATH` で指定して読み込む
2. **ルックアップ → フォールバック パイプライン**: Stage 1 辞書検索 → Stage 2-6 Rule-based G2P のフォールバックチェーン
3. **Phonemizer ABC 完全準拠**: `phonemize()`, `phonemize_with_prosody()`, `get_phoneme_id_map()` の3メソッド実装
4. **ProsodyInfo 構築**: a1=0 (未使用/将来の声調アクセント用), a2=ストレスレベル (0/1/2), a3=単語音素数
5. **辞書なしモード**: 辞書ファイルなしでもRule-based G2Pのみで動作する (CI/テスト環境向け)

辞書ヒット語は100%正確なIPA出力、辞書ミス語はRule-based G2Pにフォールバックする2段構えのパイプラインを構築する。

## 懸念点と影響範囲

| # | 懸念点 | 影響度 | 緩和策 |
|---|--------|--------|--------|
| 1 | 辞書ロードパフォーマンス | 中 | Core辞書 (~238K語) のロード時間が3秒以下であることを要求。JSON.gz形式 (~2.3MB) を使用してI/O負荷を軽減。ロード時間の計測テストを追加して監視する |
| 2 | メモリ使用量 | 中 | Python dict のオーバーヘッドにより Core辞書で ~30-40MB、Full辞書で ~100-120MB を消費する。Core辞書をデフォルトとし、NFR としてメモリ使用量 50MB以下を目標とする。Full辞書使用時はドキュメントで注意喚起する |
| 3 | 辞書なし時の品質劣化 | 低 | 辞書なしモードではRule-based G2Pのみで動作するため、精度が低下する (辞書ヒット語が全てOOV扱い)。CI/テスト環境では許容範囲。本番利用時は辞書指定を推奨するドキュメントを整備する |
| 4 | 複合語分割の精度 | 低 | 右→左走査 + 結合形素3パターンのヒューリスティックであり、全ての複合語を正しく分割できない場合がある。Phase 4 (M4.2) で形態論的分割の改善を予定しているため、Phase 1 では80%程度の精度を許容する |
| 5 | 環境変数の優先順位衝突 | 低 | `dict_path` 引数と `PIPER_SV_DICT_PATH` の優先順位を明確にドキュメント化する。引数 > 環境変数 > None の順で解決する |

## レビューする内容

1. **ABC 準拠の一貫性**: `SwedishPhonemizer` のメソッドシグネチャが `SpanishPhonemizer`, `FrenchPhonemizer`, `PortuguesePhonemizer` と同一パターンであることを確認
2. **ProsodyInfo マッピングの正確性**: a1=0 (固定), a2=ストレスレベル (0/1/2), a3=単語音素数 が EN/ES と一貫していることを確認
3. **辞書ロードのエラーハンドリング**: FileNotFoundError, json.JSONDecodeError, TypeError の各例外が適切にハンドリングされることを確認
4. **正規化アルゴリズムの網羅性**: NFC正規化、小文字変換、句読点除去 (ハイフン保持) が FR-01 の仕様 (Section 1.4) と一致することを確認
5. **複合語分割の正確性**: 結合形素 (s/e/o) の検出、右→左走査、副ストレス降格が FR-01 の仕様 (Section 1.5.3) と一致することを確認
6. **辞書なしモードの動作**: 辞書パス未指定時に `self._dict = {}` のまま Rule-based G2P のみで正常動作することを確認
7. **テストカバレッジ**: 辞書ヒット/ミス/空入力/gzip/環境変数/複合語分割/ProsodyInfo の全パスが網羅されていることを確認
