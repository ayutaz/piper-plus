## 1. タスク目的とゴール

CMake プロジェクトから `find_package(PiperPlus)` + `target_link_libraries(app PiperPlus::piper_plus)` で簡単にリンクできるようにする。CMake Config パッケージは pkg-config (M3-2) と異なり、ターゲットベースの依存伝播 (include パス、リンクフラグの自動伝播) を提供し、Windows を含む全プラットフォームで一貫して動作する。

**ゴール:** install 先の `lib/cmake/PiperPlus/` に CMake Config ファイル群が生成され、以下のパターンが動作すること。

```cmake
# 消費側の CMakeLists.txt
find_package(PiperPlus REQUIRED)
add_executable(my_app main.c)
target_link_libraries(my_app PRIVATE PiperPlus::piper_plus)
# -> include パスとリンクフラグが自動伝播される
```

## 2. 実装する内容の詳細

### 2.1 変更対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `cmake/PiperPlusConfig.cmake.in` (新規) | CMake Config テンプレート |
| `cmake/PiperPlusConfigVersion.cmake.in` (新規) | バージョン互換性チェック |
| `CMakeLists.txt` | `install(EXPORT)` + Config ファイル生成・インストール |

### 2.2 Config テンプレートファイル

#### `cmake/PiperPlusConfig.cmake.in`

```cmake
# PiperPlusConfig.cmake -- CMake Config file for piper-plus C API shared library
# Generated by CMake. Do not edit manually.
#
# Usage:
#   find_package(PiperPlus REQUIRED)
#   target_link_libraries(my_app PRIVATE PiperPlus::piper_plus)
#
# Provides:
#   PiperPlus::piper_plus  -- IMPORTED shared library target
#
# Variables:
#   PiperPlus_FOUND        -- TRUE if found
#   PiperPlus_VERSION      -- Library version (e.g., "1.10.0")
#   PiperPlus_DICT_DIR     -- Path to OpenJTalk dictionary
#   PiperPlus_G2P_DICT_DIR -- Path to G2P dictionaries (CMU, pinyin)

@PACKAGE_INIT@

include(CMakeFindDependencyMacro)

# ONNX Runtime は動的リンクで同梱されているため find_dependency は不要。
# 将来 ORT が CMake Config を提供した場合は以下を有効化:
# find_dependency(onnxruntime)

# Import targets
include("${CMAKE_CURRENT_LIST_DIR}/PiperPlusTargets.cmake")

# Convenience variables
set(PiperPlus_VERSION "@piper_version@")
set(PiperPlus_DICT_DIR "${PACKAGE_PREFIX_DIR}/share/open_jtalk/dic")
set(PiperPlus_G2P_DICT_DIR "${PACKAGE_PREFIX_DIR}/share/piper/dicts")

check_required_components(PiperPlus)
```

**設計ポイント:**
- `@PACKAGE_INIT@` は `CMakePackageConfigHelpers` の `configure_package_config_file()` が展開するマクロで、relocatable なパス解決を提供する
- `PiperPlus_DICT_DIR` / `PiperPlus_G2P_DICT_DIR` を変数として公開し、消費側が辞書パスを取得可能にする
- ONNX Runtime の `find_dependency` は不要 (同梱 DLL/dylib/so が RPATH で解決される)

#### `cmake/PiperPlusConfigVersion.cmake.in` は使わない

CMake 3.15+ の `write_basic_package_version_file()` を使用するため、テンプレートファイルは不要。

### 2.3 CMakeLists.txt への変更

M3-1 の `if(PIPER_PLUS_BUILD_SHARED)` ブロック内に追加:

```cmake
if(PIPER_PLUS_BUILD_SHARED)
  include(GNUInstallDirs)
  include(CMakePackageConfigHelpers)

  # --- M3-1 で追加済みの EXPORT 付き install ---
  # install(TARGETS piper_plus EXPORT PiperPlusTargets ...)

  # --- Export targets ファイル ---
  install(EXPORT PiperPlusTargets
    FILE PiperPlusTargets.cmake
    NAMESPACE PiperPlus::
    DESTINATION ${CMAKE_INSTALL_LIBDIR}/cmake/PiperPlus
  )

  # --- Config ファイル ---
  configure_package_config_file(
    ${CMAKE_CURRENT_SOURCE_DIR}/cmake/PiperPlusConfig.cmake.in
    ${CMAKE_CURRENT_BINARY_DIR}/PiperPlusConfig.cmake
    INSTALL_DESTINATION ${CMAKE_INSTALL_LIBDIR}/cmake/PiperPlus
  )

  # --- Version ファイル ---
  write_basic_package_version_file(
    ${CMAKE_CURRENT_BINARY_DIR}/PiperPlusConfigVersion.cmake
    VERSION ${piper_version}
    COMPATIBILITY SameMajorVersion
  )

  install(FILES
    ${CMAKE_CURRENT_BINARY_DIR}/PiperPlusConfig.cmake
    ${CMAKE_CURRENT_BINARY_DIR}/PiperPlusConfigVersion.cmake
    DESTINATION ${CMAKE_INSTALL_LIBDIR}/cmake/PiperPlus
  )
endif()
```

**`COMPATIBILITY SameMajorVersion`:** SOVERSION が `1` なので、`find_package(PiperPlus 1.10)` は `1.10.0` 以上 `2.0.0` 未満にマッチする。バージョン `1.11.0` は互換、`2.0.0` は非互換。

### 2.4 Install 後のファイル構成

```
<prefix>/lib/cmake/PiperPlus/
├── PiperPlusConfig.cmake         -- find_package() のエントリポイント
├── PiperPlusConfigVersion.cmake  -- バージョン互換性チェック
├── PiperPlusTargets.cmake        -- IMPORTED ターゲット定義
└── PiperPlusTargets-release.cmake -- Release 構成のパス情報
```

### 2.5 消費側の使い方

```cmake
# CMakeLists.txt (消費側プロジェクト)
cmake_minimum_required(VERSION 3.15)
project(my_tts_app C)

# find_package は CMAKE_PREFIX_PATH で install 先を指定
find_package(PiperPlus REQUIRED)

add_executable(my_app main.c)
target_link_libraries(my_app PRIVATE PiperPlus::piper_plus)

# 辞書パスの利用
message(STATUS "OpenJTalk dict: ${PiperPlus_DICT_DIR}")
message(STATUS "G2P dicts: ${PiperPlus_G2P_DICT_DIR}")
```

```bash
cmake -B build -DCMAKE_PREFIX_PATH=/path/to/piper-plus-install
cmake --build build
```

### 4.1 ユニットテスト

### 4.2 E2E テスト

消費側 CMake プロジェクトを `examples/c-api/cmake_example/` (M3-6 で作成) として準備し、CI から以下を実行:

```bash
# install
cmake --install build --prefix /tmp/piper-plus-install

# 消費側プロジェクトをビルド
cmake -B /tmp/test-cmake-config \
  -S examples/c-api/cmake_example \
  -DCMAKE_PREFIX_PATH=/tmp/piper-plus-install
cmake --build /tmp/test-cmake-config

# 実行
LD_LIBRARY_PATH=/tmp/piper-plus-install/lib \
  /tmp/test-cmake-config/test_cmake_config
```

消費側のテストプログラム (`examples/c-api/cmake_example/main.c`):

```c
#include <piper_plus.h>
#include <stdio.h>

int main(void) {
    printf("PiperPlus version: %s\n", piper_plus_version());
    printf("API version: %d\n", piper_plus_api_version());
    return 0;
}
```

## 5. 懸念事項とレビュー項目

| 懸念 | 詳細 | 対策 |
|------|------|------|
| ONNX Runtime の IMPORTED ターゲットが EXPORT に含まれない | ORT は `find_package` で見つからないため、`target_link_libraries(piper_plus PRIVATE onnxruntime)` の PRIVATE 依存が EXPORT で問題を起こす可能性 | PRIVATE 依存はエクスポートされない。ORT は RPATH / DLL 同梱で実行時に解決される |
| Windows での `CMAKE_PREFIX_PATH` | バックスラッシュ問題 | CMake はスラッシュを正しく処理する。テストで Windows パスを確認 |
| relocatable パッケージ | install 先を移動した場合に `PiperPlusConfig.cmake` 内のパスが壊れる | `configure_package_config_file` + `@PACKAGE_INIT@` が相対パスを使って解決する |
| 複数バージョンの共存 | `find_package(PiperPlus 1.10)` と `find_package(PiperPlus 1.11)` | `SameMajorVersion` ポリシーで SOVERSION 1 系は全て互換 |

**レビュー項目:**
- [ ] `find_package(PiperPlus)` が 3 プラットフォームで成功する
- [ ] `PiperPlus::piper_plus` の INTERFACE_INCLUDE_DIRECTORIES が正しい
- [ ] `PiperPlus::piper_plus` の IMPORTED_LOCATION が正しいライブラリパスを指す
- [ ] `PiperPlusConfigVersion.cmake` のバージョン比較が正しく動作する
- [ ] M3-2 の pkg-config と同じ include/lib パスを返す

## 6. 一から作り直すとしたら

1. **ONNX Runtime を IMPORTED ターゲットとして Config に含める。** 現在の設計では ORT は RPATH/DLL 同梱で暗黙的に解決されるが、消費側が別バージョンの ORT を使いたい場合に上書きできない。`PiperPlusConfig.cmake` で `find_dependency(onnxruntime)` を使い、見つからなければ同梱版にフォールバックする方式が理想的。

2. **`PiperPlus_INCLUDE_DIR` / `PiperPlus_LIBRARY` の旧式変数も提供する。** `find_package` のモジュールモード (`FindPiperPlus.cmake`) を使うプロジェクトとの互換性のために、旧式変数を設定する Config ファイルもある。しかし、ターゲットベースの API が主流になっているため、現時点では不要と判断。

3. **vcpkg / Conan のポートファイル生成を Phase 3 に含める。** Config パッケージが正しく機能すれば、vcpkg の `portfile.cmake` は `vcpkg_cmake_install()` + `vcpkg_cmake_config_fixup()` の10行程度で済む。Phase 3 に含める価値はあるが、利用者のフィードバック待ちで Phase 4 に送る判断も妥当。
