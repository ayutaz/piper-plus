## 1. 目的とゴール

### 目的

piper-g2p パッケージに韓国語の G2P 機能を追加する。g2pk2 を用いて音韻規則 (연음화/비음화/격음화/경음화) を適用した後、ハングル音節を初声・中声・終声に分解して IPA トークン列に変換する。g2pk2 が利用不可の場合はフォールバックとしてハングル直接分解を行う。

### ゴール

- `get_phonemizer("ko").phonemize("안녕하세요")` が IPA トークン列を返す
- g2pk2 による音韻規則が適用される (インストール時)
- g2pk2 未インストール時はフォールバックで動作する (品質低下)
- NFC 正規化によりNFD分解済みハングルに対応する

## 2. 実装詳細

### 作成/変更するファイル

| ファイル | 内容 |
|---------|------|
| `src/python/g2p/piper_g2p/korean.py` | KoreanPhonemizer 実装 (IPA-first) |
| `src/python/g2p/piper_g2p/registry.py` | `_auto_register()` に ko を追加 |
| `src/python/g2p/tests/test_korean.py` | 単体テスト |

### 実装手順

1. 現在の `src/python/piper_train/phonemize/korean.py` をベースにコピー
2. 以下の変更を適用:
   - `map_sequence()` (PUA 変換) の呼び出しを削除
   - `token_mapper` への依存を除去
   - `get_phoneme_id_map()` メソッドを削除
   - インポートパスを `piper_g2p.base` に変更
3. g2pk2 フォールバック機構をそのまま移植:
   - グローバル `_g2p_instance` / `_g2p_unavailable` によるキャッシュ
   - `ImportError` / `SyntaxError` / `AttributeError` のキャッチ
   - mecab バックエンド未検出時のフォールバック
4. ハングル分解ロジック (初声19/中声21/終声28) を移植
5. `registry.py` の `_auto_register()` に ko ブロックを追加 (g2pk2 は optional)
6. テストケースを作成

### API / インターフェース

```python
from piper_g2p import get_phonemizer

phonemizer = get_phonemizer("ko")

# 基本音素化
tokens = phonemizer.phonemize("안녕하세요")
# -> ["ʔ", "a", "n", "n", "j", "ʌ", "ŋ", " ", "h", "a", "s", "e", "j", "o"]
# (g2pk2 による 연음화 適用後)

# prosody 付き
tokens, prosody = phonemizer.phonemize_with_prosody("감사합니다")
# prosody[i].a1 = 0 (韓国語はピッチアクセントなし)
# prosody[i].a2 = 0 (韓国語は語彙ストレスなし)
# prosody[i].a3 = word内のハングル音節数
```

## 4. テスト計画

### 提供範囲

KoreanPhonemizer の音素化出力が正しい IPA であること、g2pk2 の有無でフォールバックが正しく動作すること。

### Unit テスト

| テストケース | 入力 | 期待出力 (概要) |
|-------------|------|---------------|
| 基本ハングル | `"안녕"` | 初声+中声+終声の IPA 分解が正しい |
| 연음화 (liaison) | `"음악을"` | 連音規則が適用される |
| 비음화 (nasalization) | `"합니다"` | 鼻音化が適用される |
| 格音化 (aspiration) | `"축하"` | 激音化が適用される |
| 경음化 (tensification) | `"학교"` | 濃音化が適用される |
| 句読点 | `"안녕하세요!"` | `!` が含まれる |
| NFC 正規化 | NFD ハングル入力 | NFC 変換後に正しく処理 |
| g2pk2 フォールバック | (g2pk2 未インストール) | フォールバック動作 |
| PUA なし | 任意 | PUA 文字が含まれない |
| BOS/EOS なし | 任意 | `"^"` `"$"` が含まれない |

### E2E テスト

- `get_phonemizer("ko")` でインスタンスが取得できること
- g2pk2 未インストール環境でもフォールバック動作すること

## 5. 懸念事項とレビュー項目

### 懸念事項

- **g2pk2 フォールバック品質**: g2pk2 未インストール時はハングルの直接分解のみとなり、連音化・鼻音化等の音韻規則が適用されない。出力品質が大幅に低下する。この制限を明確に文書化する必要がある。
- **連音規則カバレッジ**: g2pk2 自体のカバレッジが限定的で、口語的な音韻変化 (e.g., 의→에 の弱化) は部分的にしか対応していない。
- **mecab 依存**: g2pk2 は内部で python-mecab-ko を使用する。mecab がインストールされていても辞書がない場合に `AttributeError` が発生するケースがあり、これをキャッチする必要がある。
- **グローバル状態**: `_g2p_instance` / `_g2p_unavailable` のグローバル変数はスレッドセーフではない。ドキュメントに注意事項として記載する。

### レビュー項目

- [ ] IPA トークンに PUA 文字が混入していないこと
- [ ] g2pk2 未インストール時のフォールバックが正しく動作すること
- [ ] NFC 正規化が最初に適用されること
- [ ] 各音韻規則 (連音化/鼻音化/激音化/濃音化) のテストが存在すること
- [ ] グローバル状態のスレッド安全性の制限が文書化されていること

## 6. 一から作り直すとしたら

g2pk2 への依存を排除し、ルールベースの韓国語音韻規則エンジンを自前実装する選択肢がある。韓国語の音韻規則は体系的で有限 (約 20 規則) のため、ルールベース実装は現実的。ただし正確な実装には韓国語音韻論の専門知識が必要で、g2pk2 の tensorflow 依存問題 (重い) と合わせて Phase 2 以降の検討事項とする。
