## 1. 目的とゴール

`test_phonemizer_registry.py` の L79-88 にある 2 つのテスト (`test_get_phoneme_id_map_monolingual_returns_none`, `test_get_phoneme_id_map_bilingual_returns_map`) が、`Phonemizer` インスタンスに存在しない `get_phoneme_id_map()` メソッドを呼び出しており `AttributeError` で失敗する。

**原因**: テスト作成時の想定では `get_phoneme_id_map()` が `Phonemizer` ABC のインスタンスメソッドだったが、実際にはモジュールレベル関数 `piper_plus_g2p.encode.id_maps.get_phoneme_id_map(language: str)` として実装されている。Phonemizer ABC (`base.py`) にはこのメソッドは存在しない。

**完了の定義**:
- 2 つのテストが `AttributeError` なく正常に PASS する
- テストの意図 (単言語ではマップ取得不可、複合言語ではマップが返る) が正しく検証される
- 既存の他テストに影響しない

## 2. 実装の詳細

### 変更ファイル

`src/python/tests/test_phonemizer_registry.py` (L79-88 のみ)

### 現在のコード (壊れている)

```python
# L79-88
def test_get_phoneme_id_map_monolingual_returns_none(self):
    for lang in ("ja", "en"):
        p = get_phonemizer(lang)
        assert p.get_phoneme_id_map() is None

def test_get_phoneme_id_map_bilingual_returns_map(self):
    p = get_phonemizer("ja-en")
    id_map = p.get_phoneme_id_map()
    assert id_map is not None
    assert len(id_map) > 0
```

**問題点**:
1. `p.get_phoneme_id_map()` -- `Phonemizer` ABC にこのメソッドは存在しない
2. 単言語で `None` を期待しているが、実際の `get_phoneme_id_map("ja")` は有効なマップを返す (`id_maps.py` L367-368)。一方 `get_phoneme_id_map("en")` は `ValueError` を送出する (`id_maps.py` L380-384)。

### 実際の API (`piper_plus_g2p.encode.id_maps`)

```python
@lru_cache(maxsize=8)
def get_phoneme_id_map(language: str) -> dict[str, list[int]]:
    """Return the built-in phoneme_id_map for *language*."""
    if language == "ja":
        return _build_japanese_id_map()           # 有効なマップを返す
    if language in ("ko", "sv"):
        return _build_multilingual_id_map()       # 有効なマップを返す
    if "-" in language or language == "multilingual":
        return _build_multilingual_id_map()       # 有効なマップを返す
    raise ValueError(...)                          # en, zh, es 等は ValueError
```

### 修正後のコード

テストの意図を「モジュールレベル関数 `get_phoneme_id_map()` の動作確認」に変更する。

```python
from piper_plus_g2p.encode import get_phoneme_id_map

# --- 新規クラス TestPhonemeIdMap に配置 (TestABCInterface L79-88 を削除して置換) ---

def test_get_phoneme_id_map_ja_returns_map(self):
    """Japanese has a built-in phoneme_id_map."""
    id_map = get_phoneme_id_map("ja")
    assert isinstance(id_map, dict)
    assert len(id_map) > 0

def test_get_phoneme_id_map_en_raises(self):
    """English alone has no built-in map (use multilingual or config.json)."""
    with pytest.raises(ValueError, match="No built-in phoneme_id_map"):
        get_phoneme_id_map("en")

def test_get_phoneme_id_map_composite_returns_map(self):
    """Composite language codes return the multilingual map."""
    id_map = get_phoneme_id_map("ja-en")
    assert isinstance(id_map, dict)
    assert len(id_map) > 0
```

## 5. 懸念事項とレビュー項目

1. **`lru_cache` の影響**: `get_phoneme_id_map` は `@lru_cache(maxsize=8)` が付いている。テスト間でキャッシュが残るが、同一入力に対しては同一結果を返すため問題ない。ただし `ValueError` テストの後に同じ言語コードで正常呼び出しするとキャッシュが効かない (例外はキャッシュされない) ので注意。

2. **テスト意図の変更**: 元テストは「単言語 Phonemizer には `get_phoneme_id_map` が `None` を返す」という設計を前提にしていた。しかし実際のアーキテクチャでは `get_phoneme_id_map()` は Phonemizer のメソッドではなくエンコーダ層の関数である。修正後のテストはこのアーキテクチャの実態を反映する。

3. **クラス配置**: 修正後のテストは `TestABCInterface` クラス内にあるが、`get_phoneme_id_map()` は ABC のメソッドではない。別クラス `TestPhonemeIdMap` に分離するか、既存クラスに残すか検討の余地がある。分離推奨。

4. **互換性**: `Phonemizer` ABC への `get_phoneme_id_map()` メソッド追加は行わない。これはテスト側の誤りであり、ABCの責務 (音素化) とエンコーダの責務 (ID変換) を混在させるべきではない。

## 6. 一から作り直すとしたら

もしこのテストを今から設計するなら:

1. **テストクラスの分離**: `TestABCInterface` (Phonemizer ABC の公開メソッドのみ) と `TestPhonemeIdMap` (エンコーダ層の `get_phoneme_id_map()`) を明確に分ける。ABC テストに encode 層のテストを混ぜない。

2. **パラメトライズ**: `@pytest.mark.parametrize` で言語コードと期待結果をテーブル駆動にする。

```python
class TestPhonemeIdMap:
    @pytest.mark.parametrize("lang", ["ja", "ko", "sv"])
    def test_builtin_single_lang(self, lang):
        id_map = get_phoneme_id_map(lang)
        assert len(id_map) > 0

    @pytest.mark.parametrize("lang", ["en", "zh", "es", "fr", "pt"])
    def test_no_builtin_single_lang(self, lang):
        with pytest.raises(ValueError):
            get_phoneme_id_map(lang)

    @pytest.mark.parametrize("lang", ["ja-en", "ja-en-zh-es-fr-pt"])
    def test_composite_lang(self, lang):
        id_map = get_phoneme_id_map(lang)
        assert len(id_map) > 0
```

3. **ABC インタフェーステスト**: Phonemizer ABC のテストは `phonemize()` と `phonemize_with_prosody()` の戻り値型・長さ契約のみを検証し、encode 層との結合テストは別ファイルに配置する。
