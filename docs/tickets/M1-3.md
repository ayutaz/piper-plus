## 1. 目的とゴール

`__main__.py` の L430-443 に散在する 2 つの自動設定ロジック:

1. **gin_channels 自動設定**: `(num_speakers > 1 or num_languages > 1)` かつ未指定 (`== 0`) の場合に `512` を設定
2. **freeze_dp 自動有効化**: `--resume-from-multispeaker-checkpoint` 指定時に `freeze_dp=True` を自動設定

これらを `apply_transfer_defaults()` 関数として抽出し、テスト側ではこの関数を直接呼び出すことで、本番コードの再実装 (コピペ) を排除する。

**現状の問題**:

- `test_multispeaker_transfer.py` (6 テスト) と `test_model_config.py` (3 テスト) が本番ロジックをテスト内にコピペして再実装している
- `test_model_config.py` の 3 テストは**古い条件** (`num_speakers > 1` のみ) を使用しており、本番コード (`num_speakers > 1 or num_languages > 1`) とのドリフトが発生済み
- ロジック変更時に `__main__.py` + 2 つのテストファイルの 3 箇所を同時修正する必要がある

**完了の定義**:

- `apply_transfer_defaults(args, dict_args, num_speakers, num_languages)` が `__main__.py` (または新規モジュール) に存在する
- `__main__.py` の L430-443 が `apply_transfer_defaults()` 呼び出しに置換されている
- `test_multispeaker_transfer.py` の 6 テストが `apply_transfer_defaults()` を呼び出す形に書き換えられている
- `test_model_config.py` の 3 つの gin_channels テストが `apply_transfer_defaults()` を呼び出す形に書き換えられている
- 全テストが PASS する
- テスト内にコピペされた本番ロジックが 0 箇所になる

### 2.2 抽出後の関数

```python
def apply_transfer_defaults(
    args: argparse.Namespace,
    dict_args: dict,
    num_speakers: int,
    num_languages: int,
) -> None:
    """Model creation 前に呼ばれる転移学習のデフォルト値自動設定.

    1. gin_channels: マルチスピーカー or 多言語モデルで未指定の場合 512 を設定
    2. freeze_dp: --resume-from-multispeaker-checkpoint 時に自動有効化

    Args に直接変更を加える (副作用あり)。dict_args は vars(args) を前提とする。
    """
    # gin_channels 自動設定
    # 768 は ONNX エクスポート時の数値精度低下を引き起こす
    # VitsModel.__init__ のフォールバック (512) と一致させる
    if (num_speakers > 1 or num_languages > 1) and dict_args.get(
        "gin_channels", 0
    ) == 0:
        dict_args["gin_channels"] = 512

    # freeze_dp 自動有効化
    # モデル作成前に設定しないと save_hyperparameters() に反映されない
    if getattr(args, "resume_from_multispeaker_checkpoint", None) and not args.freeze_dp:
        args.freeze_dp = True
        dict_args["freeze_dp"] = True
        _LOGGER.info(
            "Auto-enabled --freeze-dp for multispeaker→single-speaker transfer"
        )
```

## 5. 懸念事項とレビュー項目

1. **副作用の可視性**: `apply_transfer_defaults()` は `args` と `dict_args` を直接変更する (副作用あり)。`dict_args = vars(args)` の前提により、`args.freeze_dp = True` が `dict_args["freeze_dp"] = True` を暗黙的に設定する。関数のドキュメントでこの副作用を明記すること。

2. **`vars(args)` の同一性**: `dict_args = vars(args)` は `args.__dict__` と同一オブジェクトを返す。テスト側で `dict_args = vars(args)` を使う限り、`args.freeze_dp = True` を設定すれば `dict_args["freeze_dp"]` も `True` になる。ただし `dict_args = {"freeze_dp": False}` のように独立した dict を作ると同一性が崩れる。テストでは必ず `vars(args)` を使うこと。

3. **import の循環依存**: `test_multispeaker_transfer.py` が `from piper_train.__main__ import apply_transfer_defaults` する。`__main__.py` のモジュールレベルで PyTorch 等の重い import がある場合、テスト時にも実行される。ただし既に `test_model_config.py` が `from piper_train.__main__ import create_parser` を行っているため、新たな問題にはならない。

4. **関数の配置場所**: `apply_transfer_defaults()` を `__main__.py` に置くか、別モジュール (例: `transfer_utils.py`) に分離するか。本チケットでは最小変更として `__main__.py` 内に配置する。将来 `__main__.py` が肥大化した場合に分離を検討する。

5. **`_LOGGER.info` のテスト**: `apply_transfer_defaults()` 内の `_LOGGER.info(...)` 呼び出しをテストするかどうか。本チケットでは機能の正しさ (args/dict_args の値) のみを検証し、ログ出力のテストは行わない。

6. **`getattr` vs 直接アクセス**: 現在の `test_multispeaker_transfer.py` は `getattr(args, "resume_from_multispeaker_checkpoint", None)` を使うが、本番コードは `args.resume_from_multispeaker_checkpoint` を直接参照している。抽出する関数では `getattr` を使い、属性が存在しない `Namespace` でも安全に動作するようにする。

## 6. 一から作り直すとしたら

もしこのテスト + 本番コードを今から設計するなら:

1. **純粋関数への変換**: `apply_transfer_defaults()` を副作用なしの純粋関数にする。`args` を直接変更するのではなく、変更すべきキーと値の dict を返す:

```python
def compute_transfer_defaults(
    gin_channels: int,
    freeze_dp: bool,
    resume_from_multispeaker: str | None,
    num_speakers: int,
    num_languages: int,
) -> dict[str, Any]:
    """Return a dict of keys to override in dict_args."""
    overrides = {}
    if (num_speakers > 1 or num_languages > 1) and gin_channels == 0:
        overrides["gin_channels"] = 512
    if resume_from_multispeaker and not freeze_dp:
        overrides["freeze_dp"] = True
    return overrides
```

テスト側は戻り値を検証するだけでよく、`argparse.Namespace` のセットアップが不要になる。

2. **パラメトライズ**: `@pytest.mark.parametrize` でテーブル駆動テストにする:

```python
@pytest.mark.parametrize(
    "num_speakers, num_languages, gin_channels_in, expected",
    [
        (80, 1, 0, 512),    # multispeaker
        (1, 6, 0, 512),     # multilingual
        (80, 6, 0, 512),    # both
        (1, 1, 0, 0),       # single speaker + single language
        (80, 1, 256, 256),  # explicit value preserved
    ],
)
def test_gin_channels(num_speakers, num_languages, gin_channels_in, expected):
    result = compute_transfer_defaults(
        gin_channels=gin_channels_in,
        freeze_dp=False,
        resume_from_multispeaker=None,
        num_speakers=num_speakers,
        num_languages=num_languages,
    )
    assert result.get("gin_channels", gin_channels_in) == expected
```

3. **テストファイルの整理**: gin_channels テストが `test_model_config.py` と `test_multispeaker_transfer.py` の両方に存在する。一箇所に集約するか、テストの責務を明確に分ける (model config = CLI defaults、transfer = 転移学習ロジック)。

4. **テスト名の規約**: `test_<関数名>_<条件>_<期待>` 形式に統一 (例: `test_apply_transfer_defaults_multispeaker_sets_gin_512`)。

## 7. 後続タスクへの連絡事項

- **M0 との関連**: M0-1, M0-2 は `test_phonemizer_registry.py` の修正であり、本チケットとは別ファイル。コンフリクトなし。
- **import パス**: `from piper_train.__main__ import apply_transfer_defaults` を使用する。`__main__.py` のトップレベル import が重い場合、将来的に別モジュールへの分離を検討する。
- **テスト重複**: `test_model_config.py` と `test_multispeaker_transfer.py` の gin_channels テストが意味的に重複している。本チケットでは両方を `apply_transfer_defaults()` 呼び出しに書き換えるが、テスト自体の統合 (片方への集約) は行わない。M2 以降で検討すること。
- **`dict_args = vars(args)` の前提**: テスト側で `apply_transfer_defaults()` を呼ぶ際は、必ず `dict_args = vars(args)` で dict を作ること。独立した dict を渡すと `args` との同一性が崩れ、`freeze_dp` の二重設定ロジックが正しく動作しない。
