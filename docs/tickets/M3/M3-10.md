# [M3-10] あみたろモデル学習 + 公開 (JA, 商用OK)

**マイルストーン**: M3b (v2.1.0)
**カテゴリ**: モデルZoo
**工数**: 3d
**依存**: LJSpeech 形式インポート機能の追加 (`tools/prepare_ljspeech_finetune.py` の作成, 0.5d)
**ステータス**: 未着手

## 1. 目的とゴール

あみたろの声素材工房 (商用利用可能な独自ライセンス) の音声データを使用して、日本語の2番目のシングルスピーカーモデルを学習し公開する。つくよみちゃんとは異なる声質を提供することで、日本語モデルの選択肢を広げる。4スタイル (ノーマル/喜び/悲しみ/怒り) のデータが利用可能。Issue #305 対応。

**ゴール:**
- 6言語ベースモデルからの転移学習であみたろモデルを作成
- 4スタイルのうち最低1スタイル (ノーマル) でモデル公開
- FP16 ONNX エクスポート + config.json 生成
- HuggingFace リポジトリ `ayousanz/piper-plus-amitaro` に公開
- ライセンス条件を Model Card に正確に記載

### 2.2 ライセンス確認 (ブロッカー / 前提条件)

あみたろの声素材工房のライセンス条件を事前に確認する必要がある。**本タスクの全工程はライセンス確認の完了を前提とする。**

1. **配布形態**: ONNX モデルとしての二次配布が許可されるか
2. **クレジット表記**: Model Card / README にどのような表記が必要か
3. **商用利用**: 生成音声の商用利用条件
4. **改変**: 学習データとしての利用が「改変」に該当するか

> **ブロッカー**: ライセンス確認が完了するまで、データセット準備・学習を含む全工程に着手しないこと。
> **フォールバック**: ライセンス確認の結果、ONNX モデル配布が不許可の場合は、本チケットをキャンセルし Issue #305 にその旨を記録する。

### 2.3 データセット準備

```bash
# あみたろデータ -> Piper データセット形式変換
# 注: prepare_multilingual_dataset.py には --ljspeech-dir オプションは存在しない。
# LJSpeech 形式 (metadata.csv: filename|text) からの変換には
# 専用スクリプト tools/prepare_ljspeech_finetune.py を使用する。
uv run python -m piper_train.tools.prepare_ljspeech_finetune \
  --ljspeech-dir /data/piper/amitaro-normal/ \
  --output-dir /data/piper/dataset-amitaro-finetune \
  --base-config /data/piper/dataset-multilingual-6lang-filtered/config.json
```

### 2.4 学習

```bash
uv run python -m piper_train \
  --dataset-dir /data/piper/dataset-amitaro-finetune \
  --prosody-dim 16 \
  --accelerator gpu --devices 1 --precision 32-true \
  --max_epochs 500 --batch-size 4 --samples-per-speaker 4 \
  --checkpoint-epochs 50 --quality medium \
  --base_lr 2e-5 --disable_auto_lr_scaling \
  --ema-decay 0.9995 --max-phoneme-ids 400 --no-wavlm \
  --val-every-n-epochs 50 --audio-log-epochs 50 \
  --resume-from-multispeaker-checkpoint \
    /data/piper/output-multilingual-6lang/lightning_logs/version_0/checkpoints/epoch=74-step=504712.ckpt \
  --default_root_dir /data/piper/output-amitaro
```

### 2.5 少量データへの対応

424発話はつくよみちゃん (100発話) より多いが、依然として少量。学習パラメータの調整が必要な場合:

| パラメータ | 標準値 | 調整候補 | 理由 |
|-----------|--------|---------|------|
| `--max_epochs` | 500 | 300-500 | 424発話なら 300 epoch でも十分な gradient steps |
| `--base_lr` | 2e-5 | 2e-5 | つくよみちゃんと同等で問題ないはず |
| `--batch-size` | 4 | 4 | 424 / 4 = 106 batches/epoch |

### 2.6 品質検証

つくよみちゃんモデルとの比較を含む:
- 同一テキストで両モデルの音声を生成し、声質の差異を確認
- 6言語推論テスト (多言語能力保持の確認)
- 長文 (100文字超) での安定性テスト

## 6. 一から作り直すとしたら

日本語の多様な声質をカバーするなら、あみたろの4スタイルを活用したマルチスタイルモデル (style_id で切り替え) を最初から設計すべきだった。現在の VITS アーキテクチャは speaker_id による話者切り替えをサポートしているため、4スタイルを4話者として学習する方法も技術的には可能。ただし、モデルZoo戦略としてはまず「1話者1モデル」で品質を確立し、スタイル切り替えは将来のアーキテクチャ拡張として取り組む方が堅実。
