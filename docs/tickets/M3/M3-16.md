## 1. 目的とゴール

Unity 開発者が piper-plus の UPM パッケージを導入後、すぐに動作確認できるサンプルシーンと、ゲーム開発での TTS 活用方法を示すドキュメントを提供する。

**ゴール:**
- `Samples~/BasicTTS/` にワンクリックで動作するサンプルシーンを作成
- テキスト入力 -> 音声再生のデモ UI
- 非同期合成のベストプラクティスを示すサンプル
- ゲーム開発向けドキュメント (NPC 対話、UI読み上げ、字幕同期)

### 2.2 BasicTTSDemo.cs

```csharp
public class BasicTTSDemo : MonoBehaviour
{
    [SerializeField] private string modelPath = "StreamingAssets/piper-plus/tsukuyomi-chan-6lang-fp16.onnx";
    [SerializeField] private TMP_InputField textInput;
    [SerializeField] private Button synthesizeButton;
    [SerializeField] private AudioSource audioSource;
    [SerializeField] private TMP_Dropdown languageDropdown;

    private PiperTTS _tts;

    void Start()
    {
        _tts = new PiperTTS(Path.Combine(Application.streamingAssetsPath, "piper-plus", "model.onnx"));
        synthesizeButton.onClick.AddListener(OnSynthesize);
    }

    void OnSynthesize()
    {
        var clip = _tts.Synthesize(textInput.text, language: GetSelectedLanguage());
        audioSource.clip = clip;
        audioSource.Play();
    }

    void OnDestroy() => _tts?.Dispose();
}
```

### 2.3 AsyncTTSDemo.cs

メインスレッドをブロックしない非同期合成のベストプラクティス:

```csharp
public class AsyncTTSDemo : MonoBehaviour
{
    private PiperTTSAsync _tts;
    [SerializeField] private AudioSource audioSource;
    [SerializeField] private Slider progressBar;

    async void Start()
    {
        _tts = new PiperTTSAsync("...");
    }

    public async void OnSynthesizeClicked(string text)
    {
        progressBar.gameObject.SetActive(true);
        var clip = await _tts.SynthesizeAsync(text);
        progressBar.gameObject.SetActive(false);
        audioSource.clip = clip;
        audioSource.Play();
    }
}
```

### 2.4 NPCDialogDemo.cs

ゲーム開発での典型的なユースケース:

```csharp
public class NPCDialogDemo : MonoBehaviour
{
    [System.Serializable]
    public class DialogLine
    {
        public string text;
        public string language;  // "ja", "en", etc.
        public int speakerId;
    }

    [SerializeField] private DialogLine[] dialogLines;
    [SerializeField] private AudioSource audioSource;
    [SerializeField] private TMP_Text subtitleText;

    private PiperTTSAsync _tts;
    private int _currentLine;

    public async void PlayNextLine()
    {
        var line = dialogLines[_currentLine++];
        subtitleText.text = line.text;
        var clip = await _tts.SynthesizeAsync(line.text, language: line.language);
        audioSource.clip = clip;
        audioSource.Play();
    }
}
```

### 2.6 モデルファイル配置ガイド

```
Assets/
  StreamingAssets/
    piper-plus/
      tsukuyomi-chan-6lang-fp16.onnx  # モデル
      config.json                      # 設定
```

Unity の `StreamingAssets` に配置する理由:
- ビルド時にそのままコピーされる
- `Application.streamingAssetsPath` で OS 非依存のパス取得
- Android では APK 内に格納されるため、初回起動時に永続ストレージに展開する必要がある点を注記

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **モデルファイルの配布**: サンプルにモデルファイルを同梱すると数十 MB になる。初回起動時の自動 DL か、手動配置の手順書か
2. **Unity バージョン差異**: 2021.3 LTS と 2022.3 LTS で UI Toolkit / UGUI の API が異なる場合がある
3. **TextMeshPro 依存**: サンプルが TMP を使う場合、TMP Essentials のインポートが前提になる
4. **Android StreamingAssets**: Android では `StreamingAssets` は APK 内の圧縮ファイルとして格納されるため、直接ファイルパスでアクセスできない。UnityWebRequest 経由での展開が必要

### レビューチェックリスト

- [ ] サンプルシーンが Package Manager の "Import into Project" で正常にインポートされるか
- [ ] 全デモスクリプトに適切なコメント・XML doc が含まれるか
- [ ] モデルファイルの配置手順が README に明記されているか
- [ ] `Dispose()` パターンが全デモで正しく実装されているか
- [ ] ドキュメントに最小動作コード例 (3行) が含まれるか
- [ ] Android の StreamingAssets 制約について注意書きがあるか

## 6. 一から作り直すとしたら

サンプルシーンを「チュートリアル形式」で段階的に複雑度を上げる構成にする。Step 1: テキスト -> 音声 (3行)、Step 2: 非同期合成、Step 3: ストリーミング、Step 4: NPC対話。各ステップが独立したシーンになっていれば、開発者は必要な部分だけ参照できる。また、モデルファイルの配置を自動化する Editor スクリプト (初回インポート時にダイアログで DL 確認) があれば、初期セットアップの摩擦を大幅に削減できる。
