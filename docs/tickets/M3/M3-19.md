## 1. 目的とゴール

piper-plus の認知度を向上させるため、関連する Awesome リストや技術コミュニティのカタログに登録する。PR ベースでの外部リポジトリへの登録を行い、OSS エコシステムでの可視性を高める。

**ゴール:**
- Awesome TTS リストへの PR 提出
- Awesome Rust リストへの PR 提出
- その他関連 Awesome リスト (Awesome ONNX, Awesome Speech Synthesis 等) への PR 提出
- 登録数: 最低 3 件の PR 提出

### 2.2 PR テンプレート

各 Awesome リストの CONTRIBUTING.md に従い、以下の情報を含む PR を作成:

**エントリ例 (Awesome Rust):**
```markdown
- [piper-plus](https://github.com/ayutaz/piper-plus) - Multilingual neural text-to-speech engine supporting 8 languages (JA/EN/ZH/KO/ES/FR/PT/SV) with ONNX Runtime inference. Includes CLI, Python/C#/Go/WASM bindings.
```

**エントリ例 (Awesome TTS):**
```markdown
- [piper-plus](https://github.com/ayutaz/piper-plus) - VITS-based multilingual TTS system supporting 8 languages. No eSpeak-ng dependency (GPL-free). Available as Rust/Python/C#/Go/WASM libraries and CLI tools. Pre-trained models on HuggingFace.
```

### 2.3 PR 作成手順

1. 対象リポジトリをフォーク
2. CONTRIBUTING.md / ガイドラインを確認
3. 適切なカテゴリにエントリを追加
4. PR の説明に piper-plus の特徴を簡潔に記載
5. PR 提出後、メンテナの反応を待つ

### 2.4 piper-plus の差別化ポイント (PR 記載用)

| 特徴 | 詳細 |
|------|------|
| 多言語対応 | 8言語 (JA/EN/ZH/KO/ES/FR/PT/SV) |
| GPL フリー | eSpeak-ng 非依存、全 G2P が独自実装 |
| マルチプラットフォーム | Rust/Python/C#/Go/WASM/C API |
| 軽量 | FP16 ONNX モデル (~40MB)、CPU 推論可能 |
| プリビルドモデル | HuggingFace で公開済み |
| ストリーミング | 全実装でストリーミング合成対応 |

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **PR のマージは保証されない**: Awesome リストのメンテナが PR を受け入れるかは先方次第。リジェクトされた場合の代替策は不要
2. **リストの活性度**: 一部の Awesome リストはメンテナンスされていない可能性。star 数やコミット履歴で活性度を確認してから PR を作成
3. **自己推薦の是非**: Awesome リストによっては自己推薦を好まない場合がある。ガイドラインを事前確認
4. **タイミング**: M2 完了後が理想だが、現時点の機能 (8言語 G2P、4言語実装、HuggingFace モデル公開済み) で先行実行可能。Awesome リストの PR は機能追加を待たずに提出でき、後からエントリの説明を更新することも可能

### レビューチェックリスト

- [ ] 各 PR が対象リストの CONTRIBUTING ガイドラインに準拠しているか
- [ ] エントリの説明が簡潔かつ正確か
- [ ] リンクが有効か
- [ ] piper-plus の差別化ポイントが明確に伝わるか
- [ ] 自己推薦に関するガイドラインを確認したか

## 6. 一から作り直すとしたら

Awesome リストへの登録は低コストで認知度を上げる手段だが、最も効果が高いのは「使ってみた」記事や動画をコミュニティメンバーが自発的に作ることである。そのためには、README のクイックスタートが 3 ステップ以内で完了し、コピペで動くサンプルコードが充実していることが前提。Awesome リスト登録はその「入口」を作る施策であり、真の認知度向上は製品の使いやすさとドキュメントの質にかかっている。
