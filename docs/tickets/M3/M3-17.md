# [M3-17] iOS/Android ビルド CI 追加

**マイルストーン**: M3b (v2.1.0)
**カテゴリ**: ゲーム統合
**工数**: 3d
**依存**: M3-15
**ステータス**: 未着手

## 1. 目的とゴール

既存の `release-shared-lib.yml` CI を拡張し、iOS (arm64) と Android (arm64-v8a) 向けの libpiper_plus ネイティブライブラリをクロスコンパイルできるようにする。Unity UPM パッケージ (M3-15) の Plugins/ にモバイルバイナリを提供する。

**ゴール:**
- iOS arm64 向け静的ライブラリ (.a) のビルド CI
- Android arm64-v8a 向け共有ライブラリ (.so) のビルド CI (NDK)
- GitHub Releases にモバイルバイナリを追加
- 既存の Desktop ビルド (Linux/macOS/Windows) との統合

## 2. 実装内容の詳細

### 2.1 現状の CI

既存 `.github/workflows/release-shared-lib.yml`:
- Linux x86_64: `.so`
- macOS (Universal): `.dylib`
- Windows x86_64: `.dll`

追加するターゲット:
- iOS arm64: `.a` (静的ライブラリ)
- Android arm64-v8a: `.so` (NDK ビルド)

### 2.2 iOS ビルド

```yaml
ios-build:
  runs-on: macos-latest
  steps:
    - uses: actions/checkout@v4
    - name: Install dependencies
      run: |
        brew install cmake ninja
    - name: Build iOS static library
      run: |
        cmake -B build-ios \
          -G Ninja \
          -DCMAKE_TOOLCHAIN_FILE=cmake/ios.toolchain.cmake \
          -DPLATFORM=OS64 \
          -DCMAKE_OSX_ARCHITECTURES=arm64 \
          -DPIPER_PLUS_BUILD_SHARED=OFF \
          -DPIPER_PLUS_BUILD_STATIC=ON \
          -DCMAKE_BUILD_TYPE=Release \
          -DONNXRUNTIME_DIR=${{ github.workspace }}/onnxruntime-ios
        cmake --build build-ios --target piper_plus_static
    - name: Package
      run: |
        mkdir -p artifacts/ios
        cp build-ios/libpiper_plus.a artifacts/ios/
```

### 2.3 iOS CMake Toolchain

```cmake
# cmake/ios.toolchain.cmake
set(CMAKE_SYSTEM_NAME iOS)
set(CMAKE_OSX_DEPLOYMENT_TARGET "15.0")
set(CMAKE_OSX_ARCHITECTURES "arm64")
set(CMAKE_C_COMPILER_TARGET arm64-apple-ios15.0)
set(CMAKE_CXX_COMPILER_TARGET arm64-apple-ios15.0)
```

### 2.4 Android ビルド

```yaml
android-build:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - name: Setup NDK
      uses: nttld/setup-ndk@v1
      with:
        ndk-version: r26c
    - name: Build Android shared library
      run: |
        cmake -B build-android \
          -G Ninja \
          -DCMAKE_TOOLCHAIN_FILE=$ANDROID_NDK_HOME/build/cmake/android.toolchain.cmake \
          -DANDROID_ABI=arm64-v8a \
          -DANDROID_PLATFORM=android-24 \
          -DPIPER_PLUS_BUILD_SHARED=ON \
          -DCMAKE_BUILD_TYPE=Release \
          -DONNXRUNTIME_DIR=${{ github.workspace }}/onnxruntime-android
        cmake --build build-android --target piper_plus
    - name: Package
      run: |
        mkdir -p artifacts/android/arm64-v8a
        cp build-android/libpiper_plus.so artifacts/android/arm64-v8a/
```

### 2.5 ONNX Runtime モバイルビルド

iOS/Android 向けの ONNX Runtime を調達する必要がある:

| プラットフォーム | ONNX Runtime 配布 | 備考 |
|----------------|------------------|------|
| iOS | onnxruntime-ios (CocoaPods / 手動) | `onnxruntime.xcframework` |
| Android | onnxruntime-android (Maven / AAR) | `onnxruntime-android.aar` から `.so` を抽出 |

```bash
# iOS: xcframework から静的ライブラリを抽出
# Android: AAR から jni/arm64-v8a/libonnxruntime.so を抽出
```

### 2.6 CMake 変更

`cmake/PiperPlusShared.cmake` に以下を追加:

```cmake
option(PIPER_PLUS_BUILD_STATIC "Build static library" OFF)

if(PIPER_PLUS_BUILD_STATIC)
    add_library(piper_plus_static STATIC
        ${CMAKE_SOURCE_DIR}/src/cpp/piper_plus_c_api.cpp
        # ... 他のソースファイル
    )
    set_target_properties(piper_plus_static PROPERTIES
        OUTPUT_NAME piper_plus
    )
endif()
```

## 4. テスト計画

### 提供範囲（スコープ）
- IN: iOS/Android ビルド CI、CMake toolchain、リリースアーティファクト
- OUT: Unity での実機テスト (M3-16 後に手動)、iOS Simulator ビルド、armeabi-v7a サポート

### ユニットテスト

- iOS 静的ライブラリにシンボル (`piper_plus_create`, `piper_plus_synthesize` 等) が含まれるか (`nm -g`)
- Android 共有ライブラリにシンボルが含まれるか (`readelf -d`)
- ライブラリサイズが妥当な範囲か (静的: 50-150MB, 共有: 30-100MB)

### E2Eテスト

- CI で iOS ビルドが成功するか (macOS runner)
- CI で Android ビルドが成功するか (Linux runner + NDK)
- 生成されたアーティファクトが GitHub Releases にアップロードされるか
- 既存の Desktop ビルドが影響を受けないか (回帰テスト)

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **ONNX Runtime モバイルビルドの調達 (中リスク)**: iOS/Android 向けの ONNX Runtime プリビルドバイナリが公式で提供されているか確認が必要。提供されていない場合はソースからビルド (工数大幅増)
2. **iOS Bitcode**: Apple は Xcode 14 以降で Bitcode を非推奨化。Bitcode 無効でビルドする
3. **Android minSdkVersion**: ONNX Runtime Android が要求する最低 API レベルとの整合
4. **ライブラリサイズ**: モバイル向けでは ONNX Runtime 込みで 50MB 超になる可能性。UPM パッケージの総サイズが問題になる
5. **GPU プロバイダ**: iOS は CoreML、Android は NNAPI が利用可能だが、初期リリースでは CPU のみをサポートし、GPU 対応は将来課題

### レビューチェックリスト

- [ ] iOS toolchain のデプロイメントターゲットが適切か (15.0+)
- [ ] Android NDK バージョンが適切か (r26c)
- [ ] Android minSdkVersion が ONNX Runtime の要求と一致しているか
- [ ] 静的ライブラリに必要なシンボルが全て含まれているか
- [ ] 既存の Desktop ビルドが壊れていないか
- [ ] リリースアーティファクトの命名規則が統一されているか

## 6. 一から作り直すとしたら

ONNX Runtime のモバイルビルドは最も工数がかかる部分。ONNX Runtime Mobile (ORT Mobile) は軽量版で、モバイル向けに最適化されたビルドオプションがある。初回から ORT Mobile をターゲットにし、不要な OP (オペレーション) を除外してライブラリサイズを削減する戦略が望ましい。また、iOS は xcframework、Android は AAR として ONNX Runtime を統合すれば、CMake の依存管理が簡素化される。
