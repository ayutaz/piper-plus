**マイルストーン**: M3b (v2.1.0)
**カテゴリ**: モデルZoo
**工数**: 7d
**依存**: なし
**ステータス**: 未着手

## 1. 目的とゴール

韓国語 (KO) の G2P はコード上 8言語対応の一部として実装済み (g2pk2 ベース) だが、学習済みモデルが存在しない。韓国語データセットを選定・準備し、ベースモデルに KO を追加するか、シングルスピーカー KO モデルを作成する。

**ゴール:**
- 韓国語 TTS データセットの選定 (ライセンス: CC系 or 商用可能)
- 6言語ベースモデルからの転移学習で KO モデルを作成
- FP16 ONNX エクスポート + config.json 生成
- HuggingFace に公開
- KO G2P (g2pk2) の実運用での品質検証

## 2. 実装内容の詳細

### 2.0 emb_lang 拡張 (6->7言語) の技術検証 PoC (先行タスク)

**工数**: 2d (本チケット 7d のうち先頭 2d)

6言語ベースモデルに存在しない言語の `emb_lang` を追加する手順は未検証。データセット準備・学習に着手する前に、以下の PoC を実施する:

1. `emb_lang` 拡張スクリプトの作成 (セクション 2.4 参照)
2. 拡張後のチェックポイントで短時間学習 (10 epoch) を実行し、ロスが発散しないことを確認
3. 推論テスト: 既存6言語の品質が劣化していないことを確認

> **退路**: PoC で品質検証に失敗した場合 (ロス発散、既存言語の品質劣化が許容範囲外) は、本チケットを M3b スコープ外とし、8言語ベースモデル再学習を含む別チケットとして再計画する。

### 2.1 韓国語データセット候補

| データセット | ライセンス | 発話数 | 話者数 | SNR | 備考 |
|-------------|-----------|--------|--------|-----|------|
| CSS10 Korean | CC0 | ~6,600 | 1 | 中 | LibriVox由来、ライセンス制約なし |
| Zeroth-Korean | CC BY 4.0 | ~51h | 115 | 中 | 読み上げ音声、商用利用可 |
| KSS (Korean Single Speaker) | CC BY-NC-SA 4.0 | ~12,853 | 1 | 高 | **非商用のみ** (NC条項あり) |
| AIHub 韓国語音声 | 各種 | 大規模 | 多数 | 中-高 | 要利用申請 |

**推奨**: CSS10 Korean (CC0) を第一候補とする。ライセンス制約がなく、piper-plus の他の CC0 モデル (LJSpeech) と一貫性がある。品質不足の場合は Zeroth-Korean (CC-BY) にフォールバック。KSS は CC BY-NC-SA のため NC 条項が商用利用を制限し、第一候補としない。

### 2.2 データセット選定のフローチャート

```
1. CSS10 Korean (CC0) を評価
   ├─ 品質十分 → CSS10 Korean を採用
   └─ 品質不足 → 2. Zeroth-Korean (CC-BY) を評価
                     ├─ 品質十分 → Zeroth-Korean を採用
                     └─ 品質不足 → 3. KSS (NC-SA) を検討 (非商用限定モデルとして)
```

### 2.3 学習アプローチの選択

**方法 A: 6lang ベースからシングルスピーカー転移学習**
- 既存の 6lang ベースモデル (6言語) に KO は含まれていない
- `emb_lang` に ko=6 のスロットが G2P コード上は存在するが、ベースモデルの学習には含まれていない
- 転移学習時に `emb_lang[6]` (ko) を新規初期化する必要がある

**方法 B: 7言語ベースモデルを再学習**
- 6lang + KO = 7言語で事前学習をやり直す
- 工数が大きい (数日の GPU 時間)
- 他の全モデルとの互換性が崩れる

**推奨**: 方法 A。`--resume-from-multispeaker-checkpoint` の既存実装が `emb_lang` の動的拡張に対応可能か確認が必要。対応不可の場合は手動で `emb_lang` 拡張するスクリプトを作成。

### 2.4 emb_lang 拡張の実装

6lang ベースモデルは `emb_lang` が [0:5] (ja, en, zh, es, fr, pt)。KO (ko=6) を追加するには:

```python
# emb_lang 拡張スクリプト (概念)
checkpoint = torch.load("epoch=74-step=504712.ckpt")
old_emb = checkpoint["state_dict"]["enc_p.emb_lang.weight"]  # [6, dim]
new_emb = torch.zeros(7, old_emb.shape[1])
new_emb[:6] = old_emb
new_emb[6] = old_emb.mean(dim=0)  # KO を全言語平均で初期化
checkpoint["state_dict"]["enc_p.emb_lang.weight"] = new_emb
# dec, dp の emb_lang も同様に拡張
```

### 2.5 学習

```bash
uv run python -m piper_train \
  --dataset-dir /data/piper/dataset-ko-finetune \
  --prosody-dim 16 \
  --accelerator gpu --devices 1 --precision 32-true \
  --max_epochs 500 --batch-size 4 --samples-per-speaker 4 \
  --checkpoint-epochs 50 --quality medium \
  --base_lr 2e-5 --disable_auto_lr_scaling \
  --ema-decay 0.9995 --max-phoneme-ids 400 --no-wavlm \
  --val-every-n-epochs 50 --audio-log-epochs 50 \
  --resume-from-multispeaker-checkpoint \
    /data/piper/output-multilingual-6lang/checkpoints/epoch=74-step=504712-ko-extended.ckpt \
  --default_root_dir /data/piper/output-ko
```

### 2.6 品質検証

韓国語固有の検証項目:
- パッチム (받침) の発音: "한국" (han-guk), "읽다" (ik-da)
- 連音化: "한국어" -> /han.gu.gʌ/
- 激音化・濃音化: "학교" -> /hak.k͈jo/
- 助詞の発音: "은/는", "이/가"

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **データセット選定 (中リスク)**: 高品質で商用可能な KO データセットが限られている。KSS は品質が高いが NC-SA ライセンス
2. **emb_lang 拡張**: 6lang ベースモデルに存在しない言語の embedding を追加する手順が未検証。初期化方法 (ゼロ vs 平均 vs ランダム) が品質に影響する可能性
3. **g2pk2 依存**: KO G2P は g2pk2 (Apache-2.0) に依存。g2pk2 のメンテナンス状況を確認する必要がある
4. **工数超過リスク**: PoC (2d) + データ選定 + emb_lang 拡張 + 学習で 7d を超える可能性。データ選定で迷った場合は CSS10 Korean で先に進む

## 6. 一から作り直すとしたら

最初から 8言語 (6lang + KO + SV) のベースモデルを学習すべきだった。後から言語を追加する方式は emb_lang 拡張という非標準な手順が必要で、品質面でもベースモデルから学習した言語に劣る可能性がある。しかし、ベースモデルの再学習は GPU 数日分のコストがかかるため、転移学習で「まず動くモデルを出す」戦略は工数的に正しい。将来的に需要が確認できた時点で 8言語ベースモデルの再学習を検討する。
