**マイルストーン**: M3b (v2.1.0)
**カテゴリ**: モデルZoo
**工数**: 3d
**依存**: なし

## 1. 目的とゴール

英語の業界標準データセットである LJSpeech (CC0) を使用してシングルスピーカーモデルを学習し、HuggingFace に公開する。CC0 ライセンスにより商用・非商用問わず制約なく利用でき、piper-plus エコシステムの英語フラグシップモデルとなる。Issue #306 対応。

**ゴール:**
- 6言語ベースモデル (epoch=74-step=504712.ckpt) からの転移学習で LJSpeech モデルを作成
- FP16 ONNX エクスポート + config.json 生成
- HuggingFace リポジトリ `ayousanz/piper-plus-ljspeech` に公開
- `piper_plus_voices.json` にエントリ追加 (M3-18 で統合)
- `--list-models en` で表示確認

## 2. 実装内容の詳細

### 2.1 データセット準備

| 項目 | 値 |
|------|-----|
| データセット | LJSpeech-1.1 |
| ライセンス | CC0 (Public Domain) |
| 発話数 | ~13,100 |
| 話者数 | 1 (女性) |
| サンプリングレート | 22,050 Hz (元データ) -> 22,050 Hz (Piper標準) |
| 言語 | en |

```bash
# 1. LJSpeech ダウンロード
wget https://data.keithito.com/data/speech/LJSpeech-1.1.tar.bz2
tar xjf LJSpeech-1.1.tar.bz2

# 2. LJSpeech形式 -> Piper データセット形式変換
uv run python -m piper_train.tools.prepare_multilingual_dataset \
  --ljspeech-dir LJSpeech-1.1 \
  --output-dir /data/piper/dataset-ljspeech-finetune \
  --base-config /data/piper/dataset-multilingual-6lang-filtered/config.json
```

### 2.2 学習 (Template B: シングルスピーカーファインチューニング)

```bash
uv run python -m piper_train \
  --dataset-dir /data/piper/dataset-ljspeech-finetune \
  --prosody-dim 16 \
  --accelerator gpu --devices 1 --precision 32-true \
  --max_epochs 500 --batch-size 4 --samples-per-speaker 4 \
  --checkpoint-epochs 50 --quality medium \
  --base_lr 2e-5 --disable_auto_lr_scaling \
  --ema-decay 0.9995 --max-phoneme-ids 400 --no-wavlm \
  --val-every-n-epochs 50 --audio-log-epochs 50 \
  --resume-from-multispeaker-checkpoint \
    /data/piper/output-multilingual-6lang/lightning_logs/version_0/checkpoints/epoch=74-step=504712.ckpt \
  --default_root_dir /data/piper/output-ljspeech
```

### 2.3 ONNX エクスポート

```bash
CUDA_VISIBLE_DEVICES="" uv run python -m piper_train.export_onnx \
  /data/piper/output-ljspeech/lightning_logs/version_0/checkpoints/epoch=499-step=*.ckpt \
  /data/piper/output-ljspeech/ljspeech-en-medium.onnx
```

### 2.4 品質検証

```bash
# 英語テスト
CUDA_VISIBLE_DEVICES="" uv run python -m piper_train.infer_onnx \
  --model /data/piper/output-ljspeech/ljspeech-en-medium.onnx \
  --config /data/piper/dataset-ljspeech-finetune/config.json \
  --text "The quick brown fox jumps over the lazy dog." \
  --language en --output-dir /tmp/ljspeech-test

# 6言語テスト (多言語能力の保持確認)
for lang_text in "ja:こんにちは" "en:Hello world" "zh:你好世界" "es:Hola mundo" "fr:Bonjour le monde" "pt:Olá mundo"; do
  lang="${lang_text%%:*}"; text="${lang_text#*:}"
  uv run python -m piper_train.infer_onnx \
    --model /data/piper/output-ljspeech/ljspeech-en-medium.onnx \
    --config /data/piper/dataset-ljspeech-finetune/config.json \
    --text "$text" --language "$lang" --output-dir /tmp/ljspeech-test
done
```

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **LJSpeech の音声品質**: 録音は 2017年で比較的古い。SNR は中程度だが LJSpeech は業界標準として認知されている
2. **サンプリングレート**: LJSpeech は 22,050 Hz。Piper の標準と一致するため変換不要
3. **英語 prosody_features**: EN データには prosody_features (A1/A2/A3) が存在しないためゼロパディングとなる。学習済みベースモデルがこれを正しく扱えることは既に確認済み

### レビューチェックリスト

- [ ] CC0 ライセンスが Model Card に正しく記載されているか
- [ ] config.json の num_symbols が 173 (6lang 互換) であるか
- [ ] FP16 エクスポートが成功し、モデルサイズが妥当か
- [ ] 6言語全てで推論が成功するか (音声長 1-10s)
- [ ] HuggingFace リポジトリの README にサンプル音声と推論コマンドが含まれるか
- [ ] `md5_digest` が計算済みか

## 6. 一から作り直すとしたら

LJSpeech 単独の EN モデルではなく、最初から LibriTTS-R の高品質話者を選定して「EN リファレンスモデル」を作る戦略もあった。LJSpeech は CC0 の制約ゼロという利点で選定したが、品質面では HiFi-TTS (M3-11) の方が優れている可能性がある。モデルZoo戦略としては「ライセンス制約の少なさ」と「音声品質」の2軸でモデルを提供し、ユーザーに選択肢を与えるのが正しいアプローチ。
