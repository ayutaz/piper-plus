# [M3-3] Voice Cloning 対応モデル学習 (VITS2ベース)

**マイルストーン**: M3a (v2.0.0)
**カテゴリ**: Voice Cloning
**工数**: 10d
**依存**: M3-2, M3-5
**ステータス**: 未着手

---

## 1. 目的とゴール

M3-2 で追加した `speaker_embedding` 入力パスと M3-5 の VITS2 adversarial DP を組み合わせ、Voice Cloning 対応のマルチリンガルモデルを学習する。学習時は Speaker Encoder (M3-1) で抽出した embedding を使い、推論時に未知話者の参照音声から Voice Cloning を実現する。

**ゴール:**
- 6言語マルチリンガル Voice Cloning モデルの学習完了 (VITS2 + speaker_embedding)
- 学習済みモデルの ONNX エクスポート (FP16)
- Voice Cloning の品質評価 (MOS + 話者類似度)
- HuggingFace へのモデルアップロード

## 2. 実装内容の詳細

### 2.1 学習戦略

既存の 6lang ベースモデル (epoch=74, 571話者) を出発点とし、Speaker Encoder embedding を使った条件付けで追加学習する。

| フェーズ | 内容 | エポック | 目的 |
|---------|------|---------|------|
| Phase 1 | emb_g -> speaker_embedding 切り替え学習 | 30 | 既存 emb_g の知識を speaker_embedding に転移 |
| Phase 2 | 全パラメータ学習 (VITS2 DP) | 45 | VITS2 adversarial DP の最適化 |

### 2.2 データパイプラインの変更

学習時に各話者の embedding を Speaker Encoder で事前抽出し、キャッシュとして保存する。

```
dataset-multilingual-6lang-filtered/
  speaker_embeddings/          # NEW
    speaker_0000.npy           # (256,) float32
    speaker_0001.npy
    ...
    speaker_0570.npy
  speaker_embedding_cache.npz  # 全話者を1ファイルに統合
```

前処理スクリプト:
```bash
uv run python -m piper_train.speaker_encoder.extract_all \
  --dataset-dir /data/piper/dataset-multilingual-6lang-filtered \
  --encoder-model speaker_encoder.onnx \
  --output /data/piper/dataset-multilingual-6lang-filtered/speaker_embeddings/ \
  --num-samples-per-speaker 10
```

各話者から 10 発話をランダムサンプリングし、embedding を平均化して代表ベクトルとする。

### 2.3 学習コマンド

```bash
export WANDB_API_KEY=$(grep WANDB_API_KEY /data/piper/.env | cut -d= -f2) && \
NCCL_DEBUG=WARN NCCL_P2P_DISABLE=1 NCCL_IB_DISABLE=1 \
nohup /data/piper/.venv/bin/python -m piper_train \
  --dataset-dir /data/piper/dataset-multilingual-6lang-filtered \
  --prosody-dim 16 \
  --accelerator gpu --devices 4 --precision 32-true \
  --max_epochs 75 --batch-size 18 --samples-per-speaker 2 \
  --checkpoint-epochs 5 --quality medium \
  --base_lr 1e-4 --disable_auto_lr_scaling \
  --ema-decay 0.9995 \
  --max-phoneme-ids 400 \
  --no-wavlm \
  --audio-log-epochs 5 \
  --use-speaker-embedding \
  --speaker-embedding-cache /data/piper/dataset-multilingual-6lang-filtered/speaker_embedding_cache.npz \
  --vits2 \
  --resume-from-checkpoint /data/piper/output-multilingual-6lang/lightning_logs/version_0/checkpoints/epoch=74-step=504712.ckpt \
  --default_root_dir /data/piper/output-voice-cloning-6lang \
  > /data/piper/training_voice_cloning.log 2>&1 &
```

### 2.4 パラメータ設計

| パラメータ | 値 | 根拠 |
|-----------|-----|------|
| `--batch-size 18` | VITS2 DP + speaker_embedding で V100 16GB に収まるサイズ |
| `--base_lr 1e-4` | 既存モデルからの継続学習のため 2e-4 の半分 |
| `--use-speaker-embedding` | Speaker Encoder 出力を emb_g の代わりに使用 |
| `--vits2` | VITS2 adversarial DP を有効化 (M3-5) |
| `--max_epochs 75` | ~280K gradient steps (6lang と同等) |

### 2.5 推定GPU時間

| 項目 | 見積 |
|------|------|
| 前処理 (embedding 抽出) | ~2h (571話者 x 10発話) |
| Phase 1 (30 epoch) | ~37h (4x V100) |
| Phase 2 (45 epoch) | ~55h (4x V100) |
| 合計 | **~94h (~4日)** |

### 2.6 ONNX エクスポート

```bash
CUDA_VISIBLE_DEVICES="" uv run python -m piper_train.export_onnx \
  --speaker-embedding-input \
  /data/piper/output-voice-cloning-6lang/lightning_logs/version_0/checkpoints/best.ckpt \
  /data/piper/output-voice-cloning-6lang/voice-cloning-6lang.onnx
```

出力 ONNX モデルは `speaker_embedding` 入力テンソルを含む。既存の `sid` 入力も維持 (後方互換)。

### 2.7 品質評価

| メトリクス | 目標値 | 評価方法 |
|-----------|--------|---------|
| 話者類似度 (同一話者) | cosine sim > 0.80 | 参照音声 vs 合成音声の embedding 比較 |
| MOS (音質) | > 3.5 | 20人評価 x 30サンプル |
| MOS (話者類似度) | > 3.5 | 参照音声との類似度を5段階評価 |
| 既存話者の品質劣化 | MOS 差 < 0.3 | 6lang ベースモデルとの比較 |

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **GPU 時間**: ~94h (4x V100) はコスト的に1回の学習で成功させたい。ハイパーパラメータを事前に小規模データ (1000発話) でスイープすべき
2. **既存話者の品質劣化**: `emb_g` から `speaker_embedding` への切り替えで、既存話者の音質が劣化するリスク。Phase 1 で段階的に切り替えることで緩和
3. **Speaker Encoder の学習時 vs 推論時の不一致**: 学習時は同一話者の平均 embedding を使うが、推論時は1つの参照音声の embedding を使う。この不一致が Voice Cloning 品質に影響する可能性
4. **VITS2 の学習安定性**: adversarial DP は学習不安定になりやすい (M3-5 の懸念)。M3-5 を十分にテストしてから本チケットに着手すべき

### レビューチェックリスト

- [ ] WandB の学習曲線が正常 (loss 発散なし)
- [ ] Voice Cloning の話者類似度が目標値を達成
- [ ] 既存話者の品質劣化が許容範囲内
- [ ] ONNX エクスポートが正常 (既存テスト全件パス)
- [ ] FP16 変換後の品質劣化が許容範囲内
- [ ] HuggingFace にモデルがアップロードされている

## 6. 一から作り直すとしたら

**学習戦略の根本的再考:**

現在のアプローチは「既存 6lang モデルを出発点として speaker_embedding で追加学習」だが、ゼロから設計するなら以下のオプションがある:

1. **スクラッチ学習 (現案の代替)**: VITS2 + speaker_embedding をゼロから学習。既存モデルの知識を捨てる代わりに、speaker_embedding に最適化されたモデルが得られる。ただし ~200h の GPU 時間が必要
2. **2段階学習 (現案)**: 既存モデルから継続学習。GPU 時間は半分で済むが、emb_g -> speaker_embedding の切り替えで品質劣化のリスク
3. **Joint Training**: Speaker Encoder と VITS2 を end-to-end で学習 (YourTTS 方式)。最高品質が期待できるが、学習が複雑で不安定

piper-plus の現状 (GPU リソース制約、既存 6lang モデルの存在) を考えると、2段階学習 (現案) が最も現実的。ただし、Voice Cloning の品質が期待に達しない場合に Joint Training にスイッチする余地を残しておくべき。

**M3a 全体のタイムラインについて:** M3-3 は M3-2 と M3-5 の両方に依存している。VITS2 (M3-5) の実装が遅れると M3-3 も遅延する。ゼロから設計するなら、M3-5 を M3-GATE と並行して進めることで、クリティカルパスを短縮できたはず。

## 7. 後続タスクへの連絡事項

- **M3-4**: Voice Cloning モデルの ONNX 入力仕様を共有。`speaker_embedding` テンソルの前処理 (音声 -> メル -> ECAPA-TDNN -> embedding) を各ランタイムで実装する必要がある
- **M3-8**: MOS ベンチマークで Voice Cloning モデルを評価対象に含める。評価セットの設計は M3-3 の品質評価結果を参考に
