## 1. 目的とゴール

VITS の Duration Predictor (DP) を VITS2 の adversarial duration predictor にアップグレードし、より自然なプロソディ (韻律) を実現する。既存の VITS ONNX モデルとの後方互換性を維持しつつ、新規学習モデルでのみ VITS2 DP を使用する。

**ゴール:**
- VITS2 の Stochastic Duration Predictor (SDP) + Duration Discriminator を実装
- `--vits2` CLI フラグで VITS2 モードを有効化
- 既存 VITS ONNX モデルが v2.0.0 ランタイムでも動作すること (後方互換)
- VITS2 学習モデルが韻律の自然性で VITS を上回ること (MOS 比較)

## 2. 実装内容の詳細

| コンポーネント | VITS (現在) | VITS2 (新規) | 変更内容 |

| Duration Predictor | Deterministic DP | Stochastic DP + Duration Discriminator | adversarial 学習 |

**v2.0.0 のスコープ:** adversarial DP のみ。Flow の Transformer Block は v2.1.0 以降で検討。

### 2.6 ONNX エクスポートへの影響

Duration Discriminator は学習時のみ使用。推論グラフには含まれない。ONNX エクスポートの入出力テンソルは VITS1 と同一。

| 項目 | VITS1 ONNX | VITS2 ONNX | 互換性 |
|------|-----------|-----------|--------|
| 入力テンソル | input, input_lengths, scales, sid, lid | 同一 | 完全互換 |
| 出力テンソル | output, output_lengths | 同一 | 完全互換 |
| 推論ロジック | SDP (stochastic) | SDP V2 (stochastic, adversarial で学習) | 同一 |

つまり VITS2 で学習したモデルの ONNX は、VITS1 のランタイムでそのまま動作する。

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **学習不安定リスク**: adversarial 学習は GAN 学習と同様にモード崩壊のリスクがある。Duration Discriminator の学習率を VITS 本体より低く設定 (1e-4 vs 2e-4) し、warmup 期間 (最初の 10 epoch は DP のみ、Discriminator なし) を設けることで安定化
2. **既存 freeze-dp との相互作用**: ファインチューニング時の `--freeze-dp` は VITS2 の Duration Discriminator も凍結する必要がある。`configure_optimizers()` で Duration Discriminator のパラメータも除外
3. **optimizer 構成の複雑化**: VITS1 は generator + discriminator の 2 optimizer。VITS2 は generator + discriminator + duration_discriminator の 3 optimizer。PyTorch Lightning の multiple optimizer サポートを確認
4. **VITS2 論文の再現性**: VITS2 の公式実装は公開されていない。コミュニティ実装 (MB-iSTFT-VITS2 等) を参考にするが、再現性に懸念

### レビューチェックリスト

## 6. 一から作り直すとしたら

**VITS2 移行のタイミングと範囲:**

現在のアプローチは「VITS2 の adversarial DP のみを段階的に導入」だが、ゼロから設計するなら以下の選択肢がある:

1. **Adversarial DP のみ (現案)**: 最小変更で韻律改善。リスク低。ただし VITS2 の他の改善 (Flow の Transformer Block) は含まない
2. **VITS2 完全移行**: Flow の Transformer Block も含む完全な VITS2 実装。品質向上が大きいが、計算コスト増 (+20-30%) と学習不安定リスクが高い
3. **VITS2 スキップ → VITS3/Matcha-TTS**: Flow Matching ベースの最新アーキテクチャに直接ジャンプ。品質は最高だが、既存モデルとの非互換が大きい

piper-plus の「段階的改善」の方針を考えると、(1) が最も現実的。(2) は v2.1.0、(3) は v3.0.0 のロードマップとして検討すべき。

**adversarial DP の設計について:**

VITS2 論文の Duration Discriminator は比較的シンプル (3層 Conv1d) だが、より強力な Discriminator (Transformer-based, multi-scale) を使うことで品質を向上させる余地がある。ただし計算コストとのトレードオフ。

**Go/No-Go ゲートとの関係:** M3-5 は M3-GATE に依存しない (Voice Cloning とは独立した音質改善)。ゲートが No-Go でも M3-5 の成果は v2.0.0 に含まれる。この設計判断は正しい -- Voice Cloning が延期されても VITS2 による音質改善は単独で価値がある。
