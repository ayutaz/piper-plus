# [M3-11] HiFi-TTS モデル学習 + 公開 (EN, CC-BY)

**マイルストーン**: M3b (v2.1.0)
**カテゴリ**: モデルZoo
**工数**: 3d
**依存**: なし
**ステータス**: 未着手

---

## 1. 目的とゴール

HiFi-TTS データセット (CC BY 4.0) を使用して高品質な英語シングルスピーカーモデルを学習し公開する。HiFi-TTS は高 SNR で収録されたプロフェッショナル品質のデータセットで、男性/女性話者を選択可能。LJSpeech (M3-9) が CC0 で制約ゼロのモデルであるのに対し、本モデルは CC-BY で高品質な選択肢を提供する。Issue #307 対応。

**ゴール:**
- HiFi-TTS から高品質話者 (男性 or 女性) を1名選定
- 6言語ベースモデルからの転移学習でモデルを作成
- FP16 ONNX エクスポート + config.json 生成
- HuggingFace リポジトリ `ayousanz/piper-plus-hifi-tts` に公開

## 2. 実装内容の詳細

### 2.1 データセット情報

| 項目 | 値 |
|------|-----|
| データセット | HiFi-TTS (hi_fi_tts_v0) |
| ライセンス | CC BY 4.0 |
| 話者数 | 10 (6女性 + 4男性) |
| サンプリングレート | 44,100 Hz (ダウンサンプリング必要) |
| SNR | 高 (プロフェッショナル録音) |
| 言語 | en |

### 2.2 話者選定基準

以下の基準で1名を選定:

1. **発話数**: 多いほど学習品質が向上。最低 2,000 発話以上
2. **SNR**: 全話者が高品質だが、個体差を確認
3. **声質の差別化**: LJSpeech (女性) と異なる声質が望ましい。男性話者を優先検討
4. **録音時間**: 長時間のデータがある話者を優先

### 2.3 データセット準備

```bash
# 1. HiFi-TTS ダウンロード
# HuggingFace: https://huggingface.co/datasets/Bingsu/Hi-Fi_TTS
# または公式サイトから取得

# 2. 話者選定 + LJSpeech形式変換
uv run python -m piper_train.tools.prepare_multilingual_dataset \
  --ljspeech-dir /data/piper/hifi-tts-selected/ \
  --output-dir /data/piper/dataset-hifi-tts-finetune \
  --base-config /data/piper/dataset-multilingual-6lang-filtered/config.json

# 3. サンプリングレート変換 (44.1kHz -> 22.05kHz)
# prepare_multilingual_dataset が自動でリサンプリングする前提
```

### 2.4 学習

```bash
uv run python -m piper_train \
  --dataset-dir /data/piper/dataset-hifi-tts-finetune \
  --prosody-dim 16 \
  --accelerator gpu --devices 1 --precision 32-true \
  --max_epochs 500 --batch-size 4 --samples-per-speaker 4 \
  --checkpoint-epochs 50 --quality medium \
  --base_lr 2e-5 --disable_auto_lr_scaling \
  --ema-decay 0.9995 --max-phoneme-ids 400 --no-wavlm \
  --val-every-n-epochs 50 --audio-log-epochs 50 \
  --resume-from-multispeaker-checkpoint \
    /data/piper/output-multilingual-6lang/lightning_logs/version_0/checkpoints/epoch=74-step=504712.ckpt \
  --default_root_dir /data/piper/output-hifi-tts
```

### 2.5 品質検証

LJSpeech モデル (M3-9) との比較評価:
- 同一テキストでの音声品質比較
- 長文 (100語超) での安定性
- 速度・ピッチの自然さ

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **サンプリングレート変換**: 44.1kHz -> 22.05kHz のダウンサンプリングで品質劣化の可能性。適切なアンチエイリアシングフィルタが必要
2. **話者選定の主観性**: 「高品質」の判断に主観が入る。可能なら MOS スコア等の客観指標も参照
3. **CC-BY の帰属表記**: Model Card に HiFi-TTS のクレジットを正確に記載する必要

## 6. 一から作り直すとしたら

HiFi-TTS は話者数が10名いるため、マルチスピーカーモデルとして全話者を含める選択肢もあった。ユーザーが `--speaker-id` で男性/女性を切り替えられるモデルは魅力的だが、モデルサイズが増大し、シングルスピーカーモデルと比較して個々の話者品質が下がるリスクがある。モデルZoo戦略としては、まずシングルスピーカーで品質を確立し、需要があればマルチスピーカー版を追加する段階的アプローチが正しい。
