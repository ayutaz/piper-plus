**マイルストーン**: M3b (v2.1.0)
**カテゴリ**: モデルZoo
**工数**: 3d
**依存**: なし
**ステータス**: 未着手

---

## 1. 目的とゴール

SIWIS French Speech Synthesis Database (CC BY 4.0) を使用してフランス語初のシングルスピーカーモデルを学習し公開する。現在 piper-plus のモデルZoo は日本語 (つくよみちゃん, CSS10) のみ。フランス語モデルの追加により、多言語エコシステムの実用性を示す。Issue #308 対応。

**ゴール:**
- SIWIS データセット (~9,750 発話) でフランス語モデルを学習
- FP16 ONNX エクスポート + config.json 生成
- HuggingFace リポジトリ `ayousanz/piper-plus-siwis-fr` に公開
- フランス語として初のシングルスピーカーモデル

### 2.2 データセット準備

```bash
# 1. SIWIS ダウンロード
# https://datashare.ed.ac.uk/handle/10283/2353

# 2. LJSpeech 形式変換
uv run python -m piper_train.tools.prepare_multilingual_dataset \
  --ljspeech-dir /data/piper/siwis-fr/ \
  --output-dir /data/piper/dataset-siwis-fr-finetune \
  --base-config /data/piper/dataset-multilingual-6lang-filtered/config.json
```

### 2.3 学習

```bash
uv run python -m piper_train \
  --dataset-dir /data/piper/dataset-siwis-fr-finetune \
  --prosody-dim 16 \
  --accelerator gpu --devices 1 --precision 32-true \
  --max_epochs 500 --batch-size 4 --samples-per-speaker 4 \
  --checkpoint-epochs 50 --quality medium \
  --base_lr 2e-5 --disable_auto_lr_scaling \
  --ema-decay 0.9995 --max-phoneme-ids 400 --no-wavlm \
  --val-every-n-epochs 50 --audio-log-epochs 50 \
  --resume-from-multispeaker-checkpoint \
    /data/piper/output-multilingual-6lang/lightning_logs/version_0/checkpoints/epoch=74-step=504712.ckpt \
  --default_root_dir /data/piper/output-siwis-fr
```

### 2.4 品質検証

フランス語固有の検証項目:
- リエゾン (liaison) の処理: "les amis" -> /le.z‿a.mi/
- ナザール母音: "un bon vin blanc" の鼻母音
- 無音の e (schwa): "je ne sais pas" の e muet
- アクセント記号つき文字: "éléphant", "où", "ça"

```bash
# フランス語テスト
CUDA_VISIBLE_DEVICES="" uv run python -m piper_train.infer_onnx \
  --model /data/piper/output-siwis-fr/siwis-fr-medium.onnx \
  --config /data/piper/dataset-siwis-fr-finetune/config.json \
  --text "Bonjour, comment allez-vous aujourd'hui?" \
  --language fr --output-dir /tmp/siwis-test
```

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **フランス語 G2P の品質**: 現在のフランス語 G2P は規則ベース (依存なし)。リエゾン等の複雑な音韻規則が正確にカバーされているかはモデル品質に直結
2. **サンプリングレート**: SIWIS のサンプリングレートが 44.1kHz の場合、ダウンサンプリングが必要
3. **学習データとベースモデルの FR 発話**: 6lang ベースモデルは CML-TTS French (107,464 発話, 28 話者) で事前学習済み。FR の DP パラメータは十分に学習されているはず

### レビューチェックリスト

- [ ] CC BY 4.0 の帰属表記 (IDIAP) が Model Card に正しく記載されているか
- [ ] フランス語固有の音韻テスト (リエゾン、ナザール母音) に合格しているか
- [ ] config.json の num_symbols が 173 (6lang 互換) であるか
- [ ] 6言語全てで推論が成功するか
- [ ] `md5_digest` が計算済みか

## 6. 一から作り直すとしたら

フランス語モデルの品質は G2P の精度に大きく依存する。現在の規則ベースフランス語 G2P で品質が不十分な場合、espeak-ng ベースの G2P にフォールバックする選択肢もあるが、GPL 依存を回避する方針のため採用しない。代替として、フランス語の発音辞書 (Lexique 等) を活用した辞書ベース G2P の強化が将来的に必要になる可能性がある。言語カバレッジ (多くの言語で「まず出す」) と品質深堀り (1言語の完璧な G2P) のトレードオフでは、現段階では前者を優先する判断が正しい。
