## 1. 目的とゴール

`test_wavlm_discriminator.py` の `TestWavLMResampling` クラス (L99-184) は、リサンプリング品質を検証するために `resampler_sinc` という**独自のフィクスチャ**を定義し、本番コードの `WavLMDiscriminator.resampler` を使用していない。

**問題点**:
1. **本番コードとのリンクがない**: フィクスチャが `torchaudio.transforms.Resample(...)` を独立に構築しており、本番コードのパラメータ (例: `lowpass_filter_width=64`, `resampling_method="sinc_interp_hann"`) と乖離する可能性がある
2. **パラメータ同期の負担**: 本番コードで resampler のパラメータを変更した場合、テスト側のフィクスチャも手動で同期する必要がある
3. **テストの信頼性**: テストが PASS しても、本番コードの resampler が正しく動作する保証にならない

**現在のテストコード** (`src/python/tests/test_wavlm_discriminator.py` L99-184):

### 2.3 テストメソッドへの影響

`test_sinc_resampling_preserves_low_frequencies` と `test_sinc_vs_linear_aliasing` は `resampler_sinc(audio)` で呼び出しており、`torchaudio.transforms.Resample` インスタンスの `__call__` を使用する。`WavLMDiscriminator.resampler` も同じ `torchaudio.transforms.Resample` インスタンスなので、呼び出し側の変更は不要。

## 5. 懸念事項とレビュー項目

1. **WavLMDiscriminator の初期化コスト (重要度: 高)**: M2-1 と同じ懸念。`WavLMDiscriminator.__init__()` は WavLM モデル (数百 MB) をダウンロードする。`resampler_sinc` フィクスチャはテストメソッドごとに呼ばれるため、`scope="class"` に変更して初期化回数を削減すべき。ただし同クラスの `wavlm_discriminator` フィクスチャ (L24-26) は `scope="class"` で別クラス `TestWavLMDiscriminatorFeatureMapFormat` に定義されているため、`TestWavLMResampling` にも `scope="class"` のフィクスチャを追加するか、conftest.py に共有フィクスチャを移動する。

2. **resampler のみ取得する軽量代替案**: `WavLMDiscriminator` 全体を初期化すると WavLM のダウンロードが走る。resampler だけが必要な場合、以下の軽量代替案がある:
   - `disc.resampler` を `__init__` 直後に取得し、`disc` を破棄してメモリを解放
   - ただし本番コードとの紐付けという目的からは、フルインスタンスからの取得が最も信頼性が高い

3. **`resampler_linear` フィクスチャの扱い**: `resampler_linear` は本番コードに対応物がないため、比較基準として現状維持が適切。ただしコメントで「これは比較用であり本番コードの実装ではない」と明記すべき。

4. **フィクスチャスコープの整合性**: 現在の `resampler_sinc` はデフォルト `scope="function"` だが、resampler は状態を持たないため `scope="class"` で十分。初期化コスト削減のため scope 変更を推奨。

## 6. 一から作り直すとしたら

1. **conftest.py に共有 `wavlm_resampler` フィクスチャを定義**: `scope="session"` で 1 回だけ `WavLMDiscriminator` を初期化し、`.resampler` を返す。`test_vits.py` (M2-1) と `test_wavlm_discriminator.py` (M2-2) で共有。

2. **mock を使った軽量初期化**: `WavLMDiscriminator.__init__` の `WavLMModel.from_pretrained` を mock して、resampler の初期化のみ実行する。これにより数百 MB のモデルダウンロードを回避しつつ、resampler のパラメータは本番コードと同一になる:
   ```python
   with unittest.mock.patch("piper_train.vits.models.WavLMModel"):
       disc = WavLMDiscriminator(source_sample_rate=22050, target_sample_rate=16000)
   return disc.resampler
   ```

3. **パラメータの定数化**: resampler のパラメータ (`sinc_interp_hann`, `lowpass_filter_width=64`) を `WavLMDiscriminator` のクラス定数として公開すれば、テストでパラメータの一致を直接検証できる。
