## 1. 目的とゴール

`test_vits.py` の `test_wavlm_discriminator_resample` テスト (L114-134) は、WavLMDiscriminator のリサンプリングロジックを検証する目的で書かれているが、**本番コードの `WavLMDiscriminator._resample()` メソッドを一切呼ばず**、代わりに `torchaudio.functional.resample()` を直接呼び出してロジックを再実装している。

**問題点**:
1. **テストが本番コードをテストしていない**: `torchaudio.functional.resample()` の直接呼び出しは、`_resample()` メソッドのロジック (squeeze, float32 変換, `self.resampler` 経由の sinc 補間) と無関係
2. **本番コードとの乖離リスク**: `_resample()` の実装が変更されてもこのテストは影響を受けず、リグレッションを検出できない
3. **sinc vs functional の差異**: 本番コードは `torchaudio.transforms.Resample` (sinc_interp_hann, lowpass_filter_width=64) を使用するが、テストは `torchaudio.functional.resample()` (デフォルトパラメータ) を使用しており、実際には異なるリサンプリング結果となる

**現在のテストコード** (`src/python/tests/test_vits.py` L114-134):

```python
@pytest.mark.unit
@pytest.mark.training
def test_wavlm_discriminator_resample(self):
    """Test audio resampling logic used by WavLMDiscriminator"""
    import torchaudio

    source_sample_rate = 22050
    target_sample_rate = 16000

    # Create test audio (batch=2, 1 channel, 1 second at 22050Hz)
    audio = torch.randn(2, 1, source_sample_rate)

    # Squeeze channel dim and resample (same logic as WavLMDiscriminator._resample)
    audio_squeezed = audio.squeeze(1)
    resampled = torchaudio.functional.resample(
        audio_squeezed, source_sample_rate, target_sample_rate
    )

    # Expected length: 22050 * (16000 / 22050) = 16000
    expected_length = target_sample_rate
    assert resampled.shape == (2, expected_length), f"Expected shape (2, {expected_length}), got {resampled.shape}"
```

**本番コード** (`src/python/piper_train/vits/models.py` L554-628):

```python
class WavLMDiscriminator(torch.nn.Module):
    def __init__(self, ...):
        # ...
        # Initialize resampler with sinc interpolation for high-quality audio resampling
        self.resampler = None
        if source_sample_rate != target_sample_rate:
            self.resampler = torchaudio.transforms.Resample(
                orig_freq=source_sample_rate,
                new_freq=target_sample_rate,
                resampling_method="sinc_interp_hann",
                lowpass_filter_width=64,
                dtype=torch.float32,
            )

    def _resample(self, audio: torch.Tensor) -> torch.Tensor:
        """Resample audio from source to target sample rate using sinc interpolation."""
        if self.resampler is None:
            return audio.squeeze(1)
        audio_2d = audio.squeeze(1)
        audio_float = audio_2d.float()
        resampled = self.resampler(audio_float)
        return resampled  # (batch, time)
```

**完了の定義**:
- テストが `WavLMDiscriminator` をインスタンス化し、`_resample()` メソッドを直接呼び出す
- `torchaudio.functional.resample()` の直接呼び出しがテストから除去される
- 出力形状 `(batch, time)` の検証が維持される
- 既存の全テストが PASS する

## 5. 懸念事項とレビュー項目

1. **WavLMDiscriminator の初期化コスト (重要度: 高)**: `WavLMDiscriminator.__init__()` は `WavLMModel.from_pretrained("microsoft/wavlm-base-plus")` を呼び、HuggingFace から数百 MB のモデルをダウンロードする。CI 環境でのテスト速度とネットワーク依存に影響する。既存の `test_wavlm_discriminator.py` が同じ初期化を行っており (`scope="class"` フィクスチャ)、CI で既に許容されているなら問題ない。もし問題になる場合、`_resample()` のテストでは WavLM 本体を mock して resampler のみ初期化する方法を検討する。

2. **`_resample()` はプライベートメソッド**: テストがプライベートメソッドを直接呼び出すことになる。ただし `_resample()` はリサンプリングという独立した責務を持ち、`forward()` 経由のテストでは GPU が必要なため、プライベートメソッドの直接テストは許容される。

3. **既存テストとの重複**: `test_wavlm_discriminator.py` の `TestWavLMResampling` クラスが同様のリサンプリングテストを含む (ただしこちらも独自 fixture `resampler_sinc` を使用しており、M2-2 で別途修正予定)。テスト間の役割分担を明確にすること。

## 6. 一から作り直すとしたら

1. **`_resample()` を公開メソッドにする**: リサンプリングは独立した責務であり、`resample()` として公開 API にすればテストの正当性が向上する。ただし M2 は本番コード変更不要の制約があるため見送り。

2. **フィクスチャの共有**: `test_vits.py` と `test_wavlm_discriminator.py` で `WavLMDiscriminator` インスタンスを conftest.py の共有フィクスチャとして定義すれば、初期化コストを `scope="session"` で 1 回に抑えられる。

3. **resampler のみの軽量テスト**: `WavLMDiscriminator` 全体を初期化せず、`self.resampler` (= `torchaudio.transforms.Resample`) のみを取り出してテストする方法もある。ただし `_resample()` の squeeze/float32 変換ロジックもテスト対象に含めたいため、メソッド呼び出しが望ましい。
