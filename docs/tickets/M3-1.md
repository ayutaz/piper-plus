## 1. 目的とゴール

`test_swedish_phonemizer.py` L28-46 にテスト専用のヘルパー関数 `_split_ipa_to_phonemes()` が定義されている。この関数は「以前 `piper_train.phonemize.swedish` にあったものをテスト互換性のために再実装した」とコメントされているが、本番コードには対応する関数が存在しない (grep で確認済み: テストファイル以外にヒットなし)。

**現状のコード** (`test_swedish_phonemizer.py` L28-46):

```python
def _split_ipa_to_phonemes(ipa_str: str) -> list[str]:
    """Split an IPA string into individual phoneme tokens.

    Handles long vowels (V + \u02D0), stress markers, and multi-codepoint
    sequences.  This helper was previously in piper_train.phonemize.swedish
    and is reproduced here for test compatibility.
    """
    tokens: list[str] = []
    i = 0
    while i < len(ipa_str):
        ch = ipa_str[i]
        # Check if next character is the length mark (long vowel)
        if i + 1 < len(ipa_str) and ipa_str[i + 1] == "\u02D0":
            tokens.append(ch + "\u02D0")
            i += 2
        else:
            tokens.append(ch)
            i += 1
    return tokens
```

**問題点**:
1. 本番コードに対応する関数が存在しないため、docstring の「previously in `piper_train.phonemize.swedish`」は不正確
2. テスト内のヘルパーとして使用されているが、現在この関数を呼んでいるテストケースがあるか確認が必要
3. IPA 文字列の分割ロジックが正しいかの保証がない (ストレスマーカーの処理はコメントに記載あるが実装に含まれていない)

**完了の定義** (以下のいずれか):
- **Option A (本番移動)**: `_split_ipa_to_phonemes()` を `piper_plus_g2p/swedish.py` に移動し、テストはそれを import する
- **Option B (テスト内整理)**: 使用箇所がなければ削除。使用箇所があれば docstring を正確に修正し、テスト用ユーティリティとして明示する
- **Option C (conftest.py 移動)**: 複数テストファイルで使用される場合は `conftest.py` に移動

## 5. 懸念事項とレビュー項目

1. **ストレスマーカーの処理**: docstring に「stress markers」の処理があると記載されているが、実装には含まれていない (長母音マーカー `\u02D0` の処理のみ)。ストレスマーカー (`ˈ`, `ˌ`) も複合トークンとして扱うべきか検討する。

2. **本番への移動の是非**: `_split_ipa_to_phonemes` が本番コードで必要になる場面 (例: IPA 出力のトークン数カウント、音素単位の後処理) が想定されるなら Option A が適切。現時点では不要なら YAGNI 原則に従い削除または docstring 修正に留める。

## 6. 一から作り直すとしたら

SwedishPhonemizer の出力が最初からトークンリスト (`list[str]`) であれば、IPA 文字列を後からトークンに分割するヘルパーは不要。`_phonemize_word()` が既にリストを返しているため、`_split_ipa_to_phonemes()` はテスト内で IPA の結合文字列を再分割する逆変換として存在している。テスト設計として、結合前のトークンリストを直接検証する方が自然。
