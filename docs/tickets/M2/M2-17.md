```
Hi HN! I built Piper Plus, a fork of Piper TTS that replaces eSpeak-ng with
custom G2P implementations for 8 languages (JA/EN/ZH/KO/ES/FR/PT/SV).

Why:
- eSpeak-ng is GPL, limiting commercial use
- eSpeak-ng requires an external binary, complicating deployment
- eSpeak-ng doesn't run in browsers (WASM)

What's different:
- Rule-based G2P for each language (no external deps for ES/FR/PT/SV)
- Runs in browsers via WASM: `npm install piper-plus`
- 5 platforms: Python, Rust, C#, Go, JavaScript/WASM
- VITS architecture, ONNX Runtime inference
- OpenAI-compatible TTS API included
- Wyoming Protocol for Home Assistant (WIP)

Quick start:
  pip install piper-tts-plus
  echo "Hello" | piper-tts-plus --model tsukuyomi -o hello.wav

Demo: [HuggingFace space or audio samples link]
Models: https://huggingface.co/ayousanz/piper-plus-tsukuyomi-chan

Happy to answer questions about the G2P design, WASM porting, or training process.
```

## 5. 懸念事項とレビュー項目

### 懸念事項
- HN はセルフプロモーションに厳しい。「Show HN」のガイドラインに従い、技術的な内容に焦点を当てる。マーケティング的な表現は避ける
- eSpeak-ng / 元の Piper プロジェクトに対するネガティブな印象を与えないよう、「代替手段を提供」というポジションを維持する
- 投稿がフロントページに上がらない可能性がある。その場合のリカバリー策 (再投稿のタイミング、タイトル変更) を事前に検討する
- 大量のトラフィックが GitHub に来た場合、README の品質と Quick start の動作確認が重要。壊れていると第一印象が悪くなる

## 6. 一から作り直すとしたら

- ライブデモ (ブラウザで音声合成を試せる Web ページ) を用意してから投稿すべきだった。HN ユーザーは「試せるデモ」があると圧倒的にエンゲージメントが高い。GitHub リンクだけでは離脱率が高い
- 投稿前に HN で活発なユーザー (TTS/ML 関連のコメント歴がある人) に事前に連絡してフィードバックを得るべきだった
- A/B テスト的に、タイトルのバリエーションを2-3案用意し、最もクリック率が高そうなものを選ぶプロセスがあればよかった
