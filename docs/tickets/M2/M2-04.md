# [M2-4] PyPI `piper-plus` パッケージ公開 (高レベルAPI)

**マイルストーン**: M2a (v1.13.0)
**カテゴリ**: API設計
**工数**: 0.5d
**依存**: M2-3
**ステータス**: 未着手

---

## 1. 目的とゴール

M2-1 ~ M2-3 で実装・テスト済みの高レベル Python API を PyPI に `piper-plus` パッケージとして公開する。`pip install piper-plus` でエンドユーザーが即座に利用可能にする。

**ゴール:**
- PyPI に `piper-plus` パッケージを公開
- `pip install piper-plus` で基本機能が動作
- `pip install piper-plus[play]` で音声再生機能が追加
- GitHub Actions で自動リリース (タグトリガー)

## 2. 実装内容の詳細

### 2.1 pyproject.toml

配置パス: `src/python/piper_plus/pyproject.toml`

```toml
[project]
name = "piper-plus"
version = "1.13.0"
description = "High-quality multilingual neural TTS — no espeak-ng required"
readme = "src/python/piper_plus/README.md"
license = {text = "MIT"}
requires-python = ">=3.10"
dependencies = [
    "numpy>=1.24",
    "piper-plus-g2p>=0.3",
    "huggingface-hub>=0.20",
]

[project.optional-dependencies]
cpu = ["onnxruntime>=1.16"]
gpu = ["onnxruntime-gpu>=1.16"]
play = ["sounddevice>=0.4"]
dev = ["pytest>=8", "ruff>=0.4"]
```

> **onnxruntime と onnxruntime-gpu の pip 競合回避:** `onnxruntime` と `onnxruntime-gpu` は同一 Python 環境に共存できない (pip が一方を上書きする)。このため、base dependencies から `onnxruntime` を除外し、`[cpu]` と `[gpu]` の extras に分離する。ユーザーは明示的にどちらかを選択する:
> - CPU: `pip install piper-plus[cpu]`
> - GPU: `pip install piper-plus[gpu]`
>
> `PiperPlus.__init__` で ORT が import できない場合は、`ImportError` で `pip install piper-plus[cpu]` または `pip install piper-plus[gpu]` のインストール手順を案内する。

### 2.2 パッケージ構造

既存の `piper-plus-g2p` (PyPI 公開済み) との共存:
- `piper-plus`: 高レベル API (本パッケージ)
- `piper-plus-g2p`: G2P のみ (既存、依存として利用)

### 2.3 リリース CI

`.github/workflows/python-api-publish.yml`:

```yaml
on:
  push:
    tags: ["python-api-v*"]
jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      id-token: write  # Trusted Publisher
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v4
      - run: uv build src/python/piper_plus/
      - uses: pypa/gh-action-pypi-publish@release/v1
```

### 2.4 Trusted Publisher 設定

PyPI の Trusted Publisher を使用 (API トークン不要):
- Publisher: GitHub Actions
- Repository: ayutaz/piper-plus
- Workflow: python-api-publish.yml

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **パッケージ名の衝突**: PyPI に既存の `piper-plus` がないか事前確認が必要
2. **onnxruntime vs onnxruntime-gpu**: 両方を同時にインストールすると競合。extras で分離するが、ユーザーへのガイダンスが必要
3. **piper-plus-g2p のバージョン互換**: g2p 側の破壊的変更に対するバージョンピンの粒度
