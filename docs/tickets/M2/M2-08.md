#### 2.1.1 テストコマンド一覧

```markdown
## Running Tests

### Python (G2P)
```bash
cd src/python/g2p && uv run pytest -v
```

> **Note:** `uv run pytest` はワーキングディレクトリの `pyproject.toml` を参照して依存を解決する。必ず対象パッケージのディレクトリ (`src/python/g2p` 等) に `cd` してから実行すること。リポジトリルートからの `uv run pytest src/python/g2p/tests/` は依存が解決できず失敗する場合がある。

### Python (Training)
```bash
cd src/python && uv run pytest tests/ -v -m unit
```

### Rust
```bash
cd src/rust && cargo test -p piper-plus --no-fail-fast
```

### C#
```bash
dotnet test src/csharp/PiperPlus.sln -c Release
```

### Go
```bash
cd src/go && go test -v -race ./... ./phonemize/...
```

### C++
```bash
cmake -B build && cmake --build build && cd build && ctest
```

> **前提:** C++ ビルドには onnxruntime の共有ライブラリが必要。入手方法は [onnxruntime リリースページ](https://github.com/microsoft/onnxruntime/releases) からプラットフォームに合った pre-built バイナリをダウンロードし、`ONNXRUNTIME_ROOT` 環境変数でパスを指定する。詳細は `cmake/FindOnnxRuntime.cmake` を参照。

### WASM (JavaScript)
```bash
cd src/wasm/g2p && node --test test/test-*.js
```
```

#### 2.1.2 初回PR手順

```markdown
## Your First Pull Request

1. Fork the repository on GitHub
2. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/piper-plus.git
   cd piper-plus
   ```
3. Create a feature branch from `dev`:
   ```bash
   git checkout dev
   git pull origin dev
   git checkout -b feature/your-change
   ```
4. Make your changes
5. Run the relevant tests (see "Running Tests" above)
6. Run linting:
   ```bash
   ruff check && ruff format --check
   ```
7. Commit and push:
   ```bash
   git add <files>
   git commit -m "description of your change"
   git push origin feature/your-change
   ```
8. Open a PR targeting the `dev` branch
```

## 4. テスト計画

### 提供範囲（スコープ）
- IN: CONTRIBUTING.md の内容の正確性、テストコマンドの動作

## 5. 懸念事項とレビュー項目

### 懸念事項

1. **コマンドの陳腐化**: テストコマンドが変更された場合に CONTRIBUTING.md が追従しないリスク。CI のテストステップと CONTRIBUTING.md を同期する仕組みを検討
2. **C++ ビルドの前提条件**: onnxruntime の入手方法がプラットフォームにより異なる。詳細は別ドキュメントへリンク
3. **Go テストの CGo 依存**: Go テストが C ライブラリに依存する場合の前提条件

- [ ] 全テストコマンドが実際に動作するか
- [ ] Python のコマンドが `uv run` を使用しているか (`python -m pytest` ではなく)
- [ ] `dev` ブランチが PR ターゲットとして明記されているか
