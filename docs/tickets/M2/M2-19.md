# M2-19 Rust API ergonomics 改善 (SynthesisParams + Default)

**マイルストーン**: M2b (v1.14.0)
**カテゴリ**: API設計
**工数**: 2d
**依存**: なし
**ステータス**: 未着手

---

## 1. 目的とゴール

Rust 推論エンジン (`piper-core`) の公開 API をより Rust-idiomatic に改善し、ユーザーが少ないボイラープレートで音声合成を実行できるようにする。`SynthesisParams` 構造体に `Default` トレイトを実装し、旧 API は `#[deprecated]` で並行提供する。

完了時の状態:
- `SynthesisParams` 構造体がデフォルト値付きで定義されている
- `Default` トレイトが実装され、`..Default::default()` パターンで使用できる
- 旧 API に `#[deprecated(since = "...", note = "...")]` が付与されている
- 既存のテストが新旧両方の API で通る
- `piper-plus-cli` が新 API を使用するようにリファクタリングされている

## 2. 実装内容の詳細

### 新 API 設計

```rust
/// 音声合成パラメータ
#[derive(Debug, Clone)]
pub struct SynthesisParams {
    /// 話者 ID (マルチスピーカーモデル用)
    pub speaker_id: Option<i64>,
    /// 言語コード (マルチリンガルモデル用)
    pub language: Option<String>,
    /// ノイズスケール (音声の変動量、0.0-1.0)
    pub noise_scale: f32,
    /// 長さスケール (話速、1.0 = 標準)
    pub length_scale: f32,
    /// ノイズスケール W (持続時間の変動量)
    pub noise_w: f32,
    /// 文間の無音 (秒)
    pub sentence_silence: f64,
}

impl Default for SynthesisParams {
    fn default() -> Self {
        Self {
            speaker_id: None,
            language: None,
            noise_scale: 0.667,
            length_scale: 1.0,
            noise_w: 0.8,
            sentence_silence: 0.2,
        }
    }
}
```

### 使用例 (Before / After)

**Before (旧 API)**:
```rust
let result = engine.synthesize(text, 0, Some("ja"), 0.667, 1.0, 0.8, 0.2)?;
```

**After (新 API)**:
```rust
let result = engine.synthesize_with_params(text, &SynthesisParams {
    speaker_id: Some(0),
    language: Some("ja".into()),
    ..Default::default()
})?;
```

### 対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `src/rust/piper-core/src/engine.rs` | `SynthesisParams` 定義、`synthesize_with_params()` 追加 |
| `src/rust/piper-core/src/lib.rs` | `SynthesisParams` を公開 API としてエクスポート |
| `src/rust/piper-cli/src/main.rs` | 新 API を使用するようにリファクタリング |
| `src/rust/piper-python/src/lib.rs` | Python バインディングで `SynthesisParams` を使用 |

### 後方互換性

- 旧 API (`synthesize()` の個別引数版) は `#[deprecated]` を付けて残す
- セマンティックバージョニングに従い、マイナーバージョンアップ (v1.14.0) で提供
- 次のメジャーバージョンで旧 API を削除予定

```rust
#[deprecated(
    since = "1.14.0",
    note = "Use `synthesize_with_params()` with `SynthesisParams` instead"
)]
pub fn synthesize(
    &self,
    text: &str,
    speaker_id: i64,
    language: Option<&str>,
    noise_scale: f32,
    length_scale: f32,
    noise_w: f32,
    sentence_silence: f64,
) -> Result<SynthesisResult, PiperError> {
    self.synthesize_with_params(text, &SynthesisParams {
        speaker_id: Some(speaker_id),
        language: language.map(String::from),
        noise_scale,
        length_scale,
        noise_w,
        sentence_silence,
    })
}
```

## 5. 懸念事項とレビュー項目

### 懸念事項
- `SynthesisParams` の `language` フィールドを `String` にするか `&str` にするか。ライフタイムの複雑さを避けるため `Option<String>` を採用するが、呼び出し側で `.into()` が必要になる
- `sentence_silence` を `SynthesisParams` に含めるべきか。文間の無音は合成パラメータというよりポスト処理パラメータ。ただし CLI からの1対1マッピングの利便性を優先して含める
- `Builder` パターン (`SynthesisParamsBuilder`) の導入も検討したが、フィールド数が少ない (6個) ため `Default + 構造体更新構文` で十分と判断

### レビューチェックリスト
- [ ] `SynthesisParams` のデフォルト値が既存の動作と一致する
- [ ] `#[deprecated]` の `since` バージョンが正しい
- [ ] 旧 API が新 API に正しく委譲されている
- [ ] `cargo doc` でドキュメントが正しく生成される
- [ ] `cargo clippy` で警告が出ない (deprecation warning を除く)

## 6. 一から作り直すとしたら

- 最初から `SynthesisParams` パターンで設計すべきだった。Rust では引数が4つ以上の関数は構造体にまとめるのが慣例。初期実装で個別引数にした技術負債を今回解消する
- `Builder` パターン + `#[derive(TypedBuilder)]` を使えばさらに ergonomic だが、外部クレートへの依存が増える。標準ライブラリの範囲で解決するほうが依存グラフがクリーン
- `SynthesisParams` をジェネリックにして、モデル固有のパラメータを拡張可能にする設計も考えられたが、現時点では YAGNI

## 7. 後続タスクへの連絡事項

- Python バインディング (`piper-python`) は内部実装で `SynthesisParams` を使用するが、Python 側の API は変更しない (Python ではキーワード引数が自然なため)
- WASM バインディングも同様に内部で `SynthesisParams` を使用するが、JS API は変更しない
- 旧 API の削除は次のメジャーバージョン (v2.0.0) で実施。CHANGELOG に deprecation notice を記載する
- `cargo doc` の出力で新 API が目立つようにドキュメントコメントを充実させる
