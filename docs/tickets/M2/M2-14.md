## 1. 目的とゴール

webpack および Vite プロジェクトで piper-plus の WASM ファイルを正しく配置・ロードするためのガイドを作成する。バンドラー環境での WASM 配置は初心者がつまずきやすいポイントであり、コピー&ペースト可能な設定例を提供することで導入障壁を下げる。

完了時の状態:
- `docs/guides/wasm-bundler.md` に webpack 5 / Vite それぞれの設定例が記載されている
- WASM ファイルの配置パス設定、公開ディレクトリ設定、MIME タイプ設定が網羅されている
- onnxruntime-web の WASM バックエンド設定が含まれている
- Create React App / Next.js / Nuxt.js のフレームワーク別の注意点が記載されている

### Vite 設定例

```typescript
// vite.config.ts
import { defineConfig } from 'vite';

export default defineConfig({
  optimizeDeps: {
    exclude: ['piper-plus', 'onnxruntime-web'],
  },
  server: {
    headers: {
      'Cross-Origin-Opener-Policy': 'same-origin',
      'Cross-Origin-Embedder-Policy': 'require-corp',
    },
  },
});
```

### webpack 5 設定例

```javascript
// webpack.config.js
module.exports = {
  experiments: {
    asyncWebAssembly: true,
  },
  module: {
    rules: [
      {
        test: /\.wasm$/,
        type: 'asset/resource',
      },
    ],
  },
};
```

### COOP/COEP ヘッダー説明
onnxruntime-web が SharedArrayBuffer を使用する場合、サーバー側で Cross-Origin-Opener-Policy / Cross-Origin-Embedder-Policy ヘッダーの設定が必要。開発サーバーとプロダクション環境それぞれの設定方法を記載する。

## 5. 懸念事項とレビュー項目

### 懸念事項
- onnxruntime-web のバージョンによって WASM ファイルの配置パスが異なる。特定バージョン (1.24.x) を前提とした記述になるが、バージョンアップ時にガイドが古くなるリスク
- COOP/COEP ヘッダーの設定はサーバーに依存する。Netlify / Vercel / Cloudflare Pages 等のホスティングサービスごとの設定方法まで網羅すると工数が膨らむ。主要サービスのみ言及する
- Create React App は非推奨 (2023年以降) だが、既存プロジェクトで使われているケースが多い。言及は必要だが最新推奨ではないことを注記する

- [ ] webpack 設定例が webpack@5.x で動作する
- [ ] COOP/COEP の説明が正確で、設定の必要性が明確に記述されている
- [ ] フレームワーク別の注意点が最新バージョンと一致している
- [ ] トラブルシューティングがよくあるエラーメッセージをカバーしている

## 6. 一から作り直すとしたら

- バンドラー設定をプリセットパッケージ (`@piper-plus/vite-plugin`, `piper-plus-webpack-plugin`) として提供すべきだった。ドキュメントで設定をコピーさせるよりも、`import piperPlusPlugin from '@piper-plus/vite-plugin'` でゼロコンフィグにするほうがユーザー体験が良い
- GitHub リポジトリに `examples/vite/` `examples/webpack/` のような動作するサンプルプロジェクトを含めるべきだった。ドキュメントだけでは設定の正確さを保証しにくい
