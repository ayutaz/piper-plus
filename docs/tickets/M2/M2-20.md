**マイルストーン**: M2b (v1.14.0)
**カテゴリ**: 技術負債
**工数**: 1d
**依存**: なし
**ステータス**: 未着手

## 1. 目的とゴール

`piper-core` の SV (スウェーデン語) phonemizer は既に実装済み (`src/rust/piper-core/src/phonemize/swedish.rs`) だが、推論エンジンの言語登録パスとの統合テストが不足している。SV phonemizer が engine レベルで正しく選択・呼び出されることを検証し、SV モデル学習に向けた準備を完了させる。

完了時の状態:
- `piper-core` で SV 言語が指定された場合に `SwedishPhonemizer` が正しく選択される統合テストが存在する
- engine レベルの言語自動検出でスウェーデン語テキストが SV と判定される
- SV phonemizer の音素化品質テスト (M2-21 で yeager にフィードバック依頼するための基盤)
- 他の7言語と同等のテストカバレッジがSV にも存在する

## 2. 実装内容の詳細

### 現状の確認

SV phonemizer は以下のレベルで実装済み:
- `piper-plus-g2p` crate: SV G2P ロジック (PR #294 ベース)
- `piper-core/src/phonemize/swedish.rs`: Phonemizer トレイト実装
- `piper-core/src/phonemize/mod.rs`: `pub mod swedish;` でエクスポート済み

### 不足している部分

1. **engine レベルの統合テスト**: `engine.rs` で SV 言語指定時に `SwedishPhonemizer` が選択・使用されることのテスト
2. **マルチリンガル文脈でのテスト**: `MultilingualPhonemizer` でスウェーデン語テキストが正しく検出・処理されること
3. **品質テスト**: スウェーデン語の一般的な単語・フレーズで音素化結果が妥当であること

### テストファイル

| ファイル | 内容 |
|---------|------|
| `src/rust/piper-core/tests/test_swedish_integration.rs` | engine レベル統合テスト (新規) |
| `src/rust/piper-plus-g2p/tests/swedish_test.rs` | G2P 品質テスト (既存があれば拡張) |

### テストケース例

```rust
#[test]
fn test_swedish_phonemizer_selection() {
    // engine に SV 言語が設定されている場合、SwedishPhonemizer が使用される
}

#[test]
fn test_swedish_basic_words() {
    // "hej" -> /hɛj/
    // "tack" -> /tak/
    // "Stockholm" -> /stɔkhɔlm/
}

#[test]
fn test_swedish_sentence() {
    // "Hur mår du idag?" の音素化が妥当
}

#[test]
fn test_multilingual_swedish_detection() {
    // スウェーデン語テキストが SV と検出される
}
```

### 品質改善 (該当する場合)

yeager (M2-21) からのフィードバックを反映するための基盤を整備:
- 音素化結果のスナップショットテスト
- 既知の問題点をテストケースとして記録 (xfail)
- SV 特有の発音規則 (sj-ljud, tj-ljud, retroflexer) のテストカバレッジ

## 4. テスト計画

### 提供範囲 (スコープ)
- IN: SV phonemizer の統合テスト、品質テスト、engine レベルの言語選択テスト
- OUT: SV モデルの学習、SV G2P ロジックの大規模変更、他言語 phonemizer の変更

### ユニットテスト
- `SwedishPhonemizer::phonemize_with_prosody()` の基本動作テスト
- SV 特有の発音規則 (sj-ljud, tj-ljud) のテスト
- 空文字列、特殊文字のエッジケーステスト

### E2Eテスト
- engine 経由で SV テキストの音素化 -> 音素ID変換が正しく動作することを確認
- マルチリンガルモデルの config で SV 言語が含まれる場合のテスト
- CI (`rust-tests.yml`) で SV テストが3 OS で通ることを確認

## 5. 懸念事項とレビュー項目

### 懸念事項
- SV の学習済みモデルが存在しないため、実際の推論テスト (音声生成) はできない。音素化レベルのテストに限定される
- SV の発音規則はネイティブスピーカー (yeager) の知見が必要。このチケットでは基本テストを作成し、M2-21 でのフィードバックを反映する前提
- `UnicodeLanguageDetector` でスウェーデン語テキストを正しく検出できるかは、スウェーデン語特有の文字 (å, ä, ö) の扱いに依存する

### レビューチェックリスト
- [ ] 統合テストが engine レベルでの言語選択を正しく検証している
- [ ] テストケースが SV の一般的な単語・フレーズをカバーしている
- [ ] 既存の7言語のテストパターンと一貫性がある
- [ ] CI で3 OS すべてでテストがパスする
- [ ] テストケースがフィードバック反映のための拡張ポイントを持っている

## 6. 一から作り直すとしたら

- SV G2P の PR #294 をマージする際に、統合テストを必須条件にすべきだった。G2P ロジックとエンジン統合テストを同時にレビューすることで、抜け漏れを防止できた
- 音素化品質のリグレッションテストフレームワーク (ゴールデンファイル方式) を全言語共通で作成すべき。各言語の「期待される音素化結果」をファイルで管理し、変更時に差分を検出する

## 7. 後続タスクへの連絡事項

- M2-21 (yeager へのフィードバック依頼): このチケットで作成したテストケースと音素化結果をフィードバック依頼の素材として使用する。「これらの単語の音素化は正しいですか？」という具体的な質問ができる
- SV モデルの学習は M3 以降で検討。学習データの調達 (Mozilla Common Voice SV 等) が必要
- KO (韓国語) も同様に統合テストが不足している可能性がある。このチケットのパターンを KO にも適用できる
