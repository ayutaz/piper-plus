## 1. 目的とゴール

Wyoming Protocol アダプタが Home Assistant の音声パイプラインで正しく動作することを実機で検証し、HA ユーザー向けのセットアップガイドを作成する。HA の Assist パイプラインに piper-plus を TTS として組み込む手順を明文化する。

完了時の状態:
- Home Assistant 実機 (または HA Container) で Wyoming piper-plus が TTS プロバイダーとして認識される
- Assist パイプラインから音声合成が動作する
- `docs/guides/home-assistant.md` にセットアップ手順が日英で記載されている
- トラブルシューティングセクションが含まれている

## 2. 実装内容の詳細

### 統合テスト手順

1. HA Container (Docker) を起動
2. Wyoming piper-plus コンテナを起動 (`docker-compose up -d`)
3. HA > Settings > Devices & Services > Add Integration > Wyoming Protocol
4. Wyoming サーバーの URI (`tcp://<host>:10200`) を入力
5. TTS プロバイダーとして piper-plus が認識されることを確認
6. Assist パイプラインの TTS を piper-plus に設定
7. テキスト入力で音声合成を実行
8. 各言語 (JA/EN/ZH/ES/FR/PT) でテストを実行

### テストシナリオ

| # | シナリオ | 期待結果 |
|---|---------|---------|
| 1 | HA から Wyoming discovery | piper-plus が TTS として表示される |
| 2 | 短い日本語テキスト合成 | 3秒以内に音声再生 |
| 3 | 英語テキスト合成 | 言語自動検出で正しく英語として処理 |
| 4 | 長文テキスト合成 (200文字以上) | タイムアウトなく音声が生成される |
| 5 | HA 再起動後の再接続 | 自動再接続で TTS が復旧 |
| 6 | Wyoming コンテナ再起動 | モデル再ダウンロードなし、10秒以内に復旧 |

## 5. 懸念事項とレビュー項目

### 懸念事項
- HA のバージョンにより Wyoming 統合の UI が異なる。2024.x 以降を前提とするが、古いバージョンのユーザーへの対応は OUT
- Wyoming discovery の `info` イベントで返す `name` / `description` / `attribution` / `languages` の仕様が HA 側の期待と一致するか要確認
- HA アドオンストアへの公開は M2b スコープ外。ドキュメントでは Docker Compose 方式のみ記載する
- Raspberry Pi での応答速度が実用的 (5秒以内) でない場合、ドキュメントに制限事項として明記する

## 6. 一から作り直すとしたら

- HA アドオンストア形式 (`config.yaml` + `Dockerfile` in addon repo) で最初から開発すべきだった。Docker Compose よりも HA ユーザーにとって圧倒的に導入が簡単 (UI でワンクリック)。ただし HA アドオンストアへの公開にはレビュープロセスがあるため、M2b では Docker Compose で先行リリースし、アドオン化は M3 で対応する
- テストを自動化すべき。HA の Docker コンテナ + Wyoming コンテナを docker-compose で立ち上げ、Wyoming クライアントで自動テストする CI パイプラインがあれば回帰テストが容易になる
