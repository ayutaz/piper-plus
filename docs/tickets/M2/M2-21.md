**マイルストーン**: M2b (v1.14.0)
**カテゴリ**: コミュニティ
**工数**: 0.25d
**依存**: M2-20
**ステータス**: 未着手

## 1. 目的とゴール

PR #294 でスウェーデン語 G2P の完全実装を提出した Daniel Nylander (yeager, Stockholm) にフィードバックを依頼し、SV phonemizer の品質向上とコミュニティリレーションシップの強化を図る。ネイティブスピーカーの言語知識は自動テストでは検出できない発音ルールの誤りを発見するために不可欠。

完了時の状態:
- yeager に対してフィードバック依頼のコメントが GitHub Issue/Discussion またはPR上に投稿されている
- M2-20 で作成したテストケースの音素化結果を具体的に提示し、正確性の確認を依頼している
- SV 特有の発音課題 (sj-ljud, tj-ljud, retroflexer) について意見を求めている
- 継続的なコントリビューションへの感謝と今後の協力依頼が含まれている

## 2. 実装内容の詳細

### フィードバック依頼の内容

GitHub Discussion または PR #294 へのコメントとして以下を投稿:

```markdown
Hi @yeager,

Thank you again for the excellent Swedish G2P implementation in PR #294!
We've been working on integration tests for the SV phonemizer (M2-20) and
would love your feedback on the phonemization quality.

Here are some test results we'd appreciate you reviewing:

| Input | Current output | Expected? |
|-------|---------------|-----------|
| "hej" | /hɛj/ | ? |
| "tack" | /tak/ | ? |
| "Stockholm" | /stɔkhɔlm/ | ? |
| "Hur mår du idag?" | ... | ? |
| "sjuksköterskeutbildning" | ... | ? |

Specific areas we'd like your expertise on:
1. **sj-ljud handling**: Are words like "sjuk", "skjorta" correctly phonemized?
2. **tj-ljud handling**: How about "tjugo", "kärlek"?
3. **Retroflexes**: "barn", "ord" - are the retroflex assimilations captured?
4. **Pitch accent**: Any thoughts on handling Swedish word accents?

We're also planning SV model training in a future milestone. If you're
interested in helping with training data selection or quality evaluation,
we'd welcome your involvement!

Best regards
```

### 依頼のポイント

1. **具体的な質問**: 抽象的な「品質どうですか？」ではなく、具体的な単語・フレーズの音素化結果を提示して Yes/No で回答しやすくする
2. **感謝と敬意**: PR #294 の貢献に対する具体的な感謝
3. **将来の参画機会**: SV モデル学習への関与の可能性を提示
4. **低負荷**: テーブル形式で回答しやすく、5分程度で回答できる量に限定

## 4. テスト計画

## 5. 懸念事項とレビュー項目

### 懸念事項
- yeager がアクティブでない可能性がある。GitHub の最終アクティビティを確認し、1週間以上返信がない場合は催促しない (コミュニティ関係を損なわないため)
- フィードバック依頼が「仕事を押し付けている」と受け取られないよう、あくまで「お願い」のトーンで、強制感のない表現を使う
- SV の発音規則に関する議論が技術的に深くなりすぎると、回答のハードルが上がる。最初は基本的な単語レベルに限定する

## 6. 一から作り直すとしたら

- PR #294 のレビュー段階で、マージ前にネイティブスピーカーレビューを依頼するプロセスを確立すべきだった。マージ後のフィードバック依頼は後付け感がある
- SV コミュニティ (Swedish Linux Users Group 等) に広くフィードバックを求めるべきだった。1人のコントリビューターに依存するのはリスク
- 音素化結果の音声サンプル (SV モデルがなくても、音素列から合成した概念的な音声) があればフィードバックの精度が上がる

## 7. 後続タスクへの連絡事項

- フィードバックで発見された問題は新規 Issue として起票し、SV G2P の改善チケット (M3 以降) に反映する
- yeager が SV モデル学習に興味を示した場合、学習データの調達 (Mozilla Common Voice SV、国立図書館のパブリックドメイン音声等) を共同で検討する
- KO (韓国語) にも同様のネイティブスピーカーフィードバック体制を構築することを M3 で検討する
