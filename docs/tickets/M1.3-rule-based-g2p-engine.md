# [M1.3] Rule-based G2P エンジン実装

## マイルストーン紐づけ
- **マイルストーン**: M1.3
- **Phase**: Phase 1
- **対応要件**: FR-03a〜FR-03g
- **依存先チケット**: M1.2 (音素インベントリ + PUA割り当て)
- **ブロックするチケット**: M1.4 (辞書ルックアップ統合)

## 達成したい内容

espeak-ng 不使用で OOV語 (辞書に存在しない語) を IPA 音素列に変換する Rule-based G2P エンジンを `src/python/piper_train/phonemize/swedish.py` に実装する。エンジンは 7 つのサブ要件 (FR-03a〜FR-03g) をカバーし、以下の処理パイプラインを単語単位で実行する。

```
入力: 単語 (str, 小文字正規化済み)
  │
  [Stage 2] detect_loanword_suffix()     ← FR-03g (接尾辞: -tion/-sion/-age 等)
  [Stage 3] detect_loanword_prefix()     ← FR-03g (接頭辞: sch/ch/sh/ph/th)
  [Stage 4] _convert_word_native()       ← FR-03a + FR-03c + FR-03d + FR-03e
  [Stage 5] apply_retroflex()            ← FR-03b (後処理: r+C → retroflex)
  [Stage 6] detect_stress() + apply_stress_markers() ← FR-03f
  │
  出力: list[str] (IPA音素列)
```

**精度目標**: soft/hard 全問正解、sj-sound >=90%、母音長 >=95%、全体 >=95% (辞書併用で >=97-99%)

**主要な成果物**:
- `_convert_consonant()` -- 20 規則の最長一致子音変換 (FR-03a, FR-03c)
- `apply_retroflex()` -- 状態マシンによるレトロフレックス同化 (FR-03b)
- `get_vowel_phoneme()` -- 18 母音音素の長短判定 (FR-03d)
- `get_unstressed_vowel()` / `_apply_unstressed_suffix()` -- 16 接尾辞パターンの非強勢母音短縮 (FR-03e)
- `detect_stress()` -- 5 段階優先順位のストレス検出 (FR-03f)
- `detect_loanword_suffix()` / `detect_loanword_prefix()` -- 7+5 パターンのローンワード規則 (FR-03g)
- 例外リスト: `HARD_K_WORDS` (~80語), `HARD_G_WORDS` (~60語), `O_LONG_AS_OO` (~28語), `FUNCTION_WORDS` (~35語), `FINAL_M_SHORT_WORDS` (~17語), `SK_BACK_VOWEL_EXCEPTIONS` (2語)
- 形態論ヒューリスティック: `_is_hard_k()`, `_is_hard_g()` (語尾剥がし + 語幹辞書)
- `_convert_word_native()` -- Stage 4 の統合ループ
- `_convert_word_with_loanwords()` -- Stage 2-4 の統合パイプライン
