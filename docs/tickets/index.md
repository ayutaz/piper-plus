### 実行順序

1. **並行着手可能**: M1.1 (データ準備) と M1.2 (音素インベントリ) は依存関係がなく並行実施可能
2. **M1.3**: M1.2 完了後に着手 (音素インベントリが G2P 規則の前提)
3. **M1.4**: M1.1 と M1.3 の両方が完了後に着手 (辞書 + 規則エンジンの統合)
4. **M1.5**: M1.4 完了後に着手 (完成した Phonemizer をレジストリに統合)
5. **M1.6**: M1.5 完了後に着手 (全統合完了後にテストスイート実装)
6. **Phase 2/3/4**: Phase 1 (M1.6) 完了後に並行着手可能
