## 1. タスク目的とゴール

C/C++ プロジェクトから `pkg-config --cflags --libs piper_plus` で簡単にリンクできるようにする。pkg-config は CMake を使わないプロジェクト (Godot GDExtension の SCons, Meson, 手動 Makefile) で特に有用。

**ゴール:** install 先の `lib/pkgconfig/piper_plus.pc` が生成され、以下のコマンドが動作すること。

```bash
# コンパイル + リンク
gcc $(pkg-config --cflags --libs piper_plus) -o test test.c

# フラグ確認
pkg-config --cflags piper_plus      # => -I/usr/local/include
pkg-config --libs piper_plus        # => -L/usr/local/lib -lpiper_plus -lonnxruntime
pkg-config --modversion piper_plus  # => 1.10.0

# 辞書パスの取得
pkg-config --variable=dictdir piper_plus  # => /usr/local/share/open_jtalk/dic
```

## 2. 実装する内容の詳細

### 2.1 変更対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `cmake/piper_plus.pc.in` (新規) | pkg-config テンプレート |
| `CMakeLists.txt` | `configure_file()` + install ルール追加 |

### 2.2 pkg-config テンプレートファイル

`cmake/piper_plus.pc.in`:

```
# piper_plus.pc -- pkg-config file for piper-plus C API shared library
# Generated by CMake. Do not edit manually.

prefix=@CMAKE_INSTALL_PREFIX@
exec_prefix=${prefix}
libdir=${prefix}/@CMAKE_INSTALL_LIBDIR@
includedir=${prefix}/@CMAKE_INSTALL_INCLUDEDIR@

# Directories for bundled data
dictdir=${prefix}/share/open_jtalk/dic
g2p_dictdir=${prefix}/share/piper/dicts

Name: piper_plus
Description: piper-plus TTS engine C API shared library
URL: https://github.com/ayutaz/piper-plus
Version: @piper_version@
Requires:
Libs: -L${libdir} -lpiper_plus
Libs.private: -lonnxruntime
Cflags: -I${includedir}
```

**設計判断:**

| 項目 | 判断 | 根拠 |
|------|------|------|
| `Libs` に `-lonnxruntime` を含めない | `Libs.private` に配置 | 消費側は `libpiper_plus` を経由して ORT を利用するため、直接リンクは不要。`--static` 時のみ表示される |
| `dictdir` / `g2p_dictdir` 変数 | カスタム変数として追加 | `pkg-config --variable=dictdir piper_plus` でユーザーが辞書パスを取得可能。Godot GDExtension 等で有用 |
| `Requires` が空 | ORT の pkg-config は提供されていない | ORT を `Requires` に含めると `pkg-config` が見つからずエラーになる |
| `@CMAKE_INSTALL_LIBDIR@` 使用 | M3-1 の GNUInstallDirs と整合 | Debian multiarch (`lib/x86_64-linux-gnu`) 等に対応 |

### 2.3 CMakeLists.txt への変更

M3-1 の `if(PIPER_PLUS_BUILD_SHARED)` ブロック内に追加:

```cmake
if(PIPER_PLUS_BUILD_SHARED)
  include(GNUInstallDirs)

  # pkg-config ファイル生成
  configure_file(
    ${CMAKE_CURRENT_SOURCE_DIR}/cmake/piper_plus.pc.in
    ${CMAKE_CURRENT_BINARY_DIR}/piper_plus.pc
    @ONLY
  )

  install(FILES ${CMAKE_CURRENT_BINARY_DIR}/piper_plus.pc
    DESTINATION ${CMAKE_INSTALL_LIBDIR}/pkgconfig
  )
endif()
```

`@ONLY` は `${variable}` 形式の置換を無効にし、`@variable@` 形式のみ置換する。`.pc.in` 内の `${prefix}`, `${libdir}` 等は pkg-config 自身の変数展開なので、CMake に置換されてはならない。

### 2.4 Windows での扱い

Windows では pkg-config は一般的ではないが、MSYS2/vcpkg ユーザー向けに生成しておく。Windows 固有の変更は不要 -- `configure_file()` は全プラットフォームで動作する。CMake Config (M3-3) が Windows での主要な利用手段となる。

## 4. 提供範囲とテスト項目

### 4.1 ユニットテスト

| テスト | 内容 | 検証方法 |
|--------|------|---------|
| ファイル存在 | `lib/pkgconfig/piper_plus.pc` が install される | `verify_install_layout.cmake` に追加 |
| バージョン | `Version: 1.10.0` | `grep "^Version:" piper_plus.pc` |
| パス置換 | `@CMAKE_INSTALL_PREFIX@` が実際のパスに置換 | `grep "^prefix=" piper_plus.pc` |
| pkg-config 変数 | `${libdir}` が残っていること (CMake に展開されていない) | `grep '${libdir}' piper_plus.pc` |
| Libs 出力 | `-L<path> -lpiper_plus` を含む | `pkg-config --libs piper_plus` |
| Cflags 出力 | `-I<path>` を含む | `pkg-config --cflags piper_plus` |

### 4.2 E2E テスト

```bash
# install 先を PKG_CONFIG_PATH に追加
export PKG_CONFIG_PATH=/tmp/piper-plus-install/lib/pkgconfig:$PKG_CONFIG_PATH

# コンパイル + リンクテスト
cat > /tmp/test_pkgconfig.c << 'EOF'
#include <piper_plus.h>
#include <stdio.h>
int main(void) {
    printf("version: %s\n", piper_plus_version());
    return 0;
}
EOF

gcc $(pkg-config --cflags --libs piper_plus) -o /tmp/test_pkgconfig /tmp/test_pkgconfig.c
LD_LIBRARY_PATH=/tmp/piper-plus-install/lib /tmp/test_pkgconfig

# 辞書パスの取得テスト
DICT_DIR=$(pkg-config --variable=dictdir piper_plus)
test -d "$DICT_DIR"  # 辞書ディレクトリが存在する
```

## 5. 懸念事項とレビュー項目

| 懸念 | 詳細 | 対策 |
|------|------|------|
| 相対パス vs 絶対パス | `prefix` が install 先の絶対パスに固定される | 標準的な挙動。`PKG_CONFIG_SYSROOT_DIR` でクロスコンパイル対応可能 |
| `Libs.private` の ORT パス | ORT が同梱ディレクトリにある場合、静的リンク時に `-L` が必要 | `Libs.private` に `-L${libdir} -lonnxruntime` を含める (同じ lib/ に同梱) |
| multiarch 環境 | Debian の `lib/x86_64-linux-gnu` に install される場合 | `GNUInstallDirs` が自動検出。`@CMAKE_INSTALL_LIBDIR@` で対応済み |

**レビュー項目:**
- [ ] `pkg-config --libs` の出力で実際にコンパイル・リンクが成功する
- [ ] `pkg-config --variable=dictdir` で辞書パスが取得可能
- [ ] Linux / macOS で動作確認
- [ ] `piper_plus.pc` 内のバージョンが `VERSION` ファイル (`1.10.0`) と一致

## 6. 一から作り直すとしたら

1. **`Requires.private: onnxruntime` の追加検討。** ONNX Runtime が将来 pkg-config を提供する可能性がある。その場合は `Requires.private` に追加するだけで、`Libs.private` の `-lonnxruntime` は自動解決される。現時点では ORT が pkg-config を提供していないため `Libs.private` で直接指定。

2. **CMake の `PKG_CONFIG_FILE` ジェネレータ式の利用。** CMake 3.28+ では `install(TARGETS ... PKG_CONFIG)` が提案されているが、まだ安定していない。手動テンプレートが最も堅実。

3. **pkg-config ファイルの自動テスト。** `ctest` から `pkg-config --validate piper_plus.pc` を呼び出すテストを最初から組み込むべきだった。後から追加するより、テンプレート作成と同時が理想。
