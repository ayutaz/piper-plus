## 1. 目的とゴール

`test_token_mapper_impl.py` L16-18 および `test_phonemize.py` L14-16 に、テスト専用のヘルパー関数 `map_sequence()` が定義されている。この関数は 1 行のリスト内包表記であり、独立した関数として存在する意義が薄い。

**現状のコード** (`test_token_mapper_impl.py` L16-18, `test_phonemize.py` L14-16):

```python
def map_sequence(tokens: list[str]) -> list[str]:
    """Map a sequence of IPA tokens through PUA mapping."""
    return [map_token(t) for t in tokens]
```

**使用箇所** (`test_token_mapper_impl.py` L83-120):

```python
def test_map_sequence_basic(self):
    sequence = ["k", "o", "n", "n", "i", "ch", "i", "w", "a"]
    mapped = map_sequence(sequence)
    assert mapped[0] == "k"
    assert mapped[5] == "\ue00e"  # "ch" -> PUA
    # ...

def test_map_sequence_with_multiple_mappings(self):
    sequence = ["ch", "i", "ts", "u", "ky", "o"]
    mapped = map_sequence(sequence)
    assert mapped[0] == "\ue00e"  # ch
    assert mapped[2] == "\ue00f"  # ts
    # ...

def test_map_sequence_empty(self):
    assert map_sequence([]) == []

def test_map_sequence_no_mappings(self):
    sequence = ["a", "i", "u", "e", "o"]
    mapped = map_sequence(sequence)
    assert mapped == sequence
```

**問題点**:
1. `map_sequence()` は `[map_token(t) for t in tokens]` の 1 行ラッパーであり、テストヘルパーとしての抽象化価値がほぼない
2. 同一定義が 2 ファイル (`test_token_mapper_impl.py`, `test_phonemize.py`) に重複している
3. テスト名が `test_map_sequence_*` であり、あたかも本番の `map_sequence` 関数をテストしているように見えるが、実際にはテスト内定義の関数を自己テストしている

**完了の定義**:
- `map_sequence()` のテスト専用定義を削除し、テストケース内でインライン化 (`[map_token(t) for t in sequence]`) する
- テスト名を `test_map_token_on_sequence_*` 等に変更し、`map_token` (本番関数) を直接テストしていることを明示する
