## 1. 目的とゴール

piper-plus の README を訪れた新規ユーザーが、30秒以内に音声合成を体験できる導線を整備する。競合 (Kokoro 4行、Coqui 3行) と同等以下のステップ数で「音が出る」体験を提供し、初回離脱率を下げる。

完了時の状態:
- README.md の目次直後に「30秒で試す」セクションが存在する
- 3パターン (プリビルドバイナリ / Python CLI / npm ブラウザ) のクイックスタートが記載されている
- HF サンプル音声が details タグで折りたたまれて掲載されている
- 日本語 README と英語 README (README_EN.md) の両方が更新されている

### セクション構成

「30秒で試す」セクションを `## 主要機能` の直前、目次の直後に挿入する。

```markdown
## 30秒で試す

### プリビルドバイナリ (推奨)

\```bash
# macOS / Linux — GitHub Releases からダウンロード:
# https://github.com/ayutaz/piper-plus/releases/latest
./piper --download-model tsukuyomi
./piper --model tsukuyomi --text "こんにちは" -f hello.wav
\```

### Python

\```bash
pip install piper-tts-plus
python -m piper_train.infer_onnx --model /path/to/model.onnx --text "こんにちは" --output-dir .
\```

### npm (ブラウザ)

\```js
import { PiperPlus } from "piper-plus";
const piper = await PiperPlus.initialize("tsukuyomi");
const audio = await piper.synthesize("Hello, world!");
audio.play();
\```

</details>
```

### 注意事項
- Python 例は `piper_train.infer_onnx` モジュールを直接実行する形式。高レベル API (`PiperPlus` クラス) は M2-1 で提供予定
- プリビルドバイナリ例は Rust CLI のモデル自動解決 (`--model tsukuyomi`) を活用する
- サンプル音声の URL は M1-5 で HF にアップロードされる wav ファイルを参照するが、M1-5 への依存は持たない (リンク切れは許容し、M1-5 完了後に有効になる)

## 6. 一から作り直すとしたら

- 「30秒で試す」より「10秒で試す」を目指すべきだった。ブラウザデモへのリンク1つで十分かもしれない (WebAssembly デモが既に存在する)
- 3パターン並列ではなく、タブ UI (GitHub では不可) やユーザーの環境を聞く分岐フローのほうが認知負荷が低い。ただし GitHub Markdown の制約上、details タグでの分岐が現実的
- Python の高レベル API がないため CLI 例になっている。M2-1 を先行させれば `from piper_plus import PiperPlus` の3行例が書けた。フェーズの優先順位としてはAPI設計を先にすべきだったかもしれない
