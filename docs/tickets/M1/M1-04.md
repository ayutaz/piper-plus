## 1. 目的とゴール

Hugging Face Collections 機能を使い、piper-plus 関連のモデル・デモ・データセットを一つのコレクションにまとめる。ユーザーが piper-plus のエコシステム全体を一目で把握できるようにし、関連リソースへの回遊率を向上させる。

完了時の状態:
- `ayousanz` アカウントに「piper-plus: Multilingual Neural TTS」コレクションが作成されている
- 全モデル、Spaces デモ、データセットがコレクションに含まれている
- コレクションに説明文が記載されている

## 6. 一から作り直すとしたら

- HF Organization (`piper-plus`) を作成してモデルを移管するほうが、コレクションよりもブランディングとして強い。ただし Organization 作成は別途検討が必要
- コレクション作成を API (`huggingface_hub`) で行い、CI から自動更新する仕組みにすれば、新モデル追加時にコレクションの更新漏れを防げる
- コレクションの説明文に比較表 (vs Kokoro, vs Coqui) を入れるとインパクトがあるが、メンテナンスコストが高い

## 7. 後続タスクへの連絡事項

- コレクション URL を README や各モデルカードから参照する場合は、URL をチームに共有すること
- 新しいモデルやデータセットを HF にアップロードした際は、コレクションへの追加を忘れないこと
