# M1-3 HF 全モデルに pipeline_tag / library_name 追加

**マイルストーン**: M1 (v1.12.0)
**カテゴリ**: モデル配布
**工数**: 1d
**依存**: なし
**ステータス**: 未着手

---

## 1. 目的とゴール

Hugging Face Hub 上の piper-plus 全モデルに `pipeline_tag: text-to-speech` と `library_name: onnxruntime` メタデータを追加する。これにより:
- HF Hub の TTS タスクページ (`huggingface.co/models?pipeline_tag=text-to-speech`) に piper-plus モデルが表示される
- モデルカードに「Text-to-Speech」タスクバッジが付く
- HF の検索・フィルタリングで発見可能になる

完了時の状態:
- 3つの HF モデルリポジトリすべてで `pipeline_tag` と `library_name` が設定されている
- HF Hub の TTS フィルタで piper-plus モデルが表示される

## 2. 実装内容の詳細

### 対象モデル
1. `ayousanz/piper-plus-tsukuyomi-chan`
2. `ayousanz/piper-plus-base`
3. `ayousanz/piper-plus-css10-ja-6lang`

### 変更内容

各モデルの `README.md` (モデルカード) の YAML フロントマターに以下を追加:

```yaml
---
pipeline_tag: text-to-speech
library_name: onnxruntime
language:
  - ja
  - en
  - zh
  - es
  - fr
  - pt
tags:
  - piper
  - piper-plus
  - vits
  - tts
  - onnx
  - multilingual
license: mit
---
```

### 作業手順
1. `huggingface_hub` CLI または Web UI でモデルカードを編集
2. 既存のフロントマターがあればマージ、なければ新規追加
3. `language` タグは各モデルの対応言語に合わせる (tsukuyomi-chan と base は6言語、css10-ja は JA のみだが6lang ベース)

### モデルカード本文の改善
既存のモデルカード本文も合わせて標準化する:
- モデル概要 (アーキテクチャ、パラメータ数、対応言語)
- 使用方法 (Python/Rust/C#/npm の各プラットフォーム)
- ライセンス情報
- 学習データ情報

## 5. 懸念事項とレビュー項目

### 懸念事項
- `library_name: onnxruntime` は HF の公式サポートライブラリ名のリストに含まれているか確認が必要。含まれていない場合は `library_name: piper` などカスタム名を使う
- 既存のモデルカードに YAML フロントマターが既にある場合、フィールドの重複に注意
- `language` タグの粒度: `ja` vs `jpn` (ISO 639-1 vs 639-3)。HF は ISO 639-1 を推奨

### レビューチェックリスト
- [ ] 3モデルすべてで YAML フロントマターが正しく設定されている
- [ ] HF Hub の TTS フィルタページでモデルが表示される
- [ ] `language` タグが各モデルの実際の対応言語と一致している
- [ ] ライセンス表記が MIT で統一されている
- [ ] モデルカード本文が3モデルで一貫したフォーマットになっている

## 6. 一から作り直すとしたら

- モデルカードの管理をリポジトリ内のテンプレート (`docs/templates/hf-model-card.md`) で行い、CI で自動デプロイする仕組みにすべきだった。手動更新は忘れられやすい
- `library_name` は `onnxruntime` よりも `piper-plus` を登録申請してカスタムライブラリとして認知させるほうが長期的にはブランディングに有利。ただし HF の審査が必要
- 3モデルを個別に更新するのではなく、`huggingface_hub` Python ライブラリを使ったスクリプトで一括更新するほうが再現性が高い
