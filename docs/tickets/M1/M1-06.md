**マイルストーン**: M1 (v1.12.0)
**カテゴリ**: モデル配布

**依存**: M1-3
**ステータス**: 未着手

## 1. 目的とゴール

piper-plus のモデルは元祖 Piper (`rhasspy/piper`) および OHF-Voice/piper1-gpl と互換性がない。この事実を全モデルカードに明記し、ユーザーが誤って元祖 Piper ランタイムで piper-plus モデルを使用しようとして混乱することを防ぐ。

完了時の状態:
- 3つの HF モデルカードすべてに非互換性セクションが追加されている
- 非互換の具体的理由 (音素体系、config 形式、ONNX 入力テンソル) が明記されている
- piper-plus ランタイムの入手方法が案内されている

## 2. 実装内容の詳細

### 対象モデル
1. `ayousanz/piper-plus-tsukuyomi-chan`
2. `ayousanz/piper-plus-base`
3. `ayousanz/piper-plus-css10-ja-6lang`

### 追加セクション

各モデルカードの冒頭 (メタデータ直後) に以下の警告セクションを追加:

```markdown
> **Important**: This model is designed for **piper-plus** and is **NOT compatible** with the original [rhasspy/piper](https://github.com/rhasspy/piper) or [OHF-Voice/piper1-gpl](https://github.com/OHF-Voice/piper1-gpl). See [Compatibility](#compatibility) below.

## Compatibility

This model requires **piper-plus** runtime. It is **not compatible** with the original Piper due to:

| Aspect | Original Piper | piper-plus |
|--------|---------------|------------|
| G2P | eSpeak-ng (GPL) | Custom G2P (MIT, 8 languages) |
| Phoneme set | eSpeak IPA | piper-plus phoneme IDs (173 symbols) |
| ONNX inputs | `input`, `input_lengths`, `scales` | `input`, `input_lengths`, `scales`, `sid` (Required: multi-speaker), `lid` (Required: multi-lang), `prosody` (Optional) |
| Config format | `config.json` (eSpeak phonemes) | `config.json` (piper-plus phoneme_id_map) |
| Prosody | N/A | A1/A2/A3 prosody features |
| Multi-language | Single language per model | Up to 6 languages per model |

### How to use

Install piper-plus from one of the supported platforms:

- **Python**: `pip install piper-tts-plus`
- **Rust**: `cargo install piper-plus-cli`
- **C#**: `dotnet tool install -g PiperPlus.Cli`
- **npm**: `npm install piper-plus` (browser only)
- **Go**: `go install github.com/ayutaz/piper-plus/src/go/cmd/piper-plus@latest`

GitHub: https://github.com/ayutaz/piper-plus
```

## 5. 懸念事項とレビュー項目

### 懸念事項
- 非互換性を強調しすぎると、元祖 Piper ユーザーが piper-plus を敬遠する可能性がある。「非互換」よりも「拡張」「進化」としてのポジティブなフレーミングが望ましい
- ONNX 入力テンソルの名前は将来変更される可能性がある。バージョニングの仕組みが必要
- `prosody` テンソルはオプショナルだが、テーブルでは必須入力のように見える

## 6. 一から作り直すとしたら

- モデルカードではなく、リポジトリのドキュメント (`docs/compatibility.md`) に詳細な非互換性ドキュメントを作成し、モデルカードからはリンクするだけにしたほうがメンテナンスしやすい
- ONNX モデル自体にメタデータ (custom_metadata_map) として `runtime: piper-plus` を埋め込めば、ランタイムが自動判別できる。モデルカードの注記に頼らない仕組みが望ましい
- 元祖 Piper モデルを piper-plus 形式に変換するツール (`scripts/convert_piper_model.py`) を提供すれば、非互換性の問題自体が緩和される
