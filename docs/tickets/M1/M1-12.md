## 1. 目的とゴール

現在 ghcr.io のみで配布している piper-plus API の Docker イメージを Docker Hub でも公開する。Docker Hub は最も一般的なレジストリであり、`docker pull` のデフォルトレジストリでもあるため、ユーザーの導入ハードルを下げる。

完了時の状態:
- GitHub Actions workflow が追加され、リリースタグ push 時に Docker Hub へ自動公開される
- `docker pull ayousanz/piper-plus-api:latest` でイメージを取得できる
- ghcr.io と Docker Hub の両方にイメージが公開される (デュアルパブリッシュ)

## 2. 実装内容の詳細

### 対象ファイル
- `.github/workflows/docker-publish.yml` (新規作成)
- 既存の `.github/workflows/docker-build.yml` への追記も検討

### GitHub Actions Workflow

```yaml
# .github/workflows/docker-publish.yml
name: Docker Hub Publish

on:
  push:
    tags:
      - 'v*'
  workflow_dispatch:

env:
  DOCKER_IMAGE: ayousanz/piper-plus-api

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: |
            ${{ env.DOCKER_IMAGE }}
            ghcr.io/${{ github.repository_owner }}/piper-plus-api
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=raw,value=latest,enable={{is_default_branch}}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          file: docker/python-inference/Dockerfile
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
          platforms: linux/amd64,linux/arm64
```

### リポジトリ Secrets の設定

以下の Secrets をリポジトリ設定に追加する必要がある:

| Secret 名 | 説明 |
|-----------|------|
| `DOCKERHUB_USERNAME` | Docker Hub ユーザー名 |
| `DOCKERHUB_TOKEN` | Docker Hub アクセストークン (Read/Write) |

### Docker Hub リポジトリ設定

1. Docker Hub で `ayousanz/piper-plus-api` リポジトリを作成
2. Description: "piper-plus: Multilingual Neural TTS API (OpenAI compatible)"
3. README (Docker Hub 上の Overview) を設定

### タグ戦略

| タグ | 例 | 説明 |
|-----|-----|------|
| `latest` | `ayousanz/piper-plus-api:latest` | dev ブランチの最新 |
| `v{version}` | `ayousanz/piper-plus-api:v1.12.0` | セマンティックバージョン |
| `v{major}.{minor}` | `ayousanz/piper-plus-api:v1.12` | マイナーバージョン |

### マルチアーキテクチャ

`linux/amd64` と `linux/arm64` のマルチプラットフォームイメージをビルドする。Apple Silicon (M1/M2/M3) ユーザーがネイティブ速度で利用できる。

## 5. 懸念事項とレビュー項目

### 懸念事項
- Docker Hub の無料プランには rate limit がある (匿名: 100 pulls/6h、認証済み: 200 pulls/6h)。piper-plus の利用規模では問題ないが、CI からの pull で制限にかかる可能性
- マルチアーキテクチャビルド (`linux/arm64`) は QEMU エミュレーションが必要で、ビルド時間が大幅に増加する (20-40分)。初回は `linux/amd64` のみにして、arm64 は後追いでもよい
- Docker Hub の Secrets 管理: `DOCKERHUB_TOKEN` はアクセストークンを使い、パスワード直接入力は避ける
- 既存の `docker-build.yml` との機能重複に注意。統合するか分離するか判断が必要

## 6. 一から作り直すとしたら

- ghcr.io と Docker Hub のデュアルパブリッシュではなく、Docker Hub 一本に統一するほうがシンプル。ただし GitHub Actions の `packages: write` 権限で ghcr.io は自然に使えるため、デュアルパブリッシュのコストは低い
- Dockerfile 自体を最適化 (マルチステージビルド、.dockerignore の整備) してからイメージ公開すべきだった。大きなイメージを公開すると pull 時間が長く、第一印象が悪い
- Docker Hub の代わりに (または加えて) Amazon ECR Public や Artifact Registry も選択肢。ただし Docker Hub の認知度が圧倒的に高い
- イメージにモデルを同梱するオプション (`piper-plus-api:latest-with-model`) を用意すれば、`docker run` 一発で音が出る体験を提供できる
