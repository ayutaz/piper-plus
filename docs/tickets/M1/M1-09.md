## AnythingLLM

AnythingLLM は TTS プロバイダーとして OpenAI 互換エンドポイントをサポートしています。

### 設定手順
1. AnythingLLM の Settings > Text-to-Speech を開く
2. Provider: "OpenAI Compatible" を選択
3. API Base URL: `http://localhost:8000/v1`
4. Model: `tsukuyomi` (利用可能なモデルは `/v1/models` で確認)

## LangChain

### openai ライブラリ経由 (推奨)

\```python
from openai import OpenAI

client = OpenAI(
    api_key="dummy",  # piper-plus API は認証不要
    base_url="http://localhost:8000/v1",
)
response = client.audio.speech.create(
    model="tsukuyomi",
    voice="default",
    input="こんにちは",
)
response.stream_to_file("hello.wav")
\```

### LangChain ネイティブ (要確認)

> **Note**: `langchain_community.tools.OpenAITTS` クラスの存在は未確認です。LangChain のバージョンによっては利用できない場合があります。利用可能かどうかを事前に確認してください。

\```python
# 要確認: OpenAITTS の存在確認が必要
from langchain_community.tools import OpenAITTS

tts = OpenAITTS(
    api_key="dummy",
    base_url="http://localhost:8000/v1",
    model="tsukuyomi",
)
audio = tts.invoke("こんにちは")
\```

### OpenAI 互換 API の位置づけ

全ガイドで共通するパターンは「OpenAI 互換 API 経由」であることを強調。`docker/python-inference/inference.py` が提供する以下のエンドポイントが鍵:

| エンドポイント | メソッド | 用途 |
|-------------|--------|------|
| `/v1/audio/speech` | POST | 音声合成 |
| `/v1/models` | GET | モデル一覧 |
| `/v1/audio/speech/languages` | GET | 対応言語一覧 |
| `/health` | GET | ヘルスチェック |

## 6. 一から作り直すとしたら

- 各ツールの統合ガイドを個別ファイル (`docs/guides/anythingllm.md`, `docs/guides/langchain.md`) にするか、1つのファイルにまとめるか。1ファイルのほうがユーザーは全体像を把握しやすいが、長くなると読みにくい
- コード例よりも、実際の接続を示す短い動画 (GIF) のほうがインパクトがある。ただし GIF の作成・メンテコストが高い
- OpenAI 互換 API ではなく、各ツールのプラグイン/拡張を作成するほうが UX は良いが、メンテナンスコストが非常に高い
- LangChain は API 変更が頻繁なため、バージョンをピン留めしたコード例にすべき
