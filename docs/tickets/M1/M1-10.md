## 1. 目的とゴール

Ollama (LLM 推論) と piper-plus (TTS) を組み合わせた docker-compose 構成を提供する。`docker compose up` 一発で「LLM テキスト生成 + 音声合成」のパイプラインが動く環境を構築し、LLM 開発者が piper-plus を簡単に TTS バックエンドとして採用できるようにする。

完了時の状態:
- `docker/ollama-stack/docker-compose.yml` が作成されている
- `docker compose up` で Ollama + piper-plus API が起動する
- サンプルスクリプトで LLM 生成テキストの音声合成が動作する

## 2. 実装内容の詳細

### ファイル構成

```
docker/ollama-stack/
  docker-compose.yml
  README.md
  example.py          # 使用例スクリプト
  .env.example         # 環境変数テンプレート
```

### docker-compose.yml

```yaml
services:
  ollama:
    image: ollama/ollama
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    restart: unless-stopped

  piper-api:
    build:
      context: ../..
      dockerfile: docker/python-inference/Dockerfile
    ports:
      - "8000:8000"
    volumes:
      - ./models:/app/models:ro
    environment:
      - PIPER_DEFAULT_MODEL=/app/models/tsukuyomi-6lang-v2-fixed.onnx
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 10s
      timeout: 5s
      retries: 3

volumes:
  ollama_data:
```

### example.py

```python
#!/usr/bin/env python3
"""Ollama + piper-plus 統合デモ

Usage:
    python example.py "日本の首都について教えてください"
"""
import sys
import requests

def generate_text(prompt: str) -> str:
    """Ollama API でテキスト生成"""
    resp = requests.post(
        "http://localhost:11434/api/generate",
        json={"model": "llama3.2", "prompt": prompt, "stream": False},
    )
    return resp.json()["response"]

def synthesize_speech(text: str, output_path: str = "output.wav") -> None:
    """piper-plus API で音声合成"""
    resp = requests.post(
        "http://localhost:8000/v1/audio/speech",
        json={"input": text, "model": "tsukuyomi", "voice": "default"},
    )
    with open(output_path, "wb") as f:
        f.write(resp.content)
    print(f"Audio saved to {output_path}")

if __name__ == "__main__":
    prompt = sys.argv[1] if len(sys.argv) > 1 else "Hello, tell me about yourself."
    print(f"Generating text for: {prompt}")
    text = generate_text(prompt)
    print(f"Generated: {text[:100]}...")
    synthesize_speech(text)
```

## 5. 懸念事項とレビュー項目

### 懸念事項
- Ollama の Docker イメージは大きい (~1GB)。初回起動に時間がかかる
- piper-plus API の Docker イメージビルドに既存の `docker/python-inference/Dockerfile` を使うが、ビルドコンテキストがリポジトリルートになる必要がある (下記「実装上の注意点」参照)
- Ollama のモデルダウンロードは `docker compose up` の後に別途必要。これは UX として不便だが、モデルの自動ダウンロードは compose の範囲外
- GPU 対応 (NVIDIA) は `deploy.resources.reservations.devices` で設定可能だが、デフォルトでは CPU のみとする

### レビューチェックリスト
- [ ] `docker compose up -d` がエラーなく起動する
- [ ] piper-plus API の healthcheck が正しく動作する
- [ ] ポートの衝突がないこと (11434, 8000)
- [ ] volumes のパスが正しいこと
- [ ] example.py が API エンドポイントを正しく呼んでいること
- [ ] README の手順が正確で再現可能であること
- [ ] `.env.example` に機密情報が含まれていないこと

### 実装上の注意点
- `piper-api` サービスの `build.context` はリポジトリルート (`../..`) を指定する必要がある。`docker/python-inference/Dockerfile` が `src/python/` などルート直下のディレクトリを COPY するため。`docker compose up` を `docker/ollama-stack/` から実行する場合、ビルドコンテキストの相対パスに注意すること

## 6. 一から作り直すとしたら

- Ollama だけでなく、vLLM / llama.cpp / LocalAI も含めた複数の LLM バックエンドに対応する汎用テンプレートにすべきだった
- docker-compose ではなく Kubernetes (Helm chart) も提供すれば、プロダクション利用のハードルが下がる
- `example.py` ではなく、ストリーミング対応の統合スクリプト (LLM トークン生成 -> 文単位で TTS -> 逐次再生) のほうがデモとしてインパクトが大きい
- piper-plus API の Docker イメージを事前ビルドして公開 (M1-12) すれば、`build:` ではなく `image:` で指定でき、起動時間が大幅に短縮される
