# v1.12 → v1.13 migration guide

Issue [#527](https://github.com/ayutaz/piper-plus/issues/527) の Python 3.13 + CUDA 12.8 + Ubuntu 24.04 統一に伴う **breaking change の対応手順**。 v1.12 から v1.13 にアップグレードするユーザー向け。

設計詳細は [`docs/reference/python-313/`](../reference/python-313/) の 5 文書 (要求定義 / 要件定義 / マイルストーン / 未決事項 / 実装計画) を参照。

---

## Docker base image upgrade

### 影響

- `docker/python-train/Dockerfile` と `docker/python-inference/Dockerfile` の base image が:
  - `nvidia/cuda:12.6.3-cudnn-*-ubuntu22.04` → `nvidia/cuda:12.8.1-cudnn-*-ubuntu24.04`
  - Ubuntu 22.04 (Jammy、 EOL 2027-04) → Ubuntu 24.04 (Noble、 EOL 2029-04)
  - CUDA 12.6.3 → CUDA 12.8.1 (Blackwell sm_120 正式対応)

### 必要な対応

**host 側 (Docker 実行環境):**

1. **NVIDIA driver を R570 以上にアップグレード**:

   ```sh
   nvidia-smi  # "Driver Version: 570.x" 以上であることを確認
   ```

   R525 (CUDA 12.6 相当) 以下の host では新 image が起動失敗します。 driver bump はサーバー管理者に依頼してください。

2. **`nvidia-container-toolkit` を 1.14 以上にアップグレード**:

   ```sh
   nvidia-container-cli --version  # 1.14+ を確認
   ```

   1.10 以下では CUDA 12.8 image を `--gpus all` で実行する際に device mount が失敗するケースがあります。

3. **`docker run` で `--gpus all` が動くことを確認**:

   ```sh
   docker run --rm --gpus all nvidia/cuda:12.8.1-base-ubuntu24.04 nvidia-smi
   ```

### Fallback (v1.12 image を継続利用)

driver を上げられない場合は **v1.12 Docker image tag を継続利用** してください。 v1.12 系の tag は registry で長期保持されます (Issue #527 / DR-006 / OQ-14 で確定)。

```sh
# 旧 image 利用例
docker pull <registry>/piper-plus-gpu:1.12.0
docker pull <registry>/piper-plus-cpu:1.12.0
```

## Python 3.13 default

### 影響

- Docker 全 image の Python interpreter が 3.11 → 3.13 に統一
  - CPU image: `python:3.11-slim-bookworm` → `python:3.13-slim-trixie`
  - distroless: `distroless/python3-debian12` (= 3.11) → `distroless/python3-debian13` (= 3.13)
  - CUDA image: apt `python3.11` → deadsnakes PPA `python3.13`
- CI workflow 20 個の default `python-version` が `'3.11'` → `'3.13'` に
- ドキュメント (`CONTRIBUTING.md` / `README*.md` 等) の Python 推奨 version が 3.11+ → 3.13+ に

### 互換性

**`requires-python = ">=3.11"` は据置** (Issue #527 / NFR-01 / C-01)。 PyPI 経由の `pip install piper-plus` は Python 3.11 / 3.12 / 3.13 すべてで動作継続します。

### 必要な対応 (3rd party 利用者)

- Docker image を使う場合: host driver bump (上記 [Docker base image upgrade](#docker-base-image-upgrade)) のみ
- PyPI 経由: 対応不要 (3.11/3.12 サポート維持)
- ローカル開発環境: Python 3.13 への切替を推奨 (`uv python install 3.13`)

## PyTorch upgrade

### 影響

- `docker/python-train/Dockerfile` の torch wheel が:
  - `torch==2.2.1+cu121` → `torch==2.11.0+cu128`
  - `torchaudio==2.2.1+cu121` → `torchaudio==2.11.0+cu128`
  - **`torchvision==0.17.1+cu121` は削除** (piper-train が未使用、 FR-02-05)
- `pyproject.toml` の floor pin: `torch>=2.11.0` (DR-002 と整合)

### 互換性

**新 GPU 起動の必須条件** (Issue #527 / DR-001):

- RTX 5090 (Blackwell sm_120) は CUDA 12.8 wheel (cu128) が必須、 cu121 wheel では起動不可
- Ada 6000 (sm_89) / T4 (sm_75) は cu121 でも動くが、 cu128 で Tensor Core / cuDNN 9.5+ の恩恵

### 必要な対応

ローカルで torch を直接 install している場合:

```sh
# 新環境
uv pip install --index-url https://download.pytorch.org/whl/cu128 \
    "torch==2.11.0+cu128" "torchaudio==2.11.0+cu128"
```

## Checkpoint resume non-support

### 影響 (重要)

**v1.13 では torch 2.2 系で生成された旧 ckpt の resume を保証しません** (Issue #527 / DR-006):

- `--resume-from-multispeaker-checkpoint /data/piper/output-multilingual-6lang/.../*.ckpt` (torch 2.2 製) は **未サポート**
- PyTorch upstream の互換ポリシー: model weights は forward 互換とされるが、 optimizer state_dict と internal key 構造に変更があり、 piper-train で resume 失敗する可能性
- 既存 ONNX モデル (HuggingFace 配布の `piper-plus-base` 等) の **推論は forward 互換で動作継続**

### 必要な対応

#### 推論ユーザー (大多数)

**対応不要**。 既存 ONNX モデルは v1.13 ランタイム / Docker image で動作します。

#### 学習ユーザーで旧 ckpt から継続学習が必要な場合

選択肢:

**A. v1.12 Docker image で継続学習** (推奨):

```sh
# v1.12 image を使い続ける
docker run --gpus all <registry>/piper-plus-gpu:1.12.0 \
    python -m piper_train --resume-from-multispeaker-checkpoint <old_ckpt> ...
```

v1.12 image は registry で長期保持されます (OQ-14)。

**B. v1.13 で from scratch 再学習**:

```sh
# 新規 base 学習 (Template A、 75 epoch 規模)
python -m piper_train \
    --dataset-dir <DATASET_DIR> \
    --accelerator gpu --devices 4 --precision bf16-mixed \
    --max_epochs 75 --batch-size 32 \
    ...
```

**C. v1.13 で torch 2.11 製 base ckpt から FT** (M5 リリース後に新 base を公開予定):

公開後は `--resume-from-multispeaker-checkpoint <new_base.ckpt>` で v1.13 環境 FT 可能。

## distroless image upgrade

### 影響

- `docker/python-inference/Dockerfile.cpu.distroless` と `docker/webui/Dockerfile.distroless` で:
  - builder: `python:3.11-slim-{bookworm,trixie}` → `python:3.13-slim-trixie`
  - final: `gcr.io/distroless/python3-debian12` → `gcr.io/distroless/python3-debian13`
  - 内部パス: `/usr/local/lib/python3.11/site-packages` → `/usr/local/lib/python3.13/site-packages`
  - `PYTHONPATH` 環境変数も同様に書換

### 必要な対応 (3rd party がこれらを extends している場合)

`FROM <piper-plus-distroless>:1.12.0` を使っていてカスタム overlay を追加している場合、 内部パス参照を 3.13 に追従:

```dockerfile
# Before (v1.12)
COPY my-overlay/ /usr/local/lib/python3.11/site-packages/

# After (v1.13)
COPY my-overlay/ /usr/local/lib/python3.13/site-packages/
```

## TF32 default ON

### 影響

`src/python/piper_train/__main__.py` の `main()` で:

```python
torch.backends.cudnn.benchmark = True
# v1.13 で追加 (DR-007)
torch.backends.cuda.matmul.allow_tf32 = True
torch.backends.cudnn.allow_tf32 = True
```

- Ada/Blackwell (sm_80+): TF32 Tensor Core で matmul/conv ~1.3-1.5x 高速化
- T4 (sm_75) 以下: noop (warning なし)
- 数値精度: matmul mantissa 23-bit → 10-bit、 perceptual 影響は audio_parity Tier 4 (SNR ≥ 30dB) で検証済

### 必要な対応

通常は対応不要。

`torch.use_deterministic_algorithms(True)` を併用する research 用途は TF32 と非互換のため、 必要に応じて以下を `__main__.py` 修正で OFF にしてください (将来的に opt-out CLI flag 追加検討):

```python
torch.backends.cuda.matmul.allow_tf32 = False
torch.backends.cudnn.allow_tf32 = False
```

## bf16-mixed Template default

### 影響

`CLAUDE.md` の Template A (multi-speaker pretraining) と Template B (single-speaker FT) の `--precision` default:

- Before: `--precision 32-true` (V100 想定、 V100 で FP16-mixed は backward 遅延あり)
- After: `--precision bf16-mixed` (Ada/Blackwell で BF16 native Tensor Core 活用)

加えて `--no-wavlm` も Template から削除 (Ada/Blackwell で WavLM 有効が canonical):

```diff
- --accelerator gpu --devices 1 --precision 32-true \
+ --accelerator gpu --devices 1 --precision bf16-mixed \
- --ema-decay 0.9995 --max-phoneme-ids 400 --no-wavlm \
+ --ema-decay 0.9995 --max-phoneme-ids 400 \
```

### 必要な対応

#### 旧 V100 環境を継続利用する場合

`--precision 32-true` を明示指定:

```sh
python -m piper_train \
    --accelerator gpu --devices 4 --precision 32-true \
    --no-wavlm \
    ...
```

#### T4 (VRAM 16GB) で学習する場合

T4 は学習向きではない (VRAM 制約) が、 やる場合は:

```sh
--precision 16-mixed \  # T4 は sm_75 で BF16 native 非対応
--no-wavlm \            # VRAM 節約
--batch-size 8 \        # 16GB 上限内
```

#### Ada 6000 / RTX 5090 で学習する場合

Template の default をそのまま使ってください:

```sh
--precision bf16-mixed \
--batch-size 32 \  # Ada 48GB / RTX 5090 32GB で快適
```
