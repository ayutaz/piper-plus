# Phase 3: ONNX Runtime統合と音声合成実装

作成日: 2025-07-21  
ステータス: **計画中**

## 概要

Phase 3では、ONNX Runtime Webを統合し、実際の音声合成機能を実装します。Phase 2で完成した日本語テキスト→音素列変換パイプラインに、ニューラルネットワークベースの音声合成を追加し、ブラウザ上で完全なTTSシステムを実現します。

## 目標

1. **ONNX Runtime Web統合**
   - onnxruntime-webのセットアップ
   - WebGPU/WebGL/WASMバックエンドの設定
   - メモリ効率的な推論実装

2. **Piperモデル統合**
   - ONNX形式の音声合成モデルロード
   - 音素列から音声波形生成
   - リアルタイムファクター > 10x

3. **音声出力実装**
   - Web Audio APIとの統合
   - AudioWorkletによる低遅延再生
   - ストリーミング対応

## 技術的考慮事項

```javascript
// 推奨設定
const sessionOptions = {
    executionProviders: [
        {
            name: 'webgpu',
            deviceType: 'gpu',
            powerPreference: 'high-performance'
        },
        {
            name: 'webgl',
            contextId: 'webgl2'
        },
        {
            name: 'wasm',
            simd: true
        }
    ],
    graphOptimizationLevel: 'all',
    enableCpuMemArena: true,
    enableMemPattern: true
};
```

### メモリ管理戦略

1. **モデルサイズ**
   - 日本語モデル: 約30-50MB
   - 量子化による削減検討
   - 段階的ロード実装

2. **推論メモリ**
   - 入力バッファ: ~1KB/秒
   - 出力バッファ: ~88KB/秒（22kHz）
   - 中間バッファ: モデル依存

3. **Unity WebGL制約**
   - 256MBヒープ制限
   - SharedArrayBuffer使用不可
   - Worker通信のオーバーヘッド

### パフォーマンス目標

| 指標 | 目標値 | 優先度 |
|------|--------|--------|
| モデルロード時間 | < 3秒 | 高 |
| 初回推論時間 | < 500ms | 高 |
| RTF (Real-time Factor) | > 10x | 高 |
| メモリ使用量 | < 150MB | 中 |
| 遅延（レイテンシ） | < 300ms | 中 |

## リスクと対策

### 1. WebGPU非対応ブラウザ
**リスク**: Safari、古いブラウザでパフォーマンス低下  
**対策**: WebGL/WASMフォールバック実装

### 2. モデルサイズ
**リスク**: 初回ロード時間が長い  
**対策**: 
- モデル量子化（int8）
- CDN配信
- プログレッシブロード

### 3. メモリ不足
**リスク**: Unity WebGLで256MB制限超過  
**対策**:
- ストリーミング処理
- メモリプール再利用
- 不要データの即時解放
