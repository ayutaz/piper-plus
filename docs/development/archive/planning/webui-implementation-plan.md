# WebUI Implementation Plan for Piper TTS

## 概要
Issue #114に基づき、Gradioを使用したローカルWebUIを実装し、学習・推論環境を構築する。

## 現状分析

### 既存のWeb実装
1. **Flask HTTP Server** (`src/python_run/piper/http_server.py`)
   - シンプルなHTTP API
   - WAVファイル直接返却
   - 基本的なTTSパラメータサポート

2. **FastAPI Server** (`docker/python-inference/inference.py`)
   - モダンな非同期API
   - 自動ドキュメント生成
   - Dockerコンテナで利用

### 技術選定
**Gradio**を選定する理由：
- Issue #114で明示的に指定
- 機械学習向けに最適化されたUI
- 音声ファイルの再生・ダウンロード機能内蔵
- リアルタイムストリーミング対応
- 最小限のコードで高機能UI実装可能

## 実装計画

### フェーズ1: 基本的な推論WebUI（優先度: 高）

#### 1.1 ファイル構成
```
src/python_run/piper/
├── webui/
│   ├── __init__.py
│   ├── app.py          # Gradioアプリケーション本体
│   ├── utils.py        # ユーティリティ関数
│   └── components/     # UIコンポーネント
│       ├── __init__.py
│       ├── inference.py # 推論タブ
│       └── settings.py  # 設定タブ
```

#### 1.2 基本機能
1. **テキスト入力と音声生成**
   - テキストエリア（複数行対応）
   - 生成ボタン
   - 音声プレイヤー（再生・ダウンロード）

2. **モデル選択**
   - ドロップダウンでモデル選択
   - モデル自動検出（data_dirスキャン）
   - モデル情報表示（言語、話者数等）

3. **パラメータ調整**
   - Speaker ID（マルチスピーカー対応）
   - Length Scale（速度調整）
   - Noise Scale（音質調整）
   - Noise W（音韻幅）
   - Sentence Silence（文間の無音）

4. **バッチ処理**
   - CSVファイルアップロード
   - 複数テキストの一括処理
   - ZIPダウンロード

### フェーズ2: 学習WebUI（優先度: 中）

#### 2.1 データセット管理
1. **データアップロード**
   - 音声ファイル（WAV/MP3）アップロード
   - テキストファイルアップロード
   - データセット構造の自動検証

2. **データ前処理**
   - 音声正規化
   - テキストクリーニング
   - トレーニング/検証分割

#### 2.2 学習設定
1. **モデル設定**
   - ベースモデル選択
   - 話者数設定
   - 品質設定（low/medium/high）

2. **学習パラメータ**
   - バッチサイズ
   - 学習率
   - エポック数
   - チェックポイント保存間隔

3. **学習モニタリング**
   - リアルタイムロス表示（Gradio Plot）
   - 学習進捗バー
   - 検証音声の定期生成
   - TensorBoard統合リンク

### フェーズ3: 高度な機能（優先度: 低）

1. **ファインチューニング**
   - 既存モデルからの継続学習
   - 話者追加機能
   - ドメイン適応

2. **音声編集**
   - 音素レベルの編集
   - プロソディ調整
   - 感情パラメータ

3. **API統合**
   - WebUI経由のREST API
   - WebSocket対応（リアルタイム）
   - 認証・認可機能

## 技術仕様

### 依存関係
```txt
gradio>=4.0.0
numpy>=1.19.0
soundfile>=0.10.3
pandas>=1.3.0  # バッチ処理用
```

### 起動方法
```bash
# 推論のみ
python -m piper.webui --mode inference --model path/to/model.onnx

# 学習・推論両方
python -m piper.webui --mode full --data-dir ./data

# Dockerコンテナ
docker run -p 7860:7860 -v $(pwd)/models:/models piper-webui
```

### セキュリティ考慮事項
1. ファイルアップロード制限（サイズ、形式）
2. 実行時のリソース制限
3. ローカル実行を前提（認証なし）
4. CORS設定（必要に応じて）

## 実装優先順位

1. **必須機能**（フェーズ1）
   - 基本的な推論UI
   - モデル選択とパラメータ調整
   - 音声再生・ダウンロード

2. **重要機能**（フェーズ2の一部）
   - バッチ処理
   - 基本的な学習UI
   - 進捗モニタリング

3. **オプション機能**（フェーズ3）
   - 高度な編集機能
   - API統合
   - 認証機能

## リスクと対策

### 技術的リスク
1. **メモリ使用量**
   - 対策: ストリーミング処理、バッチサイズ制限

2. **長時間学習の管理**
   - 対策: バックグラウンドジョブ、定期保存

3. **ブラウザ互換性**
   - 対策: Gradioのビルトイン機能に依存

### 運用リスク
1. **データ損失**
   - 対策: 自動バックアップ、確認ダイアログ

2. **不正な入力**
   - 対策: 入力検証、サニタイゼーション

## 成功指標

1. **ユーザビリティ**
   - 5クリック以内で音声生成可能
   - 直感的なUI（説明書不要）

2. **パフォーマンス**
   - 1秒以内のUI応答
   - リアルタイム音声再生

3. **安定性**
   - 24時間連続稼働
   - エラーからの自動回復
