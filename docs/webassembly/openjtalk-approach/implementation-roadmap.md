## リスクと対策

### 🚨 高リスク項目

1. **辞書サイズ（103MB）**
   - 対策: 段階的圧縮アプローチ
   - Plan B: 最小辞書（10MB）から開始

2. **初期化時間**
   - 対策: WebAssembly Streaming + 非同期ロード
   - Plan B: プログレス表示でUX改善

3. **メモリ制限**
   - 対策: 積極的なメモリ管理
   - Plan B: 機能制限版の提供

### ⚠️ 中リスク項目

1. **ブラウザ互換性**
   - 対策: Chrome/Edge優先開発
   - Plan B: ポリフィル実装

2. **性能問題**
   - 対策: 早期プロファイリング
   - Plan B: Web Worker必須化
