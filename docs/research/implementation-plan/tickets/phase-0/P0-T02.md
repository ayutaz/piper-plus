## 1. タスク目的とゴール

### 1.1 目的

P0-T01 でロードした `facebook/pe-av-small` モデルに対し、16kHz・3 秒の dummy audio を入力して embedding を抽出する。これにより以下を確定する:

- 入力 shape (`[B, T]` か `[B, 1, T]` か)
- 入力 sample rate (16kHz 固定の想定を実検証)
- 出力 embedding 次元 (`peav-style-conditioning.md` の推測では 256 または 512)
- 使用 API 名 (`get_audio_embeds()` / `forward()` / `encode_audio()` のいずれか)
- L2 normalize 前後の norm 値 (Phase 4 の PE-A loss で使うベクトル正規化仕様の確認)

この情報は Phase 1 の `--style-vector-dim` デフォルト値決定と、Phase 4 の PE-A loss input preprocessing 実装に直接必要となる。

### 1.2 ゴール (Definition of Done)

- [ ] `src/python/piper_train/tools/test_pe_av_small.py` に `test_inference(model)` 関数を追加
- [ ] 16kHz × 3 秒の `torch.randn` dummy audio で推論が例外なく完了
- [ ] 使用した API 名 (`get_audio_embeds` / `forward` / その他) がログに出力
- [ ] 入力 audio の shape、出力 embedding の shape と `size(-1)` がログに出力
- [ ] L2 normalize 前の norm と正規化後の norm (期待値 1.0) がログに出力
- [ ] 入力 shape 2 候補 (`[1, 48000]` と `[1, 1, 48000]`) を順番に試行し、どちらが受理されたかを記録

## 2. 実装内容の詳細

### 2.1 対象ファイル

- `src/python/piper_train/tools/test_pe_av_small.py` (P0-T01 で作成済み、`test_inference` を追記)

### 2.2 実装手順

1. `test_inference(model)` を定義し、`device = torch.device("cuda" if torch.cuda.is_available() else "cpu")` で CPU/GPU 両対応
2. `model = model.to(device).eval()` で評価モードへ
3. dummy audio を `torch.randn(1, 16000 * 3, dtype=torch.float32, device=device)` で生成 (shape=[1, 48000])
4. 推論 API を試行する順序:
    1. `model.get_audio_embeds(audio_2d)` (shape `[B, T]`)
    2. 失敗時 `model.get_audio_embeds(audio_2d.unsqueeze(1))` (shape `[B, 1, T]`)
    3. いずれも `AttributeError` なら `model(audio_2d)` の `forward()` にフォールバックし、戻り値が dict なら `output.get("audio_embeds")` / `output.get("embeddings")` / `output.get("last_hidden_state")` の順でチェック
5. 取得した embedding に対して:
    - `embeddings.shape` を記録
    - `embeddings.size(-1)` を embedding dim として出力
    - `torch.norm(embeddings, dim=-1).item()` を normalize 前 norm として出力
    - `torch.nn.functional.normalize(embeddings, dim=-1)` を通した後の norm を出力 (1.0 に近いことを確認)
6. `(embeddings, method_name)` のタプルを返す (T03 でベンチマークに利用)

### 2.3 コード例 (抜粋)

```python
import torch
import torch.nn.functional as F

_METHOD_CANDIDATES = [
    ("get_audio_embeds_2d", lambda m, a: m.get_audio_embeds(a)),
    ("get_audio_embeds_3d", lambda m, a: m.get_audio_embeds(a.unsqueeze(1))),
    ("forward_2d", lambda m, a: m(a)),
    ("forward_3d", lambda m, a: m(a.unsqueeze(1))),
]


def _extract_embedding(output):
    if torch.is_tensor(output):
        return output
    if isinstance(output, dict):
        for key in ("audio_embeds", "embeddings", "last_hidden_state"):
            if key in output:
                return output[key]
    if hasattr(output, "audio_embeds"):
        return output.audio_embeds
    if hasattr(output, "last_hidden_state"):
        return output.last_hidden_state
    raise RuntimeError(f"Cannot extract embedding from output type={type(output).__name__}")


def test_inference(model: torch.nn.Module):
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model = model.to(device).eval()

    sample_rate = 16000
    duration_s = 3
    audio = torch.randn(1, sample_rate * duration_s, dtype=torch.float32, device=device)
    _LOGGER.info("Input audio shape: %s, sr=%d", tuple(audio.shape), sample_rate)

    embeddings = None
    method = None
    with torch.no_grad():
        for name, fn in _METHOD_CANDIDATES:
            try:
                output = fn(model, audio)
                embeddings = _extract_embedding(output)
                method = name
                break
            except (AttributeError, TypeError, RuntimeError) as err:
                _LOGGER.debug("Method %s failed: %s", name, err)

    if embeddings is None:
        raise RuntimeError("All PE-A inference method candidates failed")

    _LOGGER.info("Method used: %s", method)
    _LOGGER.info("Embedding shape: %s", tuple(embeddings.shape))
    _LOGGER.info("Embedding dim (size(-1)): %d", embeddings.size(-1))

    norm_before = torch.norm(embeddings, dim=-1).mean().item()
    embeddings_n = F.normalize(embeddings, dim=-1)
    norm_after = torch.norm(embeddings_n, dim=-1).mean().item()
    _LOGGER.info("L2 norm (before): %.4f / (after): %.4f", norm_before, norm_after)

    return embeddings, method
```

## 6. 懸念事項とレビュー項目

### 6.1 懸念事項

- モデルが Processor (e.g. `AutoProcessor.from_pretrained(...)`) による前処理を必要とする可能性。その場合、生波形をそのまま入れても RuntimeError になる
    - 対策: 4 候補すべて失敗した場合は、`AutoProcessor` を追加で試す節を T03 実装時に検討
- `torch.randn` はガウスノイズであり、実音声ではないため、embedding が縮退 (ゼロに近い) したり極端な値になる可能性あり。L2 norm が極端 (例: 0.01 未満 / 1e4 以上) の場合は Explorer に実音声サンプル (例: `torchaudio` のテスト音声) 入力を検討させる
- GPU 上で `torch.randn` 生成後に forward で dtype mismatch が出る可能性 (fp16 モデルに fp32 入力)

### 6.2 レビュー項目

- [ ] `get_audio_embeds()` と `forward()` の両方を試し、正しい API を確定していること
- [ ] 入力 shape 2D (`[B, T]`) と 3D (`[B, 1, T]`) を両方試しており、どちらが受理されたか明示されていること
- [ ] `torch.nn.functional.normalize` の結果が 1.0 (±0.0001) に収まっていること
- [ ] 推論時に `with torch.no_grad():` が使われており、Autograd graph が構築されていないこと

## 7. 一から作り直すとしたら

### 7.1 代替案の検討

- **代替案 A**: `facebook/pe-av-small` ではなく、既に piper-plus で使用実績のある `wav2vec2-large-xlsr` や `WavLM-Base+` で代用
    - 利点: 既存実装の流用、社内ノウハウ活用
    - 欠点: 知覚感情空間が最適化されておらず、fork の PE-A loss (direction + centroid + margin) と等価にならない。感情分類精度 65% 目標 (`README.md` §成功基準) 達成困難
- **代替案 B**: PE-A の代わりに Speaker Encoder (ECAPA-TDNN) の embedding を再利用
    - 利点: `src/python/piper_train/speaker_encoder/` に既存実装、追加依存なし
    - 欠点: 話者特徴と感情特徴が混在し、感情条件付けの独立性が失われる
- **代替案 C**: 実音声 (`torchaudio.utils.download_asset("...")`) を使って embedding の意味的妥当性まで確認
    - 利点: norm が縮退していないこと以上の情報が得られる
    - 欠点: 15分工数には収まらない。T03 のベンチマーク時に測るほうが合理的

### 7.2 現在の実装を選んだ理由

- Fork 実装との整合性、Phase 4 の PE-A loss (direction + centroid + margin) との 1:1 対応が必要なため、代替 embedding は採用しない
- T02 はあくまで「API が動くか」「embedding dim を確定するか」のミニマム検証であり、意味的妥当性検証は Phase 5 (fine-tune + 感情分類精度測定) で実施する

### 7.3 リファクタ機会 (将来)

- Phase 4 完了後、`_extract_embedding` ヘルパーを `src/python/piper_train/perception/pe_av_loader.py` に移行し、学習ループ・style bank 生成・PoC の三者で共有
- モデルが返す output type に応じた抽出ロジックを `dataclass` 化し、型安全性を向上
