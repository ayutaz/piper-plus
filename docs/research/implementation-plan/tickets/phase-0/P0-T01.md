## 1. タスク目的とゴール

### 1.1 目的

`facebook/pe-av-small` (arxiv:2512.19687、Meta Perception Encoder Audio-Visual) を HuggingFace Hub から `transformers.AutoModel.from_pretrained` 経由でロード可能か実検証する。元ドキュメント `phase-0-1.md §0.2` に記載の「Option A (第一候補): `AutoModel + trust_remote_code=True`」が機能するかを早期に確認し、失敗時の Option B (カスタムクラス直輸入) への切り替え判断を行う。

後続の Phase 1 (`--style-vector-dim` デフォルト値決定)、Phase 3 (`build_pea_style_bank.py` で使用する loader コード)、Phase 4 (PE-A loss のモデル初期化) すべてが本タスクで確定するクラス名・API 名・エントリポイントに依存する。

### 1.2 ゴール (Definition of Done)

- [ ] `src/python/piper_train/tools/test_pe_av_small.py` に `test_loading()` 関数を実装

- [ ] ロード成功時、`type(model).__name__` (例: `PeAvModel` / `PerceptionEncoderAV` など) を INFO ログに出力

## 2. 実装内容の詳細

### 2.1 対象ファイル

- `src/python/piper_train/tools/test_pe_av_small.py` (新規)

### 2.2 実装手順

1. `src/python/piper_train/tools/` に `test_pe_av_small.py` を作成し、`phase-0-1.md §0.4` の雛形を起点にする
2. `logging.basicConfig(level=logging.INFO, format="%(asctime)s - %(name)s - %(levelname)s - %(message)s")` をセットアップ
3. `test_loading()` を定義:
    - `transformers.AutoModel.from_pretrained("facebook/pe-av-small", trust_remote_code=True)` を呼び出し
    - `type(model).__name__`、`model.config.__class__.__name__` (可能なら)、利用可能メソッド (`dir(model)` のうち `get_*` / `encode_*` / `extract_*` で始まるもの) をフィルタして出力
    - 戻り値として `model` インスタンスを返す (T02/T03 での再利用を想定)
4. エントリポイントで `_LOGGER.info(f"transformers={transformers.__version__}, torch={torch.__version__}")` を出力
5. ロード失敗時の except 節で `logging.exception(...)` を呼び、Option B の指示を `_LOGGER.error(...)` でユーザに表示

### 2.3 コード例 (抜粋)

```python
import logging
import torch
import transformers
from transformers import AutoModel

_LOGGER = logging.getLogger(__name__)


def test_loading() -> torch.nn.Module:
    """Load facebook/pe-av-small via AutoModel.from_pretrained."""
    _LOGGER.info(
        "env: transformers=%s, torch=%s",
        transformers.__version__,
        torch.__version__,
    )
    _LOGGER.info("Loading facebook/pe-av-small from HF Hub...")
    try:
        model = AutoModel.from_pretrained(
            "facebook/pe-av-small",
            trust_remote_code=True,
        )
    except Exception:
        _LOGGER.exception("AutoModel.from_pretrained failed")
        _LOGGER.error(
            "Fallback (Option B): clone https://github.com/facebookresearch/perception_models"
            " and import PE-AV class directly. See phase-0-1.md §0.2."
        )
        raise

    cls_name = type(model).__name__
    _LOGGER.info("Model class: %s", cls_name)
    _LOGGER.info("config class: %s", type(getattr(model, "config", None)).__name__)

    candidate_methods = [m for m in dir(model) if m.startswith(("get_", "encode_", "extract_"))]
    _LOGGER.info("Candidate API methods: %s", candidate_methods)
    _LOGGER.info("has get_audio_embeds: %s", hasattr(model, "get_audio_embeds"))
    return model
```

## 6. 懸念事項とレビュー項目

### 6.1 懸念事項

- `trust_remote_code=True` でも動かない場合のフォールバック: `facebookresearch/perception_models` リポジトリから PE-AV クラスを手動インポート。HF Hub のモデルカードが `AutoModel` 未対応の場合、`AutoProcessor` / `AutoFeatureExtractor` の組合せが必要になる可能性あり

- HF Hub 接続失敗時 (ネットワーク・Rate Limit) の切り分け: `huggingface_hub.snapshot_download("facebook/pe-av-small")` が通るかを別途確認する手段を用意

## 7. 一から作り直すとしたら

### 7.1 代替案の検討

- **代替案 A**: HF Hub を経由せず、`facebookresearch/perception_models` GitHub リポジトリをサブモジュール化してソース同梱
    - 利点: `trust_remote_code` に依存しない、CI/オフライン環境でも再現可能
    - 欠点: リポジトリサイズ増大、upstream 追従コスト、Phase 0 時点では yak shaving
- **代替案 B**: ONNX 化済みの PE-A モデルを自前で作成し配布
    - 利点: transformers 依存を排除、ランタイム軽量
    - 欠点: Phase 4 で勾配を流して PE-A loss を計算する必要があるため、学習時は PyTorch モデルが必要。Phase 0 の範囲外
- **代替案 C**: `huggingface_hub.snapshot_download` で重みだけ取得し、Python クラスは自前で最小実装
    - 利点: 依存が薄い
    - 欠点: モデル定義の再実装コストが高く、fork コードとの 1:1 互換性が失われる

### 7.2 現在の実装を選んだ理由

- Fork (`yusuke-ai/piper-plus` コミット `314b3355`) が `AutoModel.from_pretrained` を前提としたコードを持っており、Phase 4 で fork の PE-A loss を取り込む際に loader 層を変えると差分レビューが爆発するため
- `trust_remote_code=True` は Apache-2.0 の `facebookresearch` 公式リポジトリからの取得であり、現時点で社内慣行上許容範囲
- 15分の工数目安に収めるには、最短経路の `AutoModel` 試行 → 失敗時のみフォールバック、が合理的

### 7.3 リファクタ機会 (将来)

- Phase 4 完了後、loader 層を `src/python/piper_train/perception/pe_av_loader.py` として独立モジュール化し、Phase 3 の `build_pea_style_bank.py` からも再利用できる構造にする
- CI で HF Hub への到達性チェックをスキップ可能にするため、環境変数 `PIPER_SKIP_PE_AV_LOAD=1` でローダーを mock 化するフックを追加
