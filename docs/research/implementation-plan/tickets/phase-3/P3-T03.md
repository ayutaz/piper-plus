## 1. タスク目的とゴール

### 1.1 目的

既存の dataset manifest (LibriTTS-R, MOE-Speech, 6lang 等) に `style_vector_path` と `emotion` フィールドを注入するツールを実装する。既存データセットに感情ラベルが存在しない場合は `--default-emotion neutral` で全サンプルに neutral を割り当てる。感情ラベルが CSV 形式で提供されている場合は `--emotion-csv` でマッピングをロードする。

このツールは、(1) 既存 6lang base model の fine-tune 時に style conditioning を有効化する、(2) CREMA-D 以外のデータセットに style bank の感情 centroid を割り当てる、(3) `style_vector_path` を指定して per-utterance embedding を参照させる、の 3 つの用途を持つ。

### 1.2 ゴール (Definition of Done)

- [ ] `src/python/piper_train/tools/inject_style_labels.py` (新規、~150 行) を実装
- [ ] CLI オプション: `--dataset-dir`, `--manifest`, `--style-vectors-dir`, `--emotion-csv`, `--default-emotion`, `--output-manifest`
- [ ] 既存 dataset.jsonl の各行を読み込み、`emotion` と `style_vector_path` フィールドを追加
- [ ] 感情マッピングは CSV (`utt_id,emotion` 形式) から読込可能、または `--default-emotion` で全サンプル同一感情
- [ ] `--style-vectors-dir` 指定時、各 utterance に対応する `<utt_id>.npy` が存在すれば `style_vector_path` にセット。存在しない場合は `None` (既存挙動維持)
- [ ] 感情バリデーション: CSV の emotion が style bank の `emotion_names` に含まれるかをチェック (オプション `--style-bank` 併用時)
- [ ] 冪等性: 既存 manifest に既に `emotion` や `style_vector_path` がある場合は上書き (警告ログ)
- [ ] 統計出力: 処理サンプル数、style_vector 割当成功数、スキップ数、感情分布

## 2. 実装内容の詳細

### 2.1 対象ファイル

- `src/python/piper_train/tools/inject_style_labels.py` (新規、~150 行)

### 2.2 実装手順

1. `phase-3-4.md §3.3` のコード例を起点にモジュール作成
2. `load_emotion_mapping_from_csv()`: `utt_id,emotion` 形式の CSV をロードして dict を返す
3. `inject_style_labels()`: dataset manifest (JSONL) の各行を読み込み、以下を追加:
   - `emotion` フィールド: `emotion_mapping.get(utt_id, default_emotion)` で設定
   - `style_vector_path` フィールド: `--style-vectors-dir/<utt_id>.npy` が存在すれば相対パス、なければ `None`
4. バリデーション: `--style-bank` 指定時、`.npz` の `emotion_names` を読込み、CSV の emotion がすべて含まれるかチェック
5. 出力: `--output-manifest` (指定なしなら入力 manifest を上書き)
6. 統計ログ: 更新件数、スキップ件数、感情分布 (`{"angry": 1200, "happy": 1100, ...}`)
7. CLI パースと main 関数

### 2.3 コード例 (phase-3-4.md §3.3 抜粋)

```python
#!/usr/bin/env python3
"""Inject style_vector_path and emotion fields into existing dataset manifest."""

import argparse
import json
import logging
from collections import Counter
from pathlib import Path
from typing import Optional

import numpy as np
from tqdm import tqdm

_LOGGER = logging.getLogger("inject_style_labels")


def load_emotion_mapping_from_csv(csv_path: Path) -> dict:
    mapping = {}
    with open(csv_path, "r") as f:
        for line in f:
            line = line.strip()
            if line and not line.startswith("#"):
                parts = line.split(",", 1)
                if len(parts) == 2:
                    utt_id, emotion = parts
                    mapping[utt_id.strip()] = emotion.strip().lower()
    return mapping


def validate_emotions_against_bank(
    emotion_mapping: dict, style_bank_path: Path
) -> set:
    """Return set of emotions in mapping that are not in style bank."""
    bank = np.load(str(style_bank_path), allow_pickle=True)
    bank_emotions = set(str(e) for e in bank["emotion_names"].tolist())
    mapping_emotions = set(emotion_mapping.values())
    missing = mapping_emotions - bank_emotions
    return missing


def inject_style_labels(
    dataset_dir: Path,
    manifest_path: Path,
    style_vectors_dir: Optional[Path] = None,
    emotion_mapping: Optional[dict] = None,
    default_emotion: str = "neutral",
    output_manifest: Optional[Path] = None,
):
    dataset_dir = Path(dataset_dir)
    manifest_path = Path(manifest_path)
    output_manifest = Path(output_manifest or manifest_path)
    emotion_mapping = emotion_mapping or {}

    updated_count = 0
    skipped_count = 0
    emotion_counter = Counter()

    with open(manifest_path, "r") as f:
        original_lines = [line.strip() for line in f if line.strip()]

    updated_lines = []
    for line in tqdm(original_lines, desc="Injecting labels"):
        try:
            item = json.loads(line)
            audio_path = item.get("audio_norm_path") or item.get("audio_path")
            utt_id = Path(audio_path).stem if audio_path else None

            emotion = None
            if utt_id and utt_id in emotion_mapping:
                emotion = emotion_mapping[utt_id]
            elif audio_path and audio_path in emotion_mapping:
                emotion = emotion_mapping[audio_path]
            else:
                emotion = default_emotion
            item["emotion"] = emotion
            emotion_counter[emotion] += 1

            if style_vectors_dir and utt_id:
                style_vec_path = Path(style_vectors_dir) / f"{utt_id}.npy"
                if style_vec_path.exists():
                    item["style_vector_path"] = str(
                        style_vec_path.relative_to(dataset_dir)
                    )
                    updated_count += 1
                else:
                    item["style_vector_path"] = None
                    skipped_count += 1
            else:
                item["style_vector_path"] = None

            updated_lines.append(json.dumps(item, ensure_ascii=False))
        except Exception as e:
            _LOGGER.warning("Failed to process line: %s", e)

    with open(output_manifest, "w") as f:
        for line in updated_lines:
            f.write(line + "\n")

    _LOGGER.info(
        "Injection complete: %d with style_vector, %d skipped, %d total",
        updated_count, skipped_count, len(updated_lines),
    )
    _LOGGER.info("Emotion distribution: %s", dict(emotion_counter))


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--dataset-dir", type=Path, required=True)
    parser.add_argument("--manifest", type=Path, required=True)
    parser.add_argument("--style-vectors-dir", type=Path, default=None)
    parser.add_argument("--emotion-csv", type=Path, default=None)
    parser.add_argument("--default-emotion", type=str, default="neutral")
    parser.add_argument("--output-manifest", type=Path, default=None)
    parser.add_argument("--style-bank", type=Path, default=None,
                        help="Optional: validate emotions against .npz style bank")
    args = parser.parse_args()

    logging.basicConfig(level=logging.INFO)

    emotion_mapping = {}
    if args.emotion_csv:
        emotion_mapping = load_emotion_mapping_from_csv(args.emotion_csv)
        _LOGGER.info("Loaded %d emotion mappings", len(emotion_mapping))

    if args.style_bank and emotion_mapping:
        missing = validate_emotions_against_bank(emotion_mapping, args.style_bank)
        if missing:
            _LOGGER.warning(
                "Emotions not in style bank: %s", missing
            )

    inject_style_labels(
        args.dataset_dir, args.manifest,
        style_vectors_dir=args.style_vectors_dir,
        emotion_mapping=emotion_mapping,
        default_emotion=args.default_emotion,
        output_manifest=args.output_manifest,
    )


if __name__ == "__main__":
    main()
```

## 6. 懸念事項とレビュー項目

### 6.1 懸念事項

- **大規模 manifest の処理時間**: 6lang manifest は 508,187 行。JSON パース + 書き出しで 1〜2 分、メモリに全行ロードするとピーク時 1〜2 GB 消費。ストリーミング処理 (行単位書き出し) にすべきか検討
- **相対パス vs 絶対パス**: `style_vector_path` を相対パスで保存すると dataset_dir 移動時に破綻しない。絶対パスは検索ツール (grep 等) での置換が容易。`Path.relative_to()` でデフォルト相対パスとする
- **utt_id の抽出ロジック**: `audio_norm_path` vs `audio_path` の使い分けが dataset によって異なる。両方を試みて、先に見つかった方を採用するフォールバック
- **既存 emotion フィールドの上書き**: 既存 manifest に `emotion` が既にある場合、警告ログ + 上書き。`--no-overwrite` オプションで skip も検討
- **JSON encoding**: `json.dumps(item)` のデフォルトは ASCII escape。日本語テキストを含む場合は `ensure_ascii=False` を指定
- **utt_id の一意性**: 複数 speaker で同じ sentence id を持つ場合 (LibriTTS-R の Chapter-Utterance 構造)、`utt_id` が衝突する可能性。`speaker_id/utt_id` 結合キーの検討

### 6.2 レビュー項目

## 7. 一から作り直すとしたら

### 7.1 代替案の検討

- **代替案 A**: pandas ベースの実装
    - 利点: CSV/JSONL 操作が簡潔、`pd.read_csv`, `pd.merge` で結合が容易
    - 欠点: pandas 依存追加、大規模 manifest で memory overhead が大きい

- **代替案 B**: SQLite ベース
    - 利点: 大規模 dataset (数百万サンプル) での索引アクセスが高速、後続ツールでの query が容易
    - 欠点: JSONL 互換性の維持が必要 (piper_train の既存入力形式)、工数が 1h を超える

- **代替案 C**: MOE-Speech などの話者 embedding を再利用して style_vector 生成
    - 利点: PE-A 抽出の計算コストを削減、既存 speaker embedding と style の統合
    - 欠点: speaker embedding と emotion embedding は別空間なので単純な流用は不可
    - 適用: speaker × emotion の共同表現を学習する別機能として検討

- **代替案 D**: YAML/TOML manifest への移行
    - 利点: 人間可読、メタデータ (license, source, emotion_ontology) を構造化
    - 欠点: JSONL との互換性を壊す、piper_train の既存実装を書き換え必要

### 7.2 現在の実装を選んだ理由

- 既存 piper_train の manifest 形式 (JSONL) に非破壊で追加フィールドを注入できる
- pandas 等の追加依存なしで標準ライブラリだけで実装可能 (1h 工数に収まる)
- `phase-3-4.md §3.3` でコード例が完全提示されており、実装リスクが最小化されている
- `--default-emotion neutral` で既存 dataset を即座に style-aware 化できる (fine-tune 開始に必要)

### 7.3 リファクタ機会 (将来)
