## 1. タスク目的とゴール

### 1.1 目的

P3-T02 で生成された `.npz` style bank の完全性を検証するスクリプトと、ユーザー向けドキュメントを作成する。検証の目的は、(1) fork `yusuke-ai/piper-plus` の PE-A loss と byte-for-byte 互換性があるか、(2) 数値的に正しい centroid が含まれているか、(3) Phase 4 の `_init_pea_emotion_loss()` が読み込み可能な形式か、を保証すること。

ドキュメントでは、(1) `.npz` スキーマ詳細、(2) CLI 使用例、(3) CREMA-D 以外の dataset 追加ガイド (ESD/EmoV-DB/JTES) を提供する。これにより、ユーザーが自前感情データから style bank を生成できるようにする。

### 1.2 ゴール (Definition of Done)

- [ ] `src/python/piper_train/tools/validate_style_bank.py` (新規、~150 行) を実装
- [ ] 検証項目:
    - [ ] `.npz` に必須キー (`emotion_names`, `emotion_centroids`, `global_centroid`) が存在
    - [ ] `emotion_names` は str list、`emotion_centroids` は float32 `[N, D]`、`global_centroid` は float32 `[D]`
    - [ ] `emotion_centroids.shape[0] == len(emotion_names)` (次元一致)
    - [ ] `emotion_centroids.shape[1] == global_centroid.shape[0]` (D 一致)
    - [ ] 各行の `emotion_centroids` の L2 norm が 1.0 ± 1e-3
    - [ ] `global_centroid` が `emotion_centroids` の行平均と近似 (cos similarity > 0.99)
    - [ ] 感情名に重複・空文字列・空白が含まれない
- [ ] CLI オプション: `--style-bank`, `--strict`, `--expected-emotions`, `--expected-dim`

- [ ] Exit code: 0 (成功) / 1 (検証失敗) で CI 組込み可能

### 2.1 対象ファイル

### 2.2 実装手順 (validate_style_bank.py)

1. `argparse` で CLI オプションをセットアップ
2. `load_style_bank()`: `np.load(allow_pickle=True)` で `.npz` をロード
3. `check_schema()`: 必須キーの存在、dtype、shape の一致を検証
4. `check_l2_norm()`: `emotion_centroids` の各行の L2 norm を計算、1.0 ± tolerance に収まるか確認
5. `check_global_centroid_consistency()`: `global_centroid` と `emotion_centroids.mean(axis=0)` の cosine similarity を計算 (ただし、P3-T02 の実装では `global_centroid` は全 embedding の raw mean で centroid の mean ではない。両方のパターンに対応)
6. `check_emotion_names_uniqueness()`: 重複チェック、空文字列チェック
7. `--strict` モードでは warning も error 扱い
8. レポート出力: 検証項目ごとの pass/fail と詳細

### 2.3 コード例

```python
#!/usr/bin/env python3
"""Validate PE-A style bank .npz file schema and numerical correctness."""

import argparse
import logging
import sys
from pathlib import Path
from typing import Optional

import numpy as np

_LOGGER = logging.getLogger("validate_style_bank")

REQUIRED_KEYS = ["emotion_names", "emotion_centroids", "global_centroid"]
L2_NORM_TOLERANCE = 1e-3
GLOBAL_CENTROID_COSSIM_THRESHOLD = 0.99


def load_style_bank(path: Path) -> dict:
    bank = np.load(str(path), allow_pickle=True)
    return {key: bank[key] for key in bank.files}


def check_schema(bank: dict) -> list:
    errors = []
    for key in REQUIRED_KEYS:
        if key not in bank:
            errors.append(f"Missing key: {key}")

    if "emotion_names" in bank:
        names = bank["emotion_names"]
        if names.dtype != object:
            errors.append(
                f"emotion_names dtype must be object (str), got {names.dtype}"
            )

    if "emotion_centroids" in bank:
        centroids = bank["emotion_centroids"]
        if centroids.dtype != np.float32:
            errors.append(
                f"emotion_centroids dtype must be float32, got {centroids.dtype}"
            )
        if centroids.ndim != 2:
            errors.append(
                f"emotion_centroids must be 2D, got {centroids.ndim}D"
            )

    if "global_centroid" in bank:
        global_c = bank["global_centroid"]
        if global_c.dtype != np.float32:
            errors.append(
                f"global_centroid dtype must be float32, got {global_c.dtype}"
            )
        if global_c.ndim != 1:
            errors.append(
                f"global_centroid must be 1D, got {global_c.ndim}D"
            )

    if "emotion_centroids" in bank and "emotion_names" in bank:
        if bank["emotion_centroids"].shape[0] != len(bank["emotion_names"]):
            errors.append(
                "Mismatch: emotion_centroids.shape[0]={} vs len(emotion_names)={}".format(
                    bank["emotion_centroids"].shape[0], len(bank["emotion_names"])
                )
            )

    if "emotion_centroids" in bank and "global_centroid" in bank:
        if bank["emotion_centroids"].shape[1] != bank["global_centroid"].shape[0]:
            errors.append(
                "Embedding dim mismatch: emotion_centroids={}, global_centroid={}".format(
                    bank["emotion_centroids"].shape[1],
                    bank["global_centroid"].shape[0],
                )
            )

    return errors


def check_l2_norm(centroids: np.ndarray, tolerance: float = L2_NORM_TOLERANCE) -> list:
    errors = []
    norms = np.linalg.norm(centroids, axis=-1)
    for i, norm in enumerate(norms):
        if abs(norm - 1.0) > tolerance:
            errors.append(
                f"Row {i} L2 norm = {norm:.6f} (expected 1.0 ± {tolerance})"
            )
    return errors


def check_global_centroid_consistency(
    centroids: np.ndarray, global_centroid: np.ndarray
) -> list:
    """Check if global_centroid is close to mean of emotion_centroids."""
    mean_of_centroids = centroids.mean(axis=0)
    # Cosine similarity
    cos_sim = np.dot(mean_of_centroids, global_centroid) / (
        np.linalg.norm(mean_of_centroids) * np.linalg.norm(global_centroid) + 1e-10
    )
    warnings = []
    if cos_sim < GLOBAL_CENTROID_COSSIM_THRESHOLD:
        warnings.append(
            "global_centroid vs mean(emotion_centroids) cos_sim = {:.4f} "
            "(< {:.2f}). This is expected if global_centroid is computed from "
            "raw (non-normalized) embeddings.".format(
                cos_sim, GLOBAL_CENTROID_COSSIM_THRESHOLD
            )
        )
    return warnings


def check_emotion_names(names) -> list:
    errors = []
    str_names = [str(n) for n in names]
    if len(set(str_names)) != len(str_names):
        errors.append(f"Duplicate emotion names found: {str_names}")
    for i, name in enumerate(str_names):
        if not name or name.isspace():
            errors.append(f"emotion_names[{i}] is empty or whitespace")
    return errors


def validate(
    style_bank_path: Path,
    strict: bool = False,
    expected_emotions: Optional[list] = None,
    expected_dim: Optional[int] = None,
) -> int:
    _LOGGER.info("Validating: %s", style_bank_path)
    bank = load_style_bank(style_bank_path)

    schema_errors = check_schema(bank)
    if schema_errors:
        for err in schema_errors:
            _LOGGER.error(err)
        return 1  # Schema errors are always fatal

    names = [str(n) for n in bank["emotion_names"]]
    centroids = bank["emotion_centroids"]
    global_c = bank["global_centroid"]

    _LOGGER.info("  emotion_names: %s", names)
    _LOGGER.info("  emotion_centroids.shape: %s", centroids.shape)
    _LOGGER.info("  global_centroid.shape: %s", global_c.shape)

    if expected_emotions:
        missing = set(expected_emotions) - set(names)
        if missing:
            _LOGGER.error("Missing expected emotions: %s", missing)
            if strict:
                return 1

    if expected_dim and centroids.shape[1] != expected_dim:
        _LOGGER.error(
            "Embedding dim %d != expected %d", centroids.shape[1], expected_dim
        )
        return 1

    name_errors = check_emotion_names(bank["emotion_names"])
    for err in name_errors:
        _LOGGER.error(err)
    if name_errors:
        return 1

    norm_errors = check_l2_norm(centroids)
    if norm_errors:
        for err in norm_errors:
            _LOGGER.error(err)
        return 1

    consistency_warnings = check_global_centroid_consistency(centroids, global_c)
    for warn in consistency_warnings:
        if strict:
            _LOGGER.error(warn)
        else:
            _LOGGER.warning(warn)
    if strict and consistency_warnings:
        return 1

    _LOGGER.info("Validation PASSED")
    return 0


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--style-bank", type=Path, required=True)
    parser.add_argument("--strict", action="store_true")
    parser.add_argument("--expected-emotions", nargs="+", default=None)
    parser.add_argument("--expected-dim", type=int, default=None)
    args = parser.parse_args()

    logging.basicConfig(level=logging.INFO, format="%(levelname)s - %(message)s")

    exit_code = validate(
        args.style_bank,
        strict=args.strict,
        expected_emotions=args.expected_emotions,
        expected_dim=args.expected_dim,
    )
    sys.exit(exit_code)


if __name__ == "__main__":
    main()
```

## 6. 懸念事項とレビュー項目

### 6.1 懸念事項

- **global_centroid の仕様揺れ**: P3-T02 実装では `global_centroid` は全 embedding (L2-normalize 済み) の raw mean。fork の実装では `F.normalize()` が適用された後にbufferに登録される。両方のパターンをサポートする必要あり (警告のみで error にしない)
- **NaN / Inf 検出**: `emotion_centroids` に NaN が含まれていると PE-A loss 計算で NaN 伝播する。本検証で必ず検出する
- **dtype の bytes vs str**: `emotion_names` は numpy object dtype で保存するが、np.bytes_ で保存されるか np.str_ で保存されるかが環境依存。両方を str 変換で受け入れる実装
- **ドキュメントの陳腐化**: `docs/features/style-bank.md` が CLI オプション変更と同期しないリスク。CLI の `--help` 出力と整合性を取る自動テストを検討
- **ESD のライセンス**: `peav-style-conditioning.md §12` では ESD = 研究目的。商用利用でこの style bank を使うと ESD の利用規約違反になる可能性。ドキュメントで明記

## 7. 一から作り直すとしたら

### 7.1 代替案の検討

- **代替案 A**: JSON Schema 形式での検証
    - 利点: 標準化された検証フレームワーク、他言語からの検証も可能
    - 欠点: JSON Schema では numpy 特有の dtype (float32 vs float64) や shape 検証が表現しにくい
    - 適用: meta情報 (report.json) の検証に限定

- **代替案 B**: pydantic モデルでの検証
    - 利点: Python 型アノテーションと一体化、IDE 補完
    - 欠点: pydantic 依存追加、numpy 配列の型表現が冗長

- **代替案 C**: ビジュアル検証ツール追加 (UMAP / t-SNE で 2D プロット)
    - 利点: 感情クラスタの分離度を視覚確認
    - 欠点: スコープ拡大 (工数 1h を超える)、依存パッケージ増加
    - 適用: 別チケット or Phase 5 評価ツールに含める

- **代替案 D**: `pytest` ベースの検証 (assertion を書く形式)
    - 利点: 既存テストフレームワークと統合、CI 組込みが容易
    - 欠点: CLI tool としての独立性が失われる、ユーザーが `pytest` を学ぶ必要がある

### 7.2 現在の実装を選んだ理由

- 単一のスクリプトで exit code 経由で CI 組込みできるシンプルな設計
- `np.load` + assertion だけで数値検証が完結し、依存パッケージが最小
- ドキュメント (`docs/features/style-bank.md`) と組み合わせることで、ユーザーへの使い方説明と CI 検証を同一ツールで実現
- 1h 工数に収まる範囲で最も価値が高い (検証項目の充実度 × ドキュメント完全性)
